## 1. 数组与哈希表

### 1.1 数组底层原理

#### 为什么数组查找是O(1)？

**内存布局**：
```
数组: [10, 20, 30, 40, 50]
内存地址：
  arr[0] → 0x1000  (存储10)
  arr[1] → 0x1004  (存储20，假设int占4字节)
  arr[2] → 0x1008  (存储30)
  arr[3] → 0x100C  (存储40)
  arr[4] → 0x1010  (存储50)
```

**访问arr[i]的计算**：
```
地址 = base_address + i * element_size
     = 0x1000 + 2 * 4
     = 0x1008  (arr[2])
```

**为什么是O(1)？**
- 数组在内存中连续存储
- 通过简单的**数学计算**直接得到地址
- 不需要遍历，只需要一次内存访问

---

#### 数组 vs 链表

| 特性 | 数组 | 链表 |
|------|------|------|
| **随机访问** | O(1) - 通过索引直接计算地址 | O(n) - 需要从头遍历 |
| **插入/删除（头部）** | O(n) - 需要移动后续元素 | O(1) - 只需修改指针 |
| **插入/删除（末尾）** | O(1)* - 直接添加 | O(n) - 需要遍历到末尾 |
| **内存** | 连续 - 缓存友好 | 分散 - 缓存不友好 |
| **空间利用** | 可能浪费（预分配） | 每个节点需要额外指针 |

**工程中如何选择？**
- **数组**：需要频繁随机访问（如排序、二分查找）
- **链表**：频繁插入/删除，且主要在头部操作（如LRU缓存）

### 1.2 动态数组实现

#### 为什么需要动态数组？

静态数组大小固定，动态数组可以自动扩容。

#### 手写动态数组

```python
class ArrayList:
    """
    动态数组实现
    
    扩容策略：容量不足时，扩容到2倍
    """
    def __init__(self):
        self.capacity = 10  # 初始容量
        self.size = 0       # 当前元素个数
        self.data = [None] * self.capacity
    
    def append(self, value):
        """
        添加元素
        时间复杂度：均摊O(1)
        """
        # 容量不足，扩容
        if self.size == self.capacity:
            self._resize(self.capacity * 2)
        
        self.data[self.size] = value
        self.size += 1
    
    def _resize(self, new_capacity):
        """
        扩容：创建新数组，复制旧数据
        时间复杂度：O(n)
        """
        new_data = [None] * new_capacity
        for i in range(self.size):
            new_data[i] = self.data[i]
        self.data = new_data
        self.capacity = new_capacity
    
    def get(self, index):
        """访问元素：O(1)"""
        if index < 0 or index >= self.size:
            raise IndexError("Index out of range")
        return self.data[index]
    
    def insert(self, index, value):
        """
        插入元素：O(n)
        需要移动index后的所有元素
        """
        if self.size == self.capacity:
            self._resize(self.capacity * 2)
        
        # 从后往前移动元素
        for i in range(self.size, index, -1):
            self.data[i] = self.data[i - 1]
        
        self.data[index] = value
        self.size += 1
    
    def remove(self, index):
        """
        删除元素：O(n)
        需要移动index后的所有元素
        """
        if index < 0 or index >= self.size:
            raise IndexError("Index out of range")
        
        # 向前移动元素
        for i in range(index, self.size - 1):
            self.data[i] = self.data[i + 1]
        
        self.size -= 1
        self.data[self.size] = None  # 清理引用
        
        # 缩容：元素个数小于容量1/4时，缩容到1/2
        if self.size < self.capacity // 4 and self.capacity > 10:
            self._resize(self.capacity // 2)
```

#### 为什么append是"均摊O(1)"？

**扩容分析**：
```
初始容量：10
添加第1-10个元素：无需扩容，10次O(1)
添加第11个元素：扩容到20，复制10个元素，O(10)
添加第12-20个元素：无需扩容，9次O(1)
添加第21个元素：扩容到40，复制20个元素，O(20)
...

总成本：n + (10 + 20 + 40 + ...) ≈ n + 2n = 3n
均摊成本：3n / n = O(1)
```

**为什么扩容到2倍？**
- 如果扩容到n+1：每次添加都需要扩容，O(n²)
- 如果扩容到1.5倍或2倍：均摊O(1)
- Python/Java使用2倍，Go使用1.25倍（Go针对大数组优化）

### 1.3 哈希表底层原理

#### 为什么哈希表查找是O(1)？

**核心思想**：通过哈希函数将key映射到数组索引

```
key → hash(key) → index → value

例如：
"apple" → hash("apple") = 123456 → 123456 % 10 = 6 → bucket[6]
```

#### 哈希函数设计

**好的哈希函数特点**：
1. **确定性**：相同key必须产生相同hash
2. **均匀分布**：不同key应尽量映射到不同位置
3. **快速计算**：O(1)时间

**常见哈希函数**：

```python
def hash_string(s, table_size):
    """
    字符串哈希（多项式滚动哈希）
    """
    hash_value = 0
    p = 31  # 质数
    power = 1
    
    for char in s:
        hash_value = (hash_value + ord(char) * power) % table_size
        power = (power * p) % table_size
    
    return hash_value

def hash_int(key, table_size):
    """整数哈希（除留余数法）"""
    return key % table_size
```

#### 哈希冲突解决

**问题**：不同key可能映射到同一个索引

**解决方案1：链地址法（Chaining）**

```python
class HashMapChaining:
    """
    链地址法：每个bucket存储一个链表
    
    时间复杂度：
    - 最好：O(1) - 无冲突
    - 平均：O(1 + α) - α为负载因子（n/m）
    - 最坏：O(n) - 所有key映射到同一个bucket
    """
    def __init__(self, capacity=16):
        self.capacity = capacity
        self.size = 0
        self.buckets = [[] for _ in range(capacity)]
    
    def _hash(self, key):
        """哈希函数"""
        return hash(key) % self.capacity
    
    def put(self, key, value):
        """插入/更新"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        # 查找是否已存在
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 更新
                return
        
        # 不存在，添加
        bucket.append((key, value))
        self.size += 1
        
        # 负载因子过高，扩容
        if self.size / self.capacity > 0.75:
            self._resize()
    
    def get(self, key):
        """查找"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        for k, v in bucket:
            if k == key:
                return v
        
        raise KeyError(key)
    
    def remove(self, key):
        """删除"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                self.size -= 1
                return
        
        raise KeyError(key)
    
    def _resize(self):
        """扩容：重新哈希所有元素"""
        old_buckets = self.buckets
        self.capacity *= 2
        self.buckets = [[] for _ in range(self.capacity)]
        self.size = 0
        
        for bucket in old_buckets:
            for key, value in bucket:
                self.put(key, value)
```

**链地址法优缺点**：
- ✅ 实现简单
- ✅ 删除方便
- ❌ 需要额外指针空间
- ❌ 缓存不友好（链表节点分散）

**解决方案2：开放寻址法（Open Addressing）**

```python
class HashMapOpenAddressing:
    """
    开放寻址法：线性探测
    
    冲突时，向后查找下一个空位：
    index, index+1, index+2, ...
    """
    def __init__(self, capacity=16):
        self.capacity = capacity
        self.size = 0
        self.keys = [None] * capacity
        self.values = [None] * capacity
        self.DELETED = object()  # 删除标记
    
    def _hash(self, key):
        return hash(key) % self.capacity
    
    def put(self, key, value):
        """插入/更新"""
        if self.size / self.capacity > 0.5:  # 负载因子阈值
            self._resize()
        
        index = self._hash(key)
        
        # 线性探测
        while self.keys[index] is not None:
            if self.keys[index] == key:
                self.values[index] = value  # 更新
                return
            
            if self.keys[index] == self.DELETED:
                break  # 可以复用删除位置
            
            index = (index + 1) % self.capacity  # 环形查找
        
        self.keys[index] = key
        self.values[index] = value
        self.size += 1
    
    def get(self, key):
        """查找"""
        index = self._hash(key)
        
        # 线性探测查找
        while self.keys[index] is not None:
            if self.keys[index] == key:
                return self.values[index]
            index = (index + 1) % self.capacity
        
        raise KeyError(key)
    
    def remove(self, key):
        """删除：标记为DELETED，不能直接置None"""
        index = self._hash(key)
        
        while self.keys[index] is not None:
            if self.keys[index] == key:
                self.keys[index] = self.DELETED
                self.values[index] = None
                self.size -= 1
                return
            index = (index + 1) % self.capacity
        
        raise KeyError(key)
```

**为什么删除要用标记而不是直接置None？**

```
假设：hash("a") = hash("b") = 0
插入顺序：a → bucket[0], b → bucket[1]（冲突后探测）

如果删除a时直接置None：
bucket[0] = None
查找b时：hash("b")=0，bucket[0]=None，误认为b不存在！

正确做法：
bucket[0] = DELETED
查找b时：遇到DELETED继续探测，找到bucket[1]
```

**开放寻址法优缺点**：
- ✅ 缓存友好（连续内存）
- ✅ 节省指针空间
- ❌ 删除复杂（需要标记）
- ❌ 负载因子不能太高（否则探测链很长）

---

#### 负载因子与性能

**负载因子** α = n / m（n为元素个数，m为bucket数）

| 负载因子 | 链地址法平均查找次数 | 开放寻址法平均查找次数 |
|----------|---------------------|----------------------|
| 0.5 | 1.25 | 1.5 |
| 0.75 | 1.5 | 3.0 |
| 1.0 | 2.0 | ∞（理论上） |

**工程中的选择**：
- **Java HashMap**：链地址法，α=0.75时扩容，链表长度>8转红黑树
- **Python dict**：开放寻址法，α=2/3时扩容
- **Redis dict**：链地址法，渐进式rehash避免阻塞

**哈希表的问题**：

**哈希索引的使用场景**：
- 等值查询：`WHERE id = 123`
- Memory引擎
- 自适应哈希索引（InnoDB）

#### 应用2：Redis的dict

**为什么Redis用渐进式rehash？**

```python
# 普通rehash：一次性重新哈希所有key，可能阻塞很久
def rehash_blocking(old_dict):
    new_dict = create_new_dict(old_dict.size * 2)
    for key, value in old_dict:  # ⚠️ 可能有100万个key
        new_dict.put(key, value)
    return new_dict  # 阻塞时间：O(n)

# 渐进式rehash：每次操作时迁移少量key
class RedisDict:
    def __init__(self):
        self.ht = [HashTable(), HashTable()]  # 两个哈希表
        self.rehashidx = -1  # -1表示未在rehash
    
    def put(self, key, value):
        # 正在rehash，先迁移一个bucket
        if self.rehashidx != -1:
            self._rehash_step()
        
        # 正常插入
        index = 0 if self.rehashidx == -1 else 1
        self.ht[index].put(key, value)
        
        # 触发rehash
        if self.should_resize():
            self._start_rehash()
    
    def _rehash_step(self):
        """迁移一个bucket"""
        bucket = self.ht[0].buckets[self.rehashidx]
        for key, value in bucket:
            self.ht[1].put(key, value)
        
        self.ht[0].buckets[self.rehashidx] = []
        self.rehashidx += 1
        
        # 迁移完成
        if self.rehashidx >= len(self.ht[0].buckets):
            self.ht[0] = self.ht[1]
            self.ht[1] = HashTable()
            self.rehashidx = -1
```

**优点**：
- 每次操作只迁移少量数据
- 避免长时间阻塞
- 适合单线程模型

#### 应用3：一致性哈希（分布式缓存）

**问题**：普通哈希在节点变化时，大量key需要迁移

```python
# 普通哈希：节点变化时，几乎所有key都要迁移
node_index = hash(key) % num_nodes

# 3个节点时："user123" → hash % 3 = 1 → node1
# 4个节点时："user123" → hash % 4 = 3 → node3  ⚠️ 迁移！
```

**一致性哈希**：

```python
class ConsistentHash:
    """
    一致性哈希：节点变化时，只有相邻节点的key需要迁移
    """
    def __init__(self, nodes, virtual_nodes=150):
        self.ring = {}  # 哈希环
        self.sorted_keys = []
        self.virtual_nodes = virtual_nodes
        
        for node in nodes:
            self.add_node(node)
    
    def add_node(self, node):
        """添加节点：创建virtual_nodes个虚拟节点"""
        for i in range(self.virtual_nodes):
            virtual_key = f"{node}#{i}"
            hash_value = hash(virtual_key)
            self.ring[hash_value] = node
            self.sorted_keys.append(hash_value)
        
        self.sorted_keys.sort()
    
    def get_node(self, key):
        """查找key应该存储在哪个节点"""
        if not self.ring:
            return None
        
        hash_value = hash(key)
        
        # 二分查找：找第一个>=hash_value的节点
        idx = bisect.bisect_right(self.sorted_keys, hash_value)
        if idx == len(self.sorted_keys):
            idx = 0  # 环形
        
        return self.ring[self.sorted_keys[idx]]
```

**为什么需要虚拟节点？**
- 物理节点少时，分布不均匀
- 虚拟节点让key分布更均匀

## 2. 链表

### 2.1 链表底层原理

#### 为什么链表插入是O(1)？

**数组插入**：
```
数组: [1, 2, 3, 4, 5]
插入6到索引2：
1. 移动元素：[1, 2, _, 3, 4, 5] ← 需要移动3个元素
2. 插入：[1, 2, 6, 3, 4, 5]
时间复杂度：O(n)
```

**链表插入**：
```
链表: 1 → 2 → 3 → 4 → 5
插入6到节点2后：
1. new_node.next = node2.next  // 6 → 3
2. node2.next = new_node        // 2 → 6
时间复杂度：O(1)（假设已经找到node2）
```

**为什么？**
- 链表节点在内存中不连续
- 插入只需修改指针，不需要移动元素

### 2.2 手写链表

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class LinkedList:
    """单向链表"""
    def __init__(self):
        self.head = None
        self.size = 0
    
    def add_first(self, val):
        """头部插入：O(1)"""
        new_node = ListNode(val, self.head)
        self.head = new_node
        self.size += 1
    
    def add_last(self, val):
        """尾部插入：O(n)"""
        if not self.head:
            self.head = ListNode(val)
        else:
            curr = self.head
            while curr.next:  # 遍历到末尾
                curr = curr.next
            curr.next = ListNode(val)
        self.size += 1
    
    def remove_first(self):
        """头部删除：O(1)"""
        if not self.head:
            raise Exception("List is empty")
        self.head = self.head.next
        self.size -= 1
    
    def get(self, index):
        """访问元素：O(n)"""
        if index < 0 or index >= self.size:
            raise IndexError()
        
        curr = self.head
        for _ in range(index):
            curr = curr.next
        return curr.val
```

#### 为什么需要哨兵节点（Dummy Node）？

**问题**：处理头节点时需要特殊判断

```python
# 没有哨兵节点：需要判断head
def remove_value(head, val):
    # 特殊处理头节点
    while head and head.val == val:
        head = head.next
    
    # 处理其他节点
    curr = head
    while curr and curr.next:
        if curr.next.val == val:
            curr.next = curr.next.next
        else:
            curr = curr.next
    return head

# 有哨兵节点：统一处理
def remove_value(head, val):
    dummy = ListNode(0, head)  # 哨兵节点
    curr = dummy
    
    while curr.next:
        if curr.next.val == val:
            curr.next = curr.next.next
        else:
            curr = curr.next
    
    return dummy.next  # 返回真正的头节点
```

### 2.3 双向链表

```python
class DListNode:
    def __init__(self, val=0, prev=None, next=None):
        self.val = val
        self.prev = prev
        self.next = next

class DoublyLinkedList:
    """
    双向链表：支持O(1)尾部操作
    
    使用哨兵节点简化边界处理
    """
    def __init__(self):
        self.head = DListNode()  # 哨兵头
        self.tail = DListNode()  # 哨兵尾
        self.head.next = self.tail
        self.tail.prev = self.head
        self.size = 0
    
    def add_first(self, val):
        """头部插入：O(1)"""
        self._insert_after(self.head, val)
    
    def add_last(self, val):
        """尾部插入：O(1)"""
        self._insert_before(self.tail, val)
    
    def _insert_after(self, node, val):
        """在node后插入"""
        new_node = DListNode(val, node, node.next)
        node.next.prev = new_node
        node.next = new_node
        self.size += 1
    
    def _insert_before(self, node, val):
        """在node前插入"""
        new_node = DListNode(val, node.prev, node)
        node.prev.next = new_node
        node.prev = new_node
        self.size += 1
    
    def remove_first(self):
        """删除头节点：O(1)"""
        if self.size == 0:
            raise Exception("List is empty")
        self._remove_node(self.head.next)
    
    def remove_last(self):
        """删除尾节点：O(1)"""
        if self.size == 0:
            raise Exception("List is empty")
        self._remove_node(self.tail.prev)
    
    def _remove_node(self, node):
        """删除指定节点：O(1)"""
        node.prev.next = node.next
        node.next.prev = node.prev
        self.size -= 1
```

### 2.4 链表工程应用

#### 应用1：LRU缓存（哈希表+双向链表）

**为什么需要双向链表？**
- 需要O(1)删除任意节点
- 单向链表删除节点需要O(n)找到前驱节点

```python
class LRUCache:
    """
    LRU缓存：
    - 哈希表：O(1)查找节点
    - 双向链表：O(1)移动节点到头部/删除尾部
    
    结构：
    HashMap: key → Node
    DoublyLinkedList: head ↔ node1 ↔ node2 ↔ ... ↔ tail
    （最近使用的在head，最久未使用的在tail）
    """
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key → Node
        self.head = DListNode()  # 哨兵头（最近使用）
        self.tail = DListNode()  # 哨兵尾（最久未使用）
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def get(self, key: int) -> int:
        """
        获取值：O(1)
        1. 哈希表查找节点
        2. 移动到头部（标记为最近使用）
        """
        if key not in self.cache:
            return -1
        
        node = self.cache[key]
        self._move_to_head(node)
        return node.val
    
    def put(self, key: int, value: int) -> None:
        """
        插入/更新：O(1)
        """
        if key in self.cache:
            # 已存在，更新并移到头部
            node = self.cache[key]
            node.val = value
            self._move_to_head(node)
        else:
            # 不存在，插入
            if len(self.cache) >= self.capacity:
                # 容量已满，删除尾部（LRU）
                removed = self._remove_tail()
                del self.cache[removed.key]
            
            # 创建新节点，插入头部
            new_node = DListNode(key, value)
            self.cache[key] = new_node
            self._add_to_head(new_node)
    
    def _move_to_head(self, node):
        """移动节点到头部"""
        self._remove_node(node)
        self._add_to_head(node)
    
    def _remove_node(self, node):
        """删除节点：O(1)"""
        node.prev.next = node.next
        node.next.prev = node.prev
    
    def _add_to_head(self, node):
        """添加到头部：O(1)"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
    
    def _remove_tail(self):
        """删除尾部节点：O(1)"""
        node = self.tail.prev
        self._remove_node(node)
        return node
```

**时间复杂度分析**：
- `get(key)`: O(1) - 哈希表查找 + 链表移动
- `put(key, value)`: O(1) - 哈希表插入 + 链表操作

**如果只用链表（不用哈希表）**：
- `get(key)`: O(n) - 需要遍历链表查找
- **哈希表的作用：将查找从O(n)优化到O(1)**

### 3.1 栈底层原理

#### 为什么栈是LIFO（后进先出）？

**内存模型**：
```
栈顶 →  |  30  |  ← push(30)最后进
        |  20  |  ← push(20)
        |  10  |  ← push(10)最先进
栈底
```

**操作**：
- `push(x)`: 栈顶指针+1，存储x
- `pop()`: 返回栈顶元素，栈顶指针-1

**为什么push/pop是O(1)？**
- 只操作栈顶，不涉及其他元素

### 3.2 手写栈

```python
class ArrayStack:
    """
    数组实现栈
    """
    def __init__(self, capacity=10):
        self.data = [None] * capacity
        self.top = -1  # 栈顶指针
        self.capacity = capacity
    
    def push(self, val):
        """入栈：O(1)"""
        if self.top + 1 >= self.capacity:
            raise Exception("Stack overflow")
        self.top += 1
        self.data[self.top] = val
    
    def pop(self):
        """出栈：O(1)"""
        if self.is_empty():
            raise Exception("Stack underflow")
        val = self.data[self.top]
        self.top -= 1
        return val
    
    def peek(self):
        """查看栈顶：O(1)"""
        if self.is_empty():
            raise Exception("Stack is empty")
        return self.data[self.top]
    
    def is_empty(self):
        return self.top == -1
```

#### 应用2：表达式求值

```python
def evaluate_postfix(expression):
    """
    后缀表达式求值
    
    例如："3 4 + 2 *" = (3 + 4) * 2 = 14
    
    算法：
    1. 遇到数字，入栈
    2. 遇到运算符，弹出2个数字计算，结果入栈
    """
    stack = []
    
    for token in expression.split():
        if token.isdigit():
            stack.append(int(token))
        else:
            b = stack.pop()
            a = stack.pop()
            
            if token == '+':
                stack.append(a + b)
            elif token == '-':
                stack.append(a - b)
            elif token == '*':
                stack.append(a * b)
            elif token == '/':
                stack.append(a // b)
    
    return stack[0]
```

#### 应用3：单调栈

**问题**：下一个更大元素

```python
def next_greater_element(nums):
    """
    输入：[2, 1, 2, 4, 3]
    输出：[4, 2, 4, -1, -1]
    
    解释：
    - nums[0]=2，下一个更大的是nums[3]=4
    - nums[1]=1，下一个更大的是nums[2]=2
    
    时间复杂度：O(n)
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # 单调递减栈（存储索引）
    
    for i in range(n):
        # 当前元素大于栈顶，说明找到了栈顶元素的下一个更大元素
        while stack and nums[i] > nums[stack[-1]]:
            index = stack.pop()
            result[index] = nums[i]
        
        stack.append(i)
    
    return result
```

**为什么是O(n)？**
- 每个元素最多入栈出栈各1次
- 总操作次数：2n = O(n)

### 3.4 队列底层原理

#### 循环队列实现

**问题**：普通数组实现队列，出队后前面空间浪费

```python
class CircularQueue:
    """
    循环队列：充分利用数组空间
    
    关键：
    - front: 队首指针
    - rear: 队尾指针
    - (rear + 1) % capacity = front 时队列满
    """
    def __init__(self, capacity):
        self.data = [None] * (capacity + 1)  # 多1个空间区分空/满
        self.front = 0
        self.rear = 0
        self.capacity = capacity + 1
    
    def enqueue(self, val):
        """入队：O(1)"""
        if self.is_full():
            raise Exception("Queue is full")
        self.data[self.rear] = val
        self.rear = (self.rear + 1) % self.capacity  # 循环
    
    def dequeue(self):
        """出队：O(1)"""
        if self.is_empty():
            raise Exception("Queue is empty")
        val = self.data[self.front]
        self.front = (self.front + 1) % self.capacity
        return val
    
    def is_empty(self):
        return self.front == self.rear
    
    def is_full(self):
        return (self.rear + 1) % self.capacity == self.front
```

**为什么需要多1个空间？**

```
不多1个空间：
- 空队列：front = rear = 0
- 满队列：front = rear = 0  ⚠️ 无法区分空/满！

多1个空间：
- 空：front = rear
- 满：(rear + 1) % capacity = front  ✅ 可以区分
```

### 4.3 红黑树（Java HashMap、Linux进程调度）

#### 红黑树性质

1. **节点是红色或黑色**
2. **根节点是黑色**
3. **叶子节点（NIL）是黑色**
4. **红色节点的子节点必须是黑色**（不能有连续的红节点）
5. **从根到叶子的所有路径，黑色节点数量相同**

**为什么这5条性质保证平衡？**

性质4+5推导：
```
最短路径：全是黑节点
最长路径：红黑交替

假设黑高度为h（根到叶子的黑节点数）：
- 最短路径长度：h
- 最长路径长度：2h（红黑交替）

最长路径 ≤ 2 * 最短路径
→ 近似平衡（不是严格平衡）
```

**红黑树 vs AVL树**：

| 特性 | 红黑树 | AVL树 |
|------|--------|-------|
| **平衡条件** | 最长路径≤2倍最短路径 | 左右高度差≤1 |
| **平衡程度** | 近似平衡 | 严格平衡 |
| **查找性能** | O(logn) | O(logn)，更快 |
| **插入/删除** | 更少旋转，更快 | 更多旋转 |
| **应用** | Java HashMap、Linux CFS | 数据库（要求严格平衡） |

#### 工程应用1：Java HashMap的树化

**为什么HashMap要树化？**

**为什么<6时退化回链表？**
- 避免频繁树化/退化
- 6-8是缓冲区（Hysteresis）

### 4.4 B+树（MySQL索引）

#### 为什么MySQL用B+树？

**问题**：红黑树在磁盘上效率低

```
红黑树高度：log₂(n)
- 100万数据：log₂(10⁶) ≈ 20层
- 每层1次磁盘I/O：20次I/O！
```

**B+树高度**：log_m(n)，m为节点容量

```
B+树（m=1000）：log₁₀₀₀(10⁶) ≈ 2层
- 只需2-3次I/O！
```

#### B+树原理

**结构**：
```
非叶子节点：只存key，不存data（高扇出）
叶子节点：存key+data，且通过链表连接

         [10, 20, 30]  ← 非叶子节点（索引）
        /    |    |    \
      /      |    |      \
   [1,5,8] [10,15] [20,25] [30,35,40]  ← 叶子节点（数据+链表）
      ↔       ↔       ↔        ↔
```

**B+树 vs B树**：

| 特性 | B+树 | B树 |
|------|------|-----|
| **非叶子节点** | 只存key | 存key+data |
| **叶子节点** | 存所有data，有链表 | 部分data，无链表 |
| **范围查询** | O(logn + k) - 链表遍历 | O(klogn) - 每个都要查 |
| **磁盘I/O** | 更少（非叶子节点小） | 更多 |

**为什么适合磁盘？**
1. **高扇出**：每个节点存多个key，减少高度
2. **顺序I/O**：叶子节点链表，范围查询快
3. **预读**：一次读一个磁盘页（4KB），B+树节点正好一页

---

#### 工程细节：InnoDB的B+树

**每个节点16KB**：
```
非叶子节点：
- key（8B）+ 指针（6B）= 14B
- 16KB / 14B ≈ 1170个key
- 扇出：1170

三层B+树可存储：
1170 × 1170 × 16 ≈ 2000万条记录
只需3次I/O！
```

**为什么主键建议用自增ID？**
```
自增ID：顺序插入，B+树只分裂最右节点
UUID：随机插入，频繁分裂中间节点，产生碎片
```

### 5.1 堆的底层原理

#### 为什么堆用数组存储？

- 节省指针空间
- 计算父子关系只需位运算：`i<<1`, `i>>1`

### 5.2 堆化过程（Heapify）

#### 向上调整（Bubble Up）

```python
def bubble_up(heap, i):
    """
    插入新元素后，向上调整
    时间复杂度：O(logn)
    """
    while i > 1:  # 不是根节点
        parent = i // 2
        if heap[i] < heap[parent]:  # 小根堆
            heap[i], heap[parent] = heap[parent], heap[i]
            i = parent
        else:
            break
```

#### 向下调整（Bubble Down）

```python
def bubble_down(heap, i, size):
    """
    删除根节点后，向下调整
    时间复杂度：O(logn)
    """
    while 2 * i <= size:  # 有左孩子
        # 找左右孩子中更小的
        smallest = 2 * i  # 左孩子
        if 2 * i + 1 <= size and heap[2*i+1] < heap[smallest]:
            smallest = 2 * i + 1  # 右孩子更小
        
        if heap[i] > heap[smallest]:
            heap[i], heap[smallest] = heap[smallest], heap[i]
            i = smallest
        else:
            break
```

### 5.3 手写堆

```python
class MinHeap:
    """
    小根堆实现
    """
    def __init__(self):
        self.heap = [None]  # 索引0不用
        self.size = 0
    
    def push(self, val):
        """
        插入：O(logn)
        1. 放到数组末尾
        2. 向上调整
        """
        self.heap.append(val)
        self.size += 1
        self._bubble_up(self.size)
    
    def pop(self):
        """
        删除最小值：O(logn)
        1. 用最后一个元素替换根
        2. 向下调整
        """
        if self.size == 0:
            raise Exception("Heap is empty")
        
        min_val = self.heap[1]
        self.heap[1] = self.heap[self.size]
        self.heap.pop()
        self.size -= 1
        
        if self.size > 0:
            self._bubble_down(1)
        
        return min_val
    
    def peek(self):
        """查看最小值：O(1)"""
        if self.size == 0:
            raise Exception("Heap is empty")
        return self.heap[1]
    
    def _bubble_up(self, i):
        while i > 1:
            parent = i // 2
            if self.heap[i] < self.heap[parent]:
                self.heap[i], self.heap[parent] = self.heap[parent], self.heap[i]
                i = parent
            else:
                break
    
    def _bubble_down(self, i):
        while 2 * i <= self.size:
            smallest = 2 * i
            if 2*i+1 <= self.size and self.heap[2*i+1] < self.heap[smallest]:
                smallest = 2 * i + 1
            
            if self.heap[i] > self.heap[smallest]:
                self.heap[i], self.heap[smallest] = self.heap[smallest], self.heap[i]
                i = smallest
            else:
                break
```

### 5.4 堆的工程应用

#### 应用1：Top-K问题

**问题**：从10亿个数中找最大的1000个

**方法1**：排序（❌ 时间复杂度O(nlogn)，空间O(n)）

**方法2**：小根堆（✅）

```python
def top_k_largest(nums, k):
    """
    Top-K最大值
    
    思路：维护大小为k的小根堆
    - 堆顶是第k大的数
    - 遇到更大的数，替换堆顶
    
    时间复杂度：O(nlogk)
    空间复杂度：O(k)
    """
    heap = []
    
    for num in nums:
        if len(heap) < k:
            heapq.heappush(heap, num)
        elif num > heap[0]:  # 比堆顶大
            heapq.heapreplace(heap, num)  # 替换堆顶
    
    return heap
```

**为什么用小根堆而不是大根堆？**
- 小根堆的堆顶是最小值，容易判断是否需要替换
- 大根堆无法快速判断第k大的数

**为什么不直接排序？**
- 10亿数据：排序需要O(10⁹log10⁹) ≈ 300亿次操作
- 小根堆：O(10⁹log1000) ≈ 100亿次操作，且只需1000个元素的内存

|------|--------|--------|
| **插入** | O(log₂n) | O(log₄n) = O(log₂n / 2) |
| **删除** | O(log₂n) | O(log₄n) |
| **比较次数** | 每层1次 | 每层3次（4个孩子选最小） |

**为什么Go选择四叉堆？**

1. **缓存友好**：
   ```
   二叉堆：父+2个孩子 = 3个节点
   四叉堆：父+4个孩子 = 5个节点
   
   CPU缓存行：64字节
   - timer结构：约40字节
   - 四叉堆一次可能加载父+4个孩子到缓存
   - 二叉堆需要多次缓存加载
   ```

2. **插入更快**：
   ```
   Go的定时器场景：频繁插入（创建timer），较少删除
   四叉堆插入更少层数：log₄(n) < log₂(n)
   ```

3. **实测性能更好**：
   - 二叉堆：深度大，缓存miss多
   - 四叉堆：深度小，虽然每层比较多，但总体更快

**完整实现**：

```go
// 四叉堆操作（Go runtime简化版）
func siftUp(heap []timer, i int) {
    when := heap[i].when
    tmp := heap[i]
    
    for i > 0 {
        p := (i - 1) / 4  // 父节点
        if when >= heap[p].when {
            break
        }
        heap[i] = heap[p]
        i = p
    }
    heap[i] = tmp
}

func siftDown(heap []timer, i int, n int) {
    when := heap[i].when
    tmp := heap[i]
    
    for {
        c := 4*i + 1  // 第一个孩子
        if c >= n {
            break
        }
        
        // 在4个孩子中找最小的
        w := heap[c].when
        minChild := c
        for j := c + 1; j < c+4 && j < n; j++ {
            if heap[j].when < w {
                w = heap[j].when
                minChild = j
            }
        }
        
        if w >= when {
            break
        }
        
        heap[i] = heap[minChild]
        i = minChild
    }
    heap[i] = tmp
}
```

**Go定时器优化历史**：
- **Go 1.10之前**：全局定时器堆 + 全局锁（性能瓶颈）
- **Go 1.10-1.13**：每个P一个定时器堆（减少锁竞争）
- **Go 1.14+**：网络轮询器集成，进一步优化

#### 应用3：Dijkstra最短路径算法

```python
def dijkstra(graph, start):
    """
    Dijkstra算法：单源最短路径
    
    使用堆优化：
    - 朴素版：O(n²)
    - 堆优化版：O((V+E)logV)
    """
    import heapq
    
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    
    heap = [(0, start)]  # (距离, 节点)
    visited = set()
    
    while heap:
        d, u = heapq.heappop(heap)
        
        if u in visited:
            continue
        visited.add(u)
        
        for v, weight in graph[u]:
            if dist[u] + weight < dist[v]:
                dist[v] = dist[u] + weight
                heapq.heappush(heap, (dist[v], v))
    
    return dist
```

**为什么用堆？**
- 每次需要取距离最小的节点
- 堆可以O(logn)取最小值
- 朴素版需要O(n)遍历找最小值

### 6.3 工程应用：微服务依赖检测

**问题**：检测微服务之间是否有循环依赖

```python
def has_cycle(graph):
    """
    拓扑排序检测环
    
    Kahn算法：
    1. 找入度为0的节点
    2. 删除该节点及其出边
    3. 重复，直到无入度为0的节点
    4. 如果还有剩余节点，说明有环
    """
    from collections import deque, defaultdict
    
    # 计算入度
    in_degree = defaultdict(int)
    for u in graph:
        for v in graph[u]:
            in_degree[v] += 1
    
    # 找入度为0的节点
    queue = deque([u for u in graph if in_degree[u] == 0])
    count = 0
    
    while queue:
        u = queue.popleft()
        count += 1
        
        for v in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)
    
    return count != len(graph)  # 有剩余节点 = 有环
```

## 7. 排序算法

### 7.1 快速排序原理

#### Partition过程

```python
def partition(arr, left, right):
    """
    分区：选pivot，小于pivot的放左边，大于的放右边
    
    返回：pivot的最终位置
    """
    pivot = arr[right]  # 选最后一个元素为pivot
    i = left - 1  # i指向小于pivot的区域的最后一个元素
    
    for j in range(left, right):
        if arr[j] < pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    
    # 将pivot放到正确位置
    arr[i + 1], arr[right] = arr[right], arr[i + 1]
    return i + 1

def quicksort(arr, left, right):
    """
    快速排序：O(nlogn)平均，O(n²)最坏
    """
    if left < right:
        pi = partition(arr, left, right)
        quicksort(arr, left, pi - 1)
        quicksort(arr, pi + 1, right)
```

**为什么快排快？**
- **原地排序**：不需要额外空间
- **缓存友好**：顺序访问内存
- **常数较小**：相比归并排序

**为什么最坏O(n²)？**
- 每次pivot都是最小/最大值
- 退化成选择排序

**如何避免？**
- 随机选pivot
- 三数取中（首、中、尾）

### 7.2 归并排序原理

```python
def merge_sort(arr):
    """
    归并排序：O(nlogn)，稳定排序
    """
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    """合并两个有序数组"""
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # ≤保证稳定性
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**为什么归并是稳定排序？**
- 相等元素，优先取左边 → 保持相对顺序

**为什么工程中少用归并？**
- 需要O(n)额外空间
- 快排原地排序，更节省内存

## 8. 动态规划

### 8.1 动态规划原理

#### 什么时候用DP？

**两个条件**：
1. **最优子结构**：大问题的最优解包含小问题的最优解
2. **重叠子问题**：递归过程中，子问题重复出现

#### 经典例子：斐波那契数列

**方法1**：朴素递归（❌ 指数复杂度）

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

# 时间复杂度：O(2ⁿ)
# fib(5)会计算fib(3)两次
```

**方法2**：记忆化搜索（✅ O(n)）

```python
def fib(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

# 时间复杂度：O(n)
# 每个子问题只计算一次
```

**方法3**：动态规划（✅ O(n)，空间优化）

```python
def fib(n):
    if n <= 1:
        return n
    
    # 只需保存前两个状态
    prev2, prev1 = 0, 1
    for i in range(2, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    
    return prev1

# 时间：O(n)，空间：O(1)
```

### 8.2 背包问题

#### 0-1背包

```python
def knapsack_01(weights, values, capacity):
    """
    0-1背包：每个物品只能选一次
    
    状态定义：dp[i][w] = 前i个物品，容量w的最大价值
    状态转移：
    - 不选第i个：dp[i][w] = dp[i-1][w]
    - 选第i个：dp[i][w] = dp[i-1][w-weights[i]] + values[i]
    取最大值
    """
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            # 不选第i个物品
            dp[i][w] = dp[i-1][w]
            
            # 选第i个物品（如果放得下）
            if w >= weights[i-1]:
                dp[i][w] = max(dp[i][w], 
                              dp[i-1][w - weights[i-1]] + values[i-1])
    
    return dp[n][capacity]
```

**空间优化**：

```python
def knapsack_01_optimized(weights, values, capacity):
    """
    空间优化：O(capacity)
    
    关键：从右往左更新（避免覆盖）
    """
    dp = [0] * (capacity + 1)
    
    for i in range(len(weights)):
        # ⭐ 从右往左，避免dp[w-weights[i]]被提前更新
        for w in range(capacity, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    
    return dp[capacity]
```

**为什么从右往左？**
```
从左往右：dp[w-weights[i]]可能已经更新（相当于选了两次物品）
从右往左：dp[w-weights[i]]还是上一行的值（正确）
```

### 8.3 工程应用：Redis内存淘汰算法

**LRU-K算法**：动态规划思想

```python
class LRU_K:
    """
    LRU-K：记录最近K次访问时间
    
    淘汰策略：淘汰第K次访问时间最早的
    
    为什么？
    - LRU（K=1）：一次访问就进缓存，容易被扫描污染
    - LRU-2：需要访问2次才稳定，抗污染
    """
    def __init__(self, capacity, k=2):
        self.capacity = capacity
        self.k = k
        self.cache = {}
        self.history = {}  # key → [访问时间列表]
        self.time = 0
    
    def get(self, key):
        self.time += 1
        
        if key in self.cache:
            # 记录访问时间
            if key not in self.history:
                self.history[key] = []
            self.history[key].append(self.time)
            
            # 只保留最近K次
            if len(self.history[key]) > self.k:
                self.history[key].pop(0)
            
            return self.cache[key]
        
        return -1
    
    def put(self, key, value):
        self.time += 1
        
        if len(self.cache) >= self.capacity:
            # 淘汰：找第K次访问时间最早的
            victim = min(self.history, 
                        key=lambda k: self.history[k][self.k-1] 
                                      if len(self.history[k]) >= self.k 
                                      else 0)
            del self.cache[victim]
            del self.history[victim]
        
        self.cache[key] = value
        if key not in self.history:
            self.history[key] = []
        self.history[key].append(self.time)
```

### 9.2 全排列

```python
def permute(nums):
    result = []
    
    def backtrack(path, used):
        # 终止条件：路径长度等于nums长度
        if len(path) == len(nums):
            result.append(path[:])
            return
        
        for i in range(len(nums)):
            # 跳过已使用的数字
            if used[i]:
                continue
            
            # 做选择
            path.append(nums[i])
            used[i] = True
            
            # 递归
            backtrack(path, used)
            
            # 撤销选择
            path.pop()
            used[i] = False
    
    backtrack([], [False] * len(nums))
    return result
```

### 9.3 N皇后

**问题**：在N×N棋盘上放置N个皇后，使其不相互攻击

```python
def solveNQueens(n):
    result = []
    board = [['.'] * n for _ in range(n)]
    
    def is_valid(row, col):
        # 检查列
        for i in range(row):
            if board[i][col] == 'Q':
                return False
        
        # 检查左上对角线
        i, j = row - 1, col - 1
        while i >= 0 and j >= 0:
            if board[i][j] == 'Q':
                return False
            i -= 1
            j -= 1
        
        # 检查右上对角线
        i, j = row - 1, col + 1
        while i >= 0 and j < n:
            if board[i][j] == 'Q':
                return False
            i -= 1
            j += 1
        
        return True
    
    def backtrack(row):
        # 终止条件：所有行都放置了皇后
        if row == n:
            result.append([''.join(r) for r in board])
            return
        
        # 尝试在当前行的每一列放置皇后
        for col in range(n):
            if not is_valid(row, col):
                continue
            
            # 做选择
            board[row][col] = 'Q'
            
            # 递归下一行
            backtrack(row + 1)
            
            # 撤销选择
            board[row][col] = '.'
    
    backtrack(0)
    return result
```

## 10. 贪心算法

### 10.1 贪心算法特点

- **局部最优**：每一步都做当前最优选择
- **全局最优**：希望局部最优能导致全局最优
- **不一定正确**：需要证明贪心策略的正确性

### 10.2 区间调度问题

**问题**：有N个活动，每个活动有开始时间和结束时间，求最多能参加几个活动

```python
def maxActivities(activities):
    """
    贪心策略：按结束时间排序，优先选择结束早的活动
    
    为什么贪心？
    - 结束越早，留给后续活动的时间越多
    - 局部最优 → 全局最优
    """
    # 按结束时间排序
    activities.sort(key=lambda x: x[1])
    
    count = 0
    last_end = 0
    
    for start, end in activities:
        # 如果当前活动的开始时间 >= 上一个活动的结束时间
        if start >= last_end:
            count += 1
            last_end = end
    
    return count

# 测试
```

### 10.3 跳跃游戏

**问题**：数组每个元素表示最大跳跃长度，判断能否到达最后一个位置

```python
def canJump(nums):
    """
    贪心策略：维护当前能到达的最远位置
    """
    max_reach = 0  # 当前能到达的最远位置
    
    for i in range(len(nums)):
        # 如果当前位置超过了最远可达位置，说明无法到达
        if i > max_reach:
            return False
        
        # 更新最远可达位置
        max_reach = max(max_reach, i + nums[i])
        
        # 如果已经能到达最后一个位置
        if max_reach >= len(nums) - 1:
            return True
    
    return True
```

### 11.3 只出现一次的数字

**问题**：数组中除了一个数字，其他数字都出现两次，找出只出现一次的数字

```python
def singleNumber(nums):
    """
    XOR性质：
    - a ^ a = 0
    - a ^ 0 = a
    - 交换律：a ^ b = b ^ a
    
    所以：a ^ b ^ a ^ c ^ b = (a ^ a) ^ (b ^ b) ^ c = 0 ^ 0 ^ c = c
    """
    result = 0
    for num in nums:
        result ^= num
    return result

# 测试
print(singleNumber([4, 1, 2, 1, 2]))  # 4
```

**时间复杂度**：O(n)  
**空间复杂度**：O(1)

```python
def hammingWeight(n):
    """
    技巧：n & (n-1) 清除最右边的1
    每次循环都清除一个1，直到n变为0
    """
    count = 0
    while n:
        n &= (n - 1)  # 清除最右边的1
        count += 1
    return count
```

### 12.1 前缀和

**用途**：快速计算区间和

```python
class NumArray:
    """
    问题：给定数组，多次查询区间[left, right]的和
    
    朴素方法：每次遍历O(n)
    前缀和：预处理O(n)，查询O(1)
    """
    def __init__(self, nums):
        # prefix[i] = nums[0] + nums[1] + ... + nums[i-1]
        self.prefix = [0]
        for num in nums:
            self.prefix.append(self.prefix[-1] + num)
    
    def sumRange(self, left, right):
        """
        sum(left, right) = prefix[right+1] - prefix[left]
        """
        return self.prefix[right + 1] - self.prefix[left]

# 测试
```

### 12.2 差分数组

**用途**：快速对区间进行加减操作

```python
class Difference:
    """
    问题：给定数组，多次对区间[left, right]的所有元素+val
    
    朴素方法：每次遍历O(n)
    差分数组：单次操作O(1)，最后还原O(n)
    """
    def __init__(self, nums):
        n = len(nums)
        # diff[i] = nums[i] - nums[i-1]
        self.diff = [0] * n
        self.diff[0] = nums[0]
        for i in range(1, n):
            self.diff[i] = nums[i] - nums[i - 1]
    
    def increment(self, left, right, val):
        """
        对[left, right]区间的所有元素+val
        """
        self.diff[left] += val
        if right + 1 < len(self.diff):
            self.diff[right + 1] -= val
    
    def result(self):
        """
        还原数组
        """
        n = len(self.diff)
        nums = [0] * n
        nums[0] = self.diff[0]
        for i in range(1, n):
            nums[i] = nums[i - 1] + self.diff[i]
        return nums

# 测试
diff = Difference([1, 2, 3, 4, 5])
diff.increment(1, 3, 2)  # [1,2,3,4,5] → [1,4,5,6,5]
print(diff.result())  # [1, 4, 5, 6, 5]
```

### 12.3 单调栈

**用途**：寻找下一个更大/更小元素

```python
def nextGreaterElement(nums):
    """
    问题：对于数组中的每个元素，找到右边第一个比它大的元素
    
    朴素方法：O(n^2)
    单调栈：O(n)
    """
    n = len(nums)
    result = [-1] * n
    stack = []  # 单调递减栈（存储索引）
    
    for i in range(n):
        # 当前元素 > 栈顶元素时，栈顶元素的答案就是当前元素
        while stack and nums[i] > nums[stack[-1]]:
            idx = stack.pop()
            result[idx] = nums[i]
        
        stack.append(i)
    
    return result

# 测试
print(nextGreaterElement([2, 1, 2, 4, 3]))  # [4, 2, 4, -1, -1]
```

### 12.4 滑动窗口最大值

**问题**：给定数组和窗口大小k，求每个窗口的最大值

```python
from collections import deque

def maxSlidingWindow(nums, k):
    """
    单调队列：维护窗口内的单调递减序列
    """
    result = []
    dq = deque()  # 存储索引，保持单调递减
    
    for i in range(len(nums)):
        # 移除队首的过期元素（不在窗口内）
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        
        # 移除队尾比当前元素小的元素（它们不可能是最大值）
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()
        
        dq.append(i)
        
        # 窗口形成后，记录最大值（队首元素）
        if i >= k - 1:
            result.append(nums[dq[0]])
    
    return result

# 测试
print(maxSlidingWindow([1, 3, -1, -3, 5, 3, 6, 7], 3))
# [3, 3, 5, 5, 6, 7]
```

### 12.5 二分查找专题

> 二分查找远不止"有序数组找目标值"，其核心思想是**通过判断条件缩小搜索空间**，适用于所有满足单调性/二段性的问题。

**二分查找的三种基本模板**：

```python
# 模板1：标准二分 - 查找精确值
def binary_search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2  # 防溢出
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

# 模板2：左边界二分 - 查找第一个 >= target 的位置
def lower_bound(nums, target):
    lo, hi = 0, len(nums)
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo

# 模板3：右边界二分 - 查找最后一个 <= target 的位置
def upper_bound(nums, target):
    lo, hi = 0, len(nums)
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] <= target:
            lo = mid + 1
        else:
            hi = mid
    return lo - 1
```

**高频应用场景**：

| 场景 | 核心思路 | 经典题目 |
|------|---------|---------|
| **旋转排序数组** | 先判断mid在左半段还是右半段 | LC33/LC81 |
| **峰值查找** | 比较mid与mid+1，向上坡方向移动 | LC162 |
| **二分答案（最小化最大值）** | 对答案空间二分 + 贪心验证 | LC410 分割数组 |
| **矩阵搜索** | 展开为一维或逐行二分 | LC74/LC240 |
| **求平方根/n次方** | 对结果空间 [0, x] 二分 | LC69 |

**关键易错点**：
- `lo + (hi - lo) // 2` 防止整数溢出
- `while lo <= hi` vs `while lo < hi`：取决于搜索空间是闭区间还是左闭右开
- 注意 `lo = mid + 1` 还是 `lo = mid`：错误选择导致死循环

### 12.6 字符串算法（KMP）

> KMP 算法通过预处理模式串的**最长公共前后缀（next数组/失配函数）**，实现 O(n+m) 的字符串匹配，避免暴力匹配中的回退。

**核心概念**：

- **next数组（前缀函数）**：`next[i]` = 模式串 `P[0..i]` 中最长的"既是前缀又是后缀"的长度
- **匹配过程**：文本指针永不回退，失配时只移动模式串指针到 `next[j-1]`

```python
def kmp_search(text, pattern):
    """KMP字符串匹配 - O(n+m)"""
    n, m = len(text), len(pattern)
    if m == 0:
        return 0
    
    # 构建next数组（前缀函数）
    next_arr = [0] * m
    j = 0
    for i in range(1, m):
        while j > 0 and pattern[i] != pattern[j]:
            j = next_arr[j - 1]  # 回退到上一个最长前后缀
        if pattern[i] == pattern[j]:
            j += 1
        next_arr[i] = j
    
    # 匹配
    j = 0
    results = []
    for i in range(n):
        while j > 0 and text[i] != pattern[j]:
            j = next_arr[j - 1]
        if text[i] == pattern[j]:
            j += 1
        if j == m:
            results.append(i - m + 1)
            j = next_arr[j - 1]
    return results
```

**相关算法扩展（知识点提纲）**：
- **Rabin-Karp**：滚动哈希 O(n+m)，适合多模式匹配
- **Sunday算法**：工程实践中常比KMP更快（利用"坏字符"跳跃）
- **Aho-Corasick（AC自动机）**：多模式串匹配，基于Trie+失配指针，O(n+m+k)
- **后缀数组/后缀自动机**：高级字符串结构，用于最长公共子串、重复子串等

### 12.7 并查集（Union-Find）

> 并查集是解决**动态连通性问题**的高效数据结构，支持"合并集合"和"查询是否同一集合"两种操作，均摊近 O(1)。

**核心操作**：

```python
class UnionFind:
    """路径压缩 + 按秩合并，操作均摊 O(α(n)) ≈ O(1)"""
    
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.count = n  # 连通分量个数
    
    def find(self, x):
        """查找根节点（路径压缩）"""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x, y):
        """合并两个集合（按秩合并）"""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.count -= 1
        return True
    
    def connected(self, x, y):
        return self.find(x) == self.find(y)
```

**高频应用场景**：

| 场景 | 说明 | 经典题目 |
|------|------|---------|
| **图的连通分量** | 判断无向图中连通分量数 | LC547 省份数量 |
| **冗余连接** | 加边时若两端已连通则为冗余 | LC684 |
| **最小生成树（Kruskal）** | 按边权排序，逐边加入不成环即可 | LC1584 |
| **账户合并** | 邮箱归并到同一用户 | LC721 |
| **等式方程** | 相等关系 union，不等关系检查 | LC990 |

**优化要点**：
- **路径压缩**：find 时将所有节点直接连接到根（递归或迭代）
- **按秩合并**：小树挂大树，控制树高
- 两者结合使操作均摊 O(α(n))，α 为反阿克曼函数，实际可视为常数

### 12.8 字典树（Trie）

> Trie 是一种多叉树结构，用于高效存储和检索字符串集合，特别适用于**前缀匹配**场景。

**核心实现**：

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    """字典树 - 插入/查找/前缀查询均为 O(L)，L为字符串长度"""
    
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True
    
    def search(self, word):
        """精确查找"""
        node = self._find_node(word)
        return node is not None and node.is_end
    
    def starts_with(self, prefix):
        """前缀查找"""
        return self._find_node(prefix) is not None
    
    def _find_node(self, prefix):
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node
```

**高频应用场景**：

| 场景 | 说明 | 经典题目 |
|------|------|---------|
| **自动补全** | 前缀匹配 + DFS收集所有后缀 | LC642 搜索自动补全 |
| **单词搜索II** | Trie + 回溯在矩阵中搜索 | LC212 |
| **词典中最长的单词** | 逐字符构建合法性检查 | LC720 |
| **异或最大值** | 按位建Trie，贪心选择 | LC421 |
| **回文对** | Trie存反转串，查找拼接回文 | LC336 |

**Trie 的工程扩展**：
- **压缩Trie（Radix Tree）**：将单链路径压缩为单节点，节省空间（Linux内核路由表使用）
- **双数组Trie（Double-Array Trie）**：用两个数组模拟Trie，空间极小、查找极快（中文分词 jieba/HanLP 使用）
- **AC自动机**：Trie + KMP思想的失配指针，用于多模式串同时匹配（敏感词过滤、病毒特征码检测）

### Q4: 快速排序的时间复杂度为什么是O(nlogn)？最坏情况是什么？

**答案**：
**平均O(nlogn)**：
- 每次partition将数组分成两半，需要递归logn层
- 每层需要O(n)比较，总共O(nlogn)

**最坏O(n²)**：
- 每次pivot都是最小/最大值（如有序数组、逆序数组）
- 分区退化成1和n-1，递归n层，每层O(n)

**如何优化**：
- 随机选择pivot
- 三数取中（首、中、尾的中间值）
- 小数组（<16）改用插入排序

### Q5: 为什么哈希表的查找是O(1)？什么情况下退化？

**答案**：
**O(1)原理**：
- 通过哈希函数将key映射到数组索引：`index = hash(key) % size`
- 直接通过索引访问，无需遍历

**退化情况**：
1. **哈希冲突严重**：所有key映射到同一bucket，链表长度为n，查找O(n)
2. **负载因子过高**：元素数量远超bucket数量，冲突概率增加

**工程解决**：
- Java HashMap：链表长度>8转红黑树，保证O(logn)
- 负载因子0.75时扩容
- Redis：渐进式rehash避免阻塞

### Q6: LRU缓存为什么用哈希表+双向链表？

**答案**：
**需求分析**：
- get(key)：O(1)查找 → 需要哈希表
- put时淘汰最久未使用：O(1)删除末尾 → 需要双向链表
- 访问后移动到头部：O(1)删除任意节点 → 双向链表（单向链表删除需O(n)找前驱）

**为什么不用其他结构**：
- 单向链表：删除节点需要O(n)
- 数组：删除中间元素需要O(n)移动
- 纯哈希表：无法记录访问顺序

### Q7: 堆排序为什么不如快速排序常用？

**答案**：
**理论上**：两者都是O(nlogn)

**实际差异**：
1. **缓存不友好**：堆排序访问模式跳跃（父子节点索引差距大），快排顺序访问
2. **常数因子大**：堆排序每次堆化需要多次比较（找最大孩子）
3. **不稳定**：堆排序不稳定，某些场景不适用

**堆的优势场景**：
- Top-K问题（只需要部分排序）
- 优先队列（频繁插入/删除最值）
- 外部排序（多路归并）

### Q8: 动态规划和贪心算法的区别？如何判断？

**答案**：
| 特性 | 动态规划 | 贪心算法 |
|------|---------|---------|
| **决策方式** | 考虑所有可能，取最优 | 每步只取当前最优 |
| **是否回溯** | 需要保存子问题结果 | 不需要 |
| **正确性** | 只要满足最优子结构 | 需要证明贪心选择性质 |

**典型例子**：

### Q9: 二分查找的边界条件为什么容易写错？如何记忆？

**答案**：
**三个易错点**：
1. `while lo <= hi` vs `while lo < hi`
2. `lo = mid + 1` vs `lo = mid`
3. `hi = mid - 1` vs `hi = mid`

**统一记忆法**：
```python
# 搜索区间 [lo, hi]（闭区间）
while lo <= hi:  # 区间非空时继续
    mid = lo + (hi - lo) // 2  # 防溢出
    if nums[mid] == target:
        return mid
    elif nums[mid] < target:
        lo = mid + 1  # [mid+1, hi]
    else:
        hi = mid - 1  # [lo, mid-1]
```

**不同变体的关键**：
- 找第一个>=target：`if nums[mid] >= target: hi = mid`，返回lo
- 找最后一个<=target：`if nums[mid] <= target: lo = mid`，返回hi

| **拓扑排序** | DFS | 后序遍历逆序即拓扑序 |
| **环检测** | DFS | 三色标记法 |
| **连通分量** | DFS/BFS | 都可以 |
| **迷宫最短路** | BFS | 层次扩展 |
| **路径存在性** | DFS | 空间更小 |

**空间复杂度**：
- DFS：O(h)，h为最大深度
- BFS：O(w)，w为最大宽度
- 树状结构：BFS空间可能更大

### Q13: 时间复杂度O(nlogn)的排序算法有哪些？如何选择？

**答案**：
| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 | 适用场景 |
|------|------|------|------|------|--------|---------|
| **快速排序** | O(nlogn) | O(nlogn) | O(n²) | O(logn) | ❌ | 内存排序首选 |
| **归并排序** | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | ✅ | 需要稳定排序 |
| **堆排序** | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | ❌ | 空间受限 |
| **Timsort** | O(n) | O(nlogn) | O(nlogn) | O(n) | ✅ | 部分有序数据 |

**工程选择**：
- 通用：Timsort（Python/Java默认）
- 内存敏感：堆排序
- 大量重复元素：三路快排

### Q14: 如何判断一个问题能否用动态规划解决？

**答案**：
**两个必要条件**：

1. **最优子结构**：大问题的最优解包含子问题的最优解
   - 例：最短路径问题，A→C的最短路必经过中间节点B的最短路
   
2. **重叠子问题**：不同大问题会重复用到相同子问题
   - 例：斐波那契数列，fib(5)和fib(6)都需要fib(4)

**不满足最优子结构的反例**：
- 最长简单路径（不能重复经过节点）
- 子问题的最优解可能互相冲突

**DP三步走**：
1. 定义状态：dp[i]或dp[i][j]表示什么
2. 状态转移方程：dp[i]如何由dp[i-1]等推出
3. 初始条件和边界

### Q15: 如何处理海量数据的Top-K问题？

**答案**：
**场景**：10亿个数中找最大的1000个

**方案对比**：
| 方案 | 时间 | 空间 | 适用场景 |
|------|------|------|---------|
| 全量排序 | O(nlogn) | O(n) | ❌ 内存不够 |
| 小根堆 | O(nlogk) | O(k) | ✅ 单机 |
| 分治+归并 | O(nlogk) | O(k) | ✅ 分布式 |
| 快速选择 | O(n)平均 | O(1) | ✅ 只需第K大 |

**小根堆实现**：
```python
import heapq

def topK(nums, k):
    heap = []
    for num in nums:
        if len(heap) < k:
            heapq.heappush(heap, num)
        elif num > heap[0]:
            heapq.heapreplace(heap, num)
    return heap
```

**分布式方案**：
1. 每台机器用堆找局部Top-K
2. 汇总所有机器的Top-K
3. 对K*M个数再做一次Top-K

### Q16: 单调栈解决什么问题？时间复杂度为什么是O(n)？

**答案**：
**适用问题**：
- 下一个更大/更小元素
- 柱状图中最大矩形
- 接雨水

**O(n)原理**：
- 每个元素最多入栈一次、出栈一次
- 总操作次数：2n = O(n)
- 虽然有while循环，但均摊每个元素只处理一次

**典型应用**：
```python
def nextGreater(nums):
    stack, res = [], [-1] * len(nums)
    for i, num in enumerate(nums):
        while stack and num > nums[stack[-1]]:
            res[stack.pop()] = num
        stack.append(i)
    return res
```

### Q17: 一致性哈希解决什么问题？

**答案**：
**问题**：分布式缓存节点变化时，普通哈希导致大量key迁移
```
普通哈希：hash(key) % n
节点从3变4：几乎所有key需要迁移
```

**一致性哈希**：
- 将哈希空间组织成环（0~2³²-1）
- 节点和key都映射到环上
- key存储在顺时针方向第一个节点

**优势**：
- 节点增减只影响相邻节点的key
- 迁移量从n减少到1/n

**虚拟节点**：
- 解决节点少时分布不均的问题
- 每个物理节点对应多个虚拟节点

### Q18: 红黑树和AVL树的区别？为什么实际中红黑树更常用？

**答案**：
| 特性 | AVL树 | 红黑树 |
|------|-------|--------|
| **平衡条件** | 高度差≤1（严格） | 最长路径≤2倍最短（近似） |
| **查找** | 更快（更平衡） | 略慢 |
| **插入/删除** | 更多旋转 | 更少旋转 |
| **适用场景** | 读多写少 | 读写均衡 |

**红黑树更常用的原因**：
1. 插入/删除操作更频繁时，红黑树旋转次数更少
2. 实现相对简单（AVL需要维护高度）
3. 综合性能更好（工业实践证明）

**应用**：
- 红黑树：Java HashMap、Linux CFS、C++ map
- AVL树：数据库索引（读多写少）
