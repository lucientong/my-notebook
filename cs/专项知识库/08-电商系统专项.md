## 1. 商品系统

### 1.1 SPU/SKU 模型原理

**核心概念**

| 概念 | 全称 | 含义 | 示例 |
|------|------|------|------|
| SPU | Standard Product Unit | 标准化产品单元，描述一类商品 | iPhone 15 Pro |
| SKU | Stock Keeping Unit | 库存量单位，最小可售卖单元 | iPhone 15 Pro 黑色 256G |
| 属性 | Attribute | 商品特征描述 | 颜色、内存、尺码 |
| 规格 | Specification | 属性的具体取值组合 | 黑色+256G |

**数据模型设计**

```
SPU (1) ---< (N) SKU
 |                |
 |                +--- sku_id, spu_id, price, stock, spec_json
 +--- spu_id, category_id, brand_id, title, description

Category (树形)
 +--- id, parent_id, name, level, sort_order

Attribute (属性模板)
 +--- id, category_id, name, type(枚举/输入), is_sku_attr
```

```go
    if err := json.Unmarshal([]byte(s.SpecJSON), &specMap); err != nil {

    }
    return specs, nil
```

```go
type SearchRequest struct {
    Keyword    string   `json:"keyword"`
    CategoryID int64    `json:"category_id"`
    BrandIDs   []int64  `json:"brand_ids"`
    PriceMin   int64    `json:"price_min"`
    PriceMax   int64    `json:"price_max"`
    SortBy     string   `json:"sort_by"` // "price_asc", "price_desc", "sales", "default"
    Page       int      `json:"page"`
    PageSize   int      `json:"page_size"`
}

type SearchResult struct {
    Total    int64       `json:"total"`
    Products []*Product  `json:"products"`
    Filters  []Filter    `json:"filters"` // 聚合筛选项
}

func (s *SearchService) Search(ctx context.Context, req *SearchRequest) (*SearchResult, error) {
    query := elastic.NewBoolQuery()
    
    // 关键词搜索
    if req.Keyword != "" {
        query.Must(elastic.NewMultiMatchQuery(req.Keyword, "title", "sub_title").
            Type("best_fields").
            Analyzer("ik_smart"))
    }
    // 分类过滤
    if req.CategoryID > 0 {
        query.Filter(elastic.NewTermQuery("category_id", req.CategoryID))
    }
    // 品牌过滤
    if len(req.BrandIDs) > 0 {
        query.Filter(elastic.NewTermsQuery("brand_id", intsToInterface(req.BrandIDs)...))
    }
    // 价格区间
    if req.PriceMin > 0 || req.PriceMax > 0 {
        rangeQ := elastic.NewRangeQuery("price")
        if req.PriceMin > 0 { rangeQ.Gte(req.PriceMin) }
        if req.PriceMax > 0 { rangeQ.Lte(req.PriceMax) }
        query.Filter(rangeQ)
    }
    
    search := s.esClient.Search().Index("products").Query(query).
        From((req.Page - 1) * req.PageSize).Size(req.PageSize)
    
    // 排序
    switch req.SortBy {
    case "price_asc":
        search.Sort("price", true)
    case "price_desc":
        search.Sort("price", false)
    case "sales":
        search.Sort("sales", false)
    default:
        search.Sort("_score", false)
    }
    
    result, err := search.Do(ctx)
    if err != nil {
        return nil, err
    }
    return s.parseSearchResult(result)
}
```

### 1.3 分类体系

| 方案 | 实现方式 | 查询性能 | 维护成本 | 适用场景 |
|------|---------|---------|---------|---------|
| 邻接表 | parent_id 字段 | 递归查询，较慢 | 简单 | 层级少 (<=3) |
| 路径枚举 | path="/1/3/7/" | 前缀匹配，快 | 移动节点需批量更新 | 读多写少 |
| 闭包表 | ancestor-descendant关系表 | 快 | 额外表，空间大 | 频繁查询各级子分类 |
| 嵌套集 | left/right 编号 | 范围查询，快 | 修改复杂 | 极少修改的场景 |

电商分类一般 3 级，推荐 **邻接表 + 缓存** 方案。

## 2. 购物车系统

### 2.1 存储方案对比

| 维度 | Redis 方案 | MySQL 方案 | Redis + MySQL |
|------|-----------|-----------|---------------|
| 读写性能 | 极高 | 一般 | 高 |
| 数据可靠性 | 有丢失风险 | 高 | 高 |
| 容量 | 受内存限制 | 大 | 大 |
| 登录态依赖 | 可支持游客 | 需登录 | 均可 |
| 复杂查询 | 弱 | 强 | 强 |
| 推荐场景 | 游客购物车 | 已登录持久化 | 生产推荐方案 |

**推荐方案：Redis(主) + MySQL(异步持久化)**

- 游客：仅 Redis，设 30 天过期
- 登录用户：Redis 为主，异步写 MySQL
- 登录时：合并游客购物车到用户购物车

**Redis 数据结构**

```
Key:   cart:{user_id}
Type:  HASH
Field: {sku_id}
Value: JSON{ quantity, selected, added_at, ... }

// 游客
Key:   cart:guest:{device_id}
```

### 2.2 Go 实现

```go
package cart

import (
    "context"
    "encoding/json"
    "fmt"
    "time"

    "github.com/redis/go-redis/v9"
)

type CartItem struct {
    SKUID     int64  `json:"sku_id"`
    SPUID     int64  `json:"spu_id"`
    Quantity  int    `json:"quantity"`
    Selected  bool   `json:"selected"`
    AddedAt   int64  `json:"added_at"`
}

type CartService struct {
    rdb       *redis.Client
    productSvc ProductService // 商品服务
    maxItems  int
}

func NewCartService(rdb *redis.Client, ps ProductService) *CartService {
    return &CartService{rdb: rdb, productSvc: ps, maxItems: 120}
}

func (s *CartService) cartKey(userID int64) string {
    return fmt.Sprintf("cart:%d", userID)
}

// AddItem 添加商品到购物车
func (s *CartService) AddItem(ctx context.Context, userID int64, skuID int64, quantity int) error {
    key := s.cartKey(userID)
    
    // 检查购物车商品数量上限
    count, _ := s.rdb.HLen(ctx, key).Result()
    if count >= int64(s.maxItems) {
        return fmt.Errorf("购物车已满，最多%d件商品", s.maxItems)
    }
    
    // 检查商品是否已存在
    existing, err := s.rdb.HGet(ctx, key, fmt.Sprint(skuID)).Result()
    if err == nil {
        // 已存在，累加数量
        var item CartItem
        json.Unmarshal([]byte(existing), &item)
        item.Quantity += quantity
        if item.Quantity > 99 {
            item.Quantity = 99
        }
        data, _ := json.Marshal(item)
        return s.rdb.HSet(ctx, key, fmt.Sprint(skuID), data).Err()
    }
    
    // 新增
    item := CartItem{
        SKUID:    skuID,
        SPUID:    0, // 通过商品服务获取
        Quantity: quantity,
        Selected: true,
        AddedAt:  time.Now().Unix(),
    }
    data, _ := json.Marshal(item)
    pipe := s.rdb.Pipeline()
    pipe.HSet(ctx, key, fmt.Sprint(skuID), data)
    pipe.Expire(ctx, key, 30*24*time.Hour)
    _, err = pipe.Exec(ctx)
    return err
}

// GetCart 获取购物车（附带实时价格）
func (s *CartService) GetCart(ctx context.Context, userID int64) ([]*CartItemVO, error) {
    key := s.cartKey(userID)
    all, err := s.rdb.HGetAll(ctx, key).Result()
    if err != nil {
        return nil, err
    }
    
    items := make([]*CartItemVO, 0, len(all))
    for _, v := range all {
        var item CartItem
        json.Unmarshal([]byte(v), &item)
        
        // 查询实时价格和库存
        sku, err := s.productSvc.GetSKU(ctx, item.SKUID)
        if err != nil {
            continue // SKU不存在则跳过（可标记失效）
        }
        
        vo := &CartItemVO{
            CartItem:   item,
            Title:      sku.Title,
            Image:      sku.Image,
            Price:      sku.Price,     // 实时价格
            Stock:      sku.Stock,
            Available:  sku.Status == 1 && sku.Stock > 0,
        }
        items = append(items, vo)
    }
    return items, nil
}

// MergeCart 登录时合并游客购物车
func (s *CartService) MergeCart(ctx context.Context, userID int64, guestKey string) error {
    guestItems, err := s.rdb.HGetAll(ctx, guestKey).Result()
    if err != nil || len(guestItems) == 0 {
        return nil
    }
    
    userKey := s.cartKey(userID)
    pipe := s.rdb.Pipeline()
    
    for field, value := range guestItems {
        // 游客购物车中的商品覆盖或追加到用户购物车
        exists, _ := s.rdb.HExists(ctx, userKey, field).Result()
        if !exists {
            pipe.HSet(ctx, userKey, field, value)
        }
        // 已存在则保留用户购物车中的数量（也可选择取较大值）
    }
    pipe.Del(ctx, guestKey)
    _, err = pipe.Exec(ctx)
    return err
}

type CartItemVO struct {
    CartItem
    Title     string `json:"title"`
    Image     string `json:"image"`
    Price     int64  `json:"price"`
    Stock     int64  `json:"stock"`
    Available bool   `json:"available"`
}

type ProductService interface {
    GetSKU(ctx context.Context, skuID int64) (*SKU, error)
}

type SKU struct {
    SKUID  int64  `json:"sku_id"`
    Title  string `json:"title"`
    Image  string `json:"image"`
    Price  int64  `json:"price"`
    Stock  int64  `json:"stock"`
    Status int    `json:"status"`
}
```

### 2.3 价格实时性

| 策略 | 实现方式 | 优点 | 缺点 |
|------|---------|------|------|
| 展示时查价 | 每次获取购物车实时查询SKU价格 | 价格始终最新 | 接口耗时增加 |
| 缓存价格+变更推送 | 价格变更时MQ通知刷新 | 性能好 | 有短暂不一致 |
| 下单时校验 | 提交订单时重新计算价格 | 确保交易价格正确 | 用户体验需提示 |

**最佳实践**：展示时查价(可缓存60s) + 下单时二次校验，价格变化时页面提示用户。

## 3. 订单系统

### 3.1 订单状态机 ⭐⭐⭐

```
                    [用户取消]
          +------------------------+
          |                        |
  [创建] --> 待支付 --[支付成功]--> 已支付 --[发货]--> 已发货 --[确认收货]--> 已完成
              |                      |                  |                    |
          [超时取消]            [商家取消]          [退货申请]          [售后申请]
              |                      |                  |                    |
              v                      v                  v                    v
            已取消                  已取消            退货中 --> 退款中 --> 已退款
                                                                            |
                                                                          已关闭
```

**状态枚举与转换表**

```go
package order

type OrderStatus int

const (
    OrderStatusCreated       OrderStatus = 10 // 待支付
    OrderStatusPaid          OrderStatus = 20 // 已支付
    OrderStatusShipped       OrderStatus = 30 // 已发货
    OrderStatusDelivered     OrderStatus = 40 // 已收货
    OrderStatusCompleted     OrderStatus = 50 // 已完成
    OrderStatusCancelled     OrderStatus = 60 // 已取消
    OrderStatusRefunding     OrderStatus = 70 // 退款中
    OrderStatusRefunded      OrderStatus = 80 // 已退款
    OrderStatusClosed        OrderStatus = 90 // 已关闭
)

// 状态机转换规则
var validTransitions = map[OrderStatus][]OrderStatus{
    OrderStatusCreated:   {OrderStatusPaid, OrderStatusCancelled},
    OrderStatusPaid:      {OrderStatusShipped, OrderStatusCancelled, OrderStatusRefunding},
    OrderStatusShipped:   {OrderStatusDelivered, OrderStatusRefunding},
    OrderStatusDelivered: {OrderStatusCompleted, OrderStatusRefunding},
    OrderStatusCompleted: {OrderStatusRefunding},
    OrderStatusRefunding: {OrderStatusRefunded, OrderStatusPaid}, // 退款失败可回退
    OrderStatusRefunded:  {OrderStatusClosed},
}

func CanTransition(from, to OrderStatus) bool {
    targets, ok := validTransitions[from]
    if !ok {
        return false
    }
    for _, t := range targets {
        if t == to {
            return true
        }
    }
    return false
}

// TransitionOrder 带校验的状态流转
func (s *OrderService) TransitionOrder(ctx context.Context, orderID string, newStatus OrderStatus) error {
    return s.db.Transaction(func(tx *gorm.DB) error {
        var order Order
        if err := tx.Clauses(clause.Locking{Strength: "UPDATE"}).
            Where("order_id = ?", orderID).First(&order).Error; err != nil {
            return err
        }
        if !CanTransition(order.Status, newStatus) {
            return fmt.Errorf("非法状态转换: %d -> %d", order.Status, newStatus)
        }
        // 更新状态
        order.Status = newStatus
        order.UpdatedAt = time.Now().Unix()
        return tx.Save(&order).Error
    })
}
```

### 3.2 订单号生成 ⭐⭐

| 方案 | 格式示例 | 优点 | 缺点 |
|------|---------|------|------|
| UUID | 550e8400-e29b... | 全局唯一 | 太长、无序、不可读 |
| 雪花算法 | 1382971632148... | 趋势递增、高性能 | 依赖时钟、可能暴露信息 |
| 时间+机器+序列 | 20260427100132001001 | 可读、有序 | 需分配机器ID |
| 分段式 | OD-2604-271001-3200 | 可读、含业务语义 | 实现稍复杂 |

**推荐：改良雪花算法**

```go
package orderid

import (
    "fmt"
    "sync"
    "time"
)

// OrderIDGenerator 订单号生成器
// 格式: 2位业务码 + 17位时间戳(yyyyMMddHHmmssSSS) + 2位机器ID + 4位序列号
type OrderIDGenerator struct {
    mu        sync.Mutex
    machineID int
    sequence  int
    lastMs    int64
}

func NewOrderIDGenerator(machineID int) *OrderIDGenerator {
    return &OrderIDGenerator{machineID: machineID % 100}
}

func (g *OrderIDGenerator) Generate(bizType string) string {
    g.mu.Lock()
    defer g.mu.Unlock()

    now := time.Now()
    ms := now.UnixMilli()

    if ms == g.lastMs {
        g.sequence++
        if g.sequence > 9999 {
            // 序列号溢出，等待下一毫秒
            for ms <= g.lastMs {
                ms = time.Now().UnixMilli()
            }
            g.sequence = 0
        }
    } else {
        g.sequence = 0
    }
    g.lastMs = ms

    // 格式: OD + 20260427153022123 + 01 + 0001
    timeStr := now.Format("20060102150405") + fmt.Sprintf("%03d", now.Nanosecond()/1e6)
    return fmt.Sprintf("%s%s%02d%04d", bizType, timeStr, g.machineID, g.sequence)
}
```

### 3.3 超时取消

**方案对比**

| 方案 | 原理 | 优点 | 缺点 | 推荐 |
|------|------|------|------|------|
| 定时轮询 | 定时扫描超时订单 | 简单 | 延迟大、DB压力 | 小规模 |
| 延迟队列(RabbitMQ/RocketMQ) | 消息TTL到期消费 | 精度高 | 依赖MQ | 推荐 |
| Redis过期通知 | keyspace notification | 简单 | 不可靠，可能丢失 | 不推荐 |
| 时间轮 | HashedWheelTimer | 性能高、精度好 | 内存占用，重启丢失 | 单机场景 |
| Redis ZSet | score=过期时间戳 | 可靠、高性能 | 需轮询 | 推荐 |

**Go 实现：Redis ZSet 延迟取消**

```go
package order

import (
    "context"
    "fmt"
    "time"

    "github.com/redis/go-redis/v9"
)

const (
    orderTimeoutKey = "order:timeout:zset"
    orderTimeout    = 30 * time.Minute
)

type OrderTimeoutService struct {
    rdb      *redis.Client
    orderSvc *OrderService
}

// AddTimeout 创建订单时注册超时
func (s *OrderTimeoutService) AddTimeout(ctx context.Context, orderID string) error {
    expireAt := float64(time.Now().Add(orderTimeout).Unix())
    return s.rdb.ZAdd(ctx, orderTimeoutKey, redis.Z{
        Score:  expireAt,
        Member: orderID,
    }).Err()
}

// RemoveTimeout 支付成功后移除超时
func (s *OrderTimeoutService) RemoveTimeout(ctx context.Context, orderID string) error {
    return s.rdb.ZRem(ctx, orderTimeoutKey, orderID).Err()
}

// PollTimeoutOrders 轮询超时订单（定时任务，每秒执行）
func (s *OrderTimeoutService) PollTimeoutOrders(ctx context.Context) error {
    now := float64(time.Now().Unix())
    
    // 获取所有已超时的订单
    orders, err := s.rdb.ZRangeByScore(ctx, orderTimeoutKey, &redis.ZRangeBy{
        Min: "0",
        Max: fmt.Sprintf("%f", now),
    }).Result()
    if err != nil {
        return err
    }
    
    for _, orderID := range orders {
        // 取消订单
        if err := s.orderSvc.CancelOrder(ctx, orderID, "超时自动取消"); err != nil {
            // 记录日志，下次重试
            continue
        }
        // 移除已处理的
        s.rdb.ZRem(ctx, orderTimeoutKey, orderID)
    }
    return nil
}
```

```
原始订单(用户维度)
    |
    +--- 拆单规则 ---+--- 按商家拆单（多商家平台必须拆）
                     +--- 按仓库拆单（就近发货）
                     +--- 按配送方式拆单（快递/自提/同城配送）
```

```go
// SplitOrder 拆单逻辑
func (s *OrderService) SplitOrder(ctx context.Context, parentOrder *Order, items []*OrderItem) ([]*Order, error) {
    // 按商家分组
    sellerGroups := make(map[int64][]*OrderItem)
    for _, item := range items {
        sellerGroups[item.SellerID] = append(sellerGroups[item.SellerID], item)
    }
    
    subOrders := make([]*Order, 0)
    for sellerID, sellerItems := range sellerGroups {
        // 按仓库再分组
        warehouseGroups := s.groupByWarehouse(ctx, sellerItems)
        
        for warehouseID, whItems := range warehouseGroups {
            subOrder := &Order{
                OrderID:      s.idGen.Generate("SO"),
                ParentID:     parentOrder.OrderID,
                UserID:       parentOrder.UserID,
                SellerID:     sellerID,
                WarehouseID:  warehouseID,
                Items:        whItems,
                Status:       OrderStatusCreated,
                TotalAmount:  calcSubTotal(whItems),
            }
            subOrders = append(subOrders, subOrder)
        }
    }
    
    // 分摊优惠
    s.splitDiscount(parentOrder, subOrders)
    
    return subOrders, nil
}

// splitDiscount 按比例分摊优惠金额
func (s *OrderService) splitDiscount(parent *Order, subs []*Order) {
    if parent.DiscountAmount <= 0 {
        return
    }
    var totalAmount int64
    for _, sub := range subs {
        totalAmount += sub.TotalAmount
    }
    var distributed int64
    for i, sub := range subs {
        if i == len(subs)-1 {
            // 最后一个子单分摊剩余，避免精度问题
            sub.DiscountAmount = parent.DiscountAmount - distributed
        } else {
            sub.DiscountAmount = parent.DiscountAmount * sub.TotalAmount / totalAmount
            distributed += sub.DiscountAmount
        }
        sub.PayAmount = sub.TotalAmount - sub.DiscountAmount
    }
}
```

### 4.2 超卖防护方案对比

| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| 数据库悲观锁 | SELECT FOR UPDATE | 强一致 | 性能差，死锁风险 | 低并发 |
| 数据库乐观锁 | WHERE version=X | 无锁，性能好 | 冲突重试 | 中等并发 |
| Redis原子扣减 | DECR / Lua脚本 | 极高性能 | 需要保证与DB一致 | 高并发/秒杀 |
| 分布式锁 | Redis/ZK Lock | 简单 | 锁竞争激烈时性能差 | 一般并发 |

```go
// PreDeduct 预扣库存（下单时调用）
func (s *InventoryService) PreDeduct(ctx context.Context, skuID int64, quantity int64, orderID string) error {
    return s.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        var inv Inventory
        if err := tx.Where("sku_id = ?", skuID).First(&inv).Error; err != nil {
            return fmt.Errorf("库存记录不存在: %w", err)
        }
        if inv.Available < quantity {
            return fmt.Errorf("库存不足: available=%d, need=%d", inv.Available, quantity)
        }
        
        // 乐观锁更新
        result := tx.Model(&Inventory{}).
            Where("sku_id = ? AND version = ? AND available >= ?", skuID, inv.Version, quantity).
            Updates(map[string]interface{}{
                "available": gorm.Expr("available - ?", quantity),
                "locked":    gorm.Expr("locked + ?", quantity),
                "version":   gorm.Expr("version + 1"),
            })
        if result.RowsAffected == 0 {
            return fmt.Errorf("库存扣减冲突，请重试")
        }
        
        // 记录流水
        log := &InventoryLog{
            SKUID:       skuID,
            OrderID:     orderID,
            OpType:      1, // 预扣
            Quantity:    quantity,
            BeforeAvail: inv.Available,
            AfterAvail:  inv.Available - quantity,
            CreatedAt:   time.Now().Unix(),
        }
        return tx.Create(log).Error
    })
}

// ConfirmDeduct 确认扣减（支付成功时调用）
func (s *InventoryService) ConfirmDeduct(ctx context.Context, skuID int64, quantity int64, orderID string) error {
    result := s.db.WithContext(ctx).Model(&Inventory{}).
        Where("sku_id = ? AND locked >= ?", skuID, quantity).
        Updates(map[string]interface{}{
            "locked": gorm.Expr("locked - ?", quantity),
            "sold":   gorm.Expr("sold + ?", quantity),
        })
    if result.RowsAffected == 0 {
        return fmt.Errorf("确认扣减失败")
    }
    return nil
}

// Rollback 回补库存（取消/超时时调用）
func (s *InventoryService) Rollback(ctx context.Context, skuID int64, quantity int64, orderID string) error {
    result := s.db.WithContext(ctx).Model(&Inventory{}).
        Where("sku_id = ? AND locked >= ?", skuID, quantity).
        Updates(map[string]interface{}{
            "available": gorm.Expr("available + ?", quantity),
            "locked":    gorm.Expr("locked - ?", quantity),
        })
    if result.RowsAffected == 0 {
        return fmt.Errorf("回补库存失败")
    }
    return nil
}

var deductScript = redis.NewScript(`
    local key = KEYS[1]
    local quantity = tonumber(ARGV[1])
    local stock = tonumber(redis.call('GET', key) or '0')
    if stock >= quantity then
        redis.call('DECRBY', key, quantity)
        return 1
    end
    return 0
`)

func (s *InventoryService) stockKey(skuID int64) string {

}

// RedisPreDeduct Redis原子预扣
func (s *InventoryService) RedisPreDeduct(ctx context.Context, skuID int64, quantity int64) (bool, error) {
    result, err := deductScript.Run(ctx, s.rdb, []string{s.stockKey(skuID)}, quantity).Int()
    if err != nil {
        return false, err
    }
    return result == 1, nil
}

// RedisRollback Redis库存回补
func (s *InventoryService) RedisRollback(ctx context.Context, skuID int64, quantity int64) error {
    return s.rdb.IncrBy(ctx, s.stockKey(skuID), quantity).Err()
}
```

```
选仓策略:
```

```go
type WarehouseRouter struct {

type WarehouseStock struct {
    WarehouseID int64
    City        string
    Available   int64
    Distance    float64 // 距离收货地址(km)

func (r *WarehouseRouter) Route(ctx context.Context, skuID int64, quantity int64, addressCity string) (int64, error) {
    // 查询所有有货仓库
    var stocks []WarehouseStock
    err := r.db.WithContext(ctx).Raw(`
        SELECT i.warehouse_id, w.city, i.available,
               ST_Distance_Sphere(w.location, ?) as distance
        FROM inventory i
        JOIN warehouse w ON i.warehouse_id = w.id
        WHERE i.sku_id = ? AND i.available >= ?
        ORDER BY distance ASC
    `, addressCity, skuID, quantity).Scan(&stocks).Error
    
    if err != nil || len(stocks) == 0 {
        return 0, fmt.Errorf("无可用仓库")
    }
    
    // 返回最近的有货仓库
    return stocks[0].WarehouseID, nil
}
```

### 5.3 Go 完整秒杀实现

```go
package seckill

import (
    "context"
    "encoding/json"
    "fmt"
    "sync"
    "time"

    "github.com/redis/go-redis/v9"
)

// ============ 数据结构 ============

type SeckillActivity struct {
    ActivityID int64  `json:"activity_id"`
    SKUID      int64  `json:"sku_id"`
    Price      int64  `json:"price"`       // 秒杀价(分)
    Stock      int64  `json:"stock"`       // 秒杀库存
    Limit      int    `json:"limit"`       // 每人限购
    StartTime  int64  `json:"start_time"`
    EndTime    int64  `json:"end_time"`
    Status     int    `json:"status"`      // 0-未开始 1-进行中 2-已结束
}

type SeckillRequest struct {
    UserID     int64 `json:"user_id"`
    ActivityID int64 `json:"activity_id"`
    SKUID      int64 `json:"sku_id"`
    Quantity   int   `json:"quantity"`
}

type SeckillResult struct {
    Success bool   `json:"success"`
    OrderID string `json:"order_id,omitempty"`
    Token   string `json:"token,omitempty"` // 用于查询结果
    Message string `json:"message"`
}

// ============ 秒杀服务 ============

type SeckillService struct {
    rdb       *redis.Client
    mqProducer MQProducer
    
    // 本地缓存
    localCache   sync.Map // activityID -> *SeckillActivity
    soldOutFlags sync.Map // activityID -> bool (售罄标记)
}

func NewSeckillService(rdb *redis.Client, mq MQProducer) *SeckillService {
    return &SeckillService{
        rdb:        rdb,
        mqProducer: mq,
    }
}

// ---- Lua 脚本 ----

// 原子操作：检查限购 + 扣减库存
var seckillScript = redis.NewScript(`
    -- KEYS[1]: stock key    "seckill:stock:{activityID}"
    -- KEYS[2]: buyer key    "seckill:buyers:{activityID}"
    -- ARGV[1]: user_id
    -- ARGV[2]: quantity
    -- ARGV[3]: limit (每人限购)
    
    -- 检查库存
    local stock = tonumber(redis.call('GET', KEYS[1]) or '0')
    local quantity = tonumber(ARGV[2])
    if stock < quantity then
        return -1  -- 库存不足
    end
    
    -- 检查限购
    local bought = tonumber(redis.call('HGET', KEYS[2], ARGV[1]) or '0')
    local limit = tonumber(ARGV[3])
    if bought + quantity > limit then
        return -2  -- 超出限购
    end
    
    -- 扣减库存
    redis.call('DECRBY', KEYS[1], quantity)
    -- 记录购买数量
    redis.call('HINCRBY', KEYS[2], ARGV[1], quantity)
    return 1  -- 成功
`)

// Seckill 秒杀核心流程
func (s *SeckillService) Seckill(ctx context.Context, req *SeckillRequest) (*SeckillResult, error) {
    // ====== 第1层：本地缓存校验 ======
    
    // 检查售罄标记（避免无效请求打到Redis）
    if sold, ok := s.soldOutFlags.Load(req.ActivityID); ok && sold.(bool) {
        return &SeckillResult{Success: false, Message: "商品已售罄"}, nil
    }
    
    // 检查活动状态
    activity, ok := s.getLocalActivity(req.ActivityID)
    if !ok {
        return &SeckillResult{Success: false, Message: "活动不存在"}, nil
    }
    
    now := time.Now().Unix()
    if now < activity.StartTime {
        return &SeckillResult{Success: false, Message: "活动未开始"}, nil
    }
    if now > activity.EndTime {
        return &SeckillResult{Success: false, Message: "活动已结束"}, nil
    }
    
    // ====== 第2层：Redis 原子扣减 ======
    
    stockKey := fmt.Sprintf("seckill:stock:%d", req.ActivityID)
    buyerKey := fmt.Sprintf("seckill:buyers:%d", req.ActivityID)
    
    result, err := seckillScript.Run(ctx, s.rdb, 
        []string{stockKey, buyerKey},
        req.UserID, req.Quantity, activity.Limit,
    ).Int()
    if err != nil {
        return nil, fmt.Errorf("redis执行失败: %w", err)
    }
    
    switch result {
    case -1:
        // 设置本地售罄标记
        s.soldOutFlags.Store(req.ActivityID, true)
        return &SeckillResult{Success: false, Message: "商品已售罄"}, nil
    case -2:
        return &SeckillResult{Success: false, Message: "超出限购数量"}, nil
    }
    
    // ====== 第3层：发送MQ异步创建订单 ======
    
    token := generateToken(req.UserID, req.ActivityID)
    msg := &SeckillMessage{
        UserID:     req.UserID,
        ActivityID: req.ActivityID,
        SKUID:      req.SKUID,
        Quantity:   req.Quantity,
        Price:      activity.Price,
        Token:      token,
        CreateTime: time.Now().Unix(),
    }
    
    msgBytes, _ := json.Marshal(msg)
    if err := s.mqProducer.Send(ctx, "seckill_order", string(msgBytes)); err != nil {
        // MQ发送失败，回补Redis库存
        s.rdb.IncrBy(ctx, stockKey, int64(req.Quantity))
        s.rdb.HIncrBy(ctx, buyerKey, fmt.Sprint(req.UserID), -int64(req.Quantity))
        return nil, fmt.Errorf("消息发送失败: %w", err)
    }
    
    return &SeckillResult{
        Success: true,
        Token:   token,
        Message: "秒杀成功，正在创建订单",
    }, nil
}

// getLocalActivity 获取本地缓存的活动信息

func (c *SeckillConsumer) HandleMessage(ctx context.Context, msgBody string) error {
    var msg SeckillMessage
    if err := json.Unmarshal([]byte(msgBody), &msg); err != nil {
        return err
    }
    
    resultKey := fmt.Sprintf("seckill:result:%s", msg.Token)
    
    // 1. 扣减DB库存
    if err := c.inventorySvc.PreDeduct(ctx, msg.SKUID, int64(msg.Quantity), ""); err != nil {
        // DB扣减失败，标记结果
        c.rdb.Set(ctx, resultKey, `{"success":false,"message":"库存不足"}`, 30*time.Minute)
        return nil // 不重试
    }
    
    // 2. 创建订单
    orderID, err := c.orderSvc.CreateSeckillOrder(ctx, &msg)
    if err != nil {
        // 回补DB库存
        c.inventorySvc.Rollback(ctx, msg.SKUID, int64(msg.Quantity), "")
        c.rdb.Set(ctx, resultKey, `{"success":false,"message":"订单创建失败"}`, 30*time.Minute)
        return nil
    }
    
    // 3. 设置结果
    result := SeckillResult{Success: true, OrderID: orderID, Message: "下单成功"}
    resultBytes, _ := json.Marshal(result)
    c.rdb.Set(ctx, resultKey, resultBytes, 30*time.Minute)
    
    return nil
}
```

```go
// Redis令牌桶限流（分布式）
var tokenBucketScript = redis.NewScript(`
    local key = KEYS[1]
    local capacity = tonumber(ARGV[1])    -- 桶容量
    local rate = tonumber(ARGV[2])        -- 每秒填充速率
    local now = tonumber(ARGV[3])         -- 当前时间戳(毫秒)
    local requested = tonumber(ARGV[4])   -- 请求令牌数
    
    local data = redis.call('HMGET', key, 'tokens', 'last_time')
    local tokens = tonumber(data[1]) or capacity
    local last_time = tonumber(data[2]) or now
    
    -- 计算新增的令牌
    local elapsed = (now - last_time) / 1000.0
    tokens = math.min(capacity, tokens + elapsed * rate)
    
    local allowed = 0
    if tokens >= requested then
        tokens = tokens - requested
        allowed = 1
    end
    
    redis.call('HMSET', key, 'tokens', tokens, 'last_time', now)
    redis.call('EXPIRE', key, 60)
    
    return allowed
`)

type TokenBucket struct {
    rdb      *redis.Client
    capacity int64
    rate     int64
}

func NewTokenBucket(rdb *redis.Client, capacity, rate int64) *TokenBucket {
    return &TokenBucket{rdb: rdb, capacity: capacity, rate: rate}
}

func (tb *TokenBucket) Allow(ctx context.Context, key string) (bool, error) {
    now := time.Now().UnixMilli()
    result, err := tokenBucketScript.Run(ctx, tb.rdb,
        []string{"ratelimit:" + key},
        tb.capacity, tb.rate, now, 1,
    ).Int()
    if err != nil {
        return false, err
    }
    return result == 1, nil
```

### 6.2 优惠叠加计算引擎

```
计算顺序（一般规则）:
  1. 商品级折扣（单品折扣/特价）
  2. 品类满减
  3. 店铺优惠券
  4. 平台优惠券
  5. 积分抵扣

原则:
  - 同类型优惠互斥（如两张店铺券只能用一张）
  - 不同类型可叠加（折扣+满减+优惠券）
  - 计算后不能为负（最低支付0.01元）
```

```go
package promotion

import "sort"

// PromotionType 优惠类型
type PromotionType int

const (
    PromotionDiscount   PromotionType = 1 // 折扣
    PromotionFullCut    PromotionType = 2 // 满减
    PromotionCoupon     PromotionType = 3 // 优惠券
    PromotionPlatCoupon PromotionType = 4 // 平台券
    PromotionPoints     PromotionType = 5 // 积分
)

type Promotion struct {
    ID        int64         `json:"id"`
    Type      PromotionType `json:"type"`
    Name      string        `json:"name"`
    Priority  int           `json:"priority"` // 优先级，越小越先算
    Rule      PromotionRule `json:"rule"`
}

type PromotionRule struct {
    Threshold int64   `json:"threshold"` // 门槛金额(分)
    Discount  float64 `json:"discount"`  // 折扣率(0-1)
    CutAmount int64   `json:"cut_amount"` // 减免金额(分)
    MaxCut    int64   `json:"max_cut"`    // 最大减免
}

type CalcItem struct {
    SKUID    int64 `json:"sku_id"`
    Price    int64 `json:"price"`    // 原价(分)
    Quantity int   `json:"quantity"`
}

type CalcResult struct {
    OriginalTotal  int64             `json:"original_total"`
    DiscountTotal  int64             `json:"discount_total"`
    PayTotal       int64             `json:"pay_total"`
    AppliedPromos  []*AppliedPromo   `json:"applied_promos"`
}

type AppliedPromo struct {
    PromotionID int64  `json:"promotion_id"`
    Name        string `json:"name"`
    SaveAmount  int64  `json:"save_amount"`
}

// PromotionEngine 优惠计算引擎
type PromotionEngine struct{}

func (e *PromotionEngine) Calculate(items []*CalcItem, promos []*Promotion) *CalcResult {
    // 计算原始总价
    var originalTotal int64
    for _, item := range items {
        originalTotal += item.Price * int64(item.Quantity)
    }
    
    // 按优先级排序
    sort.Slice(promos, func(i, j int) bool {
        return promos[i].Priority < promos[j].Priority
    })
    
    currentTotal := originalTotal
    appliedPromos := make([]*AppliedPromo, 0)
    
    for _, promo := range promos {
        if currentTotal <= 0 {
            break
        }
        
        var saveAmount int64
        
        switch promo.Type {
        case PromotionDiscount:
            // 折扣
            discountAmount := int64(float64(currentTotal) * (1 - promo.Rule.Discount))
            if promo.Rule.MaxCut > 0 && discountAmount > promo.Rule.MaxCut {
                discountAmount = promo.Rule.MaxCut
            }
            saveAmount = discountAmount
            
        case PromotionFullCut, PromotionCoupon, PromotionPlatCoupon:
            // 满减/优惠券
            if currentTotal >= promo.Rule.Threshold {
                saveAmount = promo.Rule.CutAmount
                if saveAmount > currentTotal {
                    saveAmount = currentTotal - 1 // 至少付1分钱
                }
            }
            
        case PromotionPoints:
            // 积分抵扣(已转换为金额)
            saveAmount = promo.Rule.CutAmount
            if saveAmount > currentTotal {
                saveAmount = currentTotal - 1
            }
        }
        
        if saveAmount > 0 {
            currentTotal -= saveAmount
            appliedPromos = append(appliedPromos, &AppliedPromo{
                PromotionID: promo.ID,
                Name:        promo.Name,
                SaveAmount:  saveAmount,
            })
        }
    }
    
    if currentTotal < 1 {
        currentTotal = 1
    }
    
    return &CalcResult{
        OriginalTotal: originalTotal,
        DiscountTotal: originalTotal - currentTotal,
        PayTotal:      currentTotal,
        AppliedPromos: appliedPromos,
    }
}
```

```go
package reconciliation

import (
    "context"
    "fmt"
    "time"

// ============ 数据结构 ============

type SystemBill struct {
    TradeNo     string `json:"trade_no"`      // 系统交易号
    ChannelNo   string `json:"channel_no"`    // 渠道交易号
    Amount      int64  `json:"amount"`        // 金额(分)
    Status      int    `json:"status"`        // 1-成功 2-退款
    PayTime     int64  `json:"pay_time"`
    Channel     string `json:"channel"`       // 支付渠道
}

type ChannelBill struct {
    ChannelNo   string `json:"channel_no"`
    MerchantNo  string `json:"merchant_no"`   // 系统交易号
    Amount      int64  `json:"amount"`
    Status      int    `json:"status"`
    PayTime     int64  `json:"pay_time"`
}

type DiffType int

const (
    DiffNone         DiffType = 0 // 无差异
    DiffAmountMiss   DiffType = 1 // 金额不一致
    DiffSystemMiss   DiffType = 2 // 系统缺失(长款)
    DiffChannelMiss  DiffType = 3 // 渠道缺失(掉单)
    DiffStatusMiss   DiffType = 4 // 状态不一致
)

type DiffRecord struct {
    ID          int64    `json:"id" gorm:"primaryKey;autoIncrement"`
    TradeNo     string   `json:"trade_no"`
    ChannelNo   string   `json:"channel_no"`
    DiffType    DiffType `json:"diff_type"`
    SysAmount   int64    `json:"sys_amount"`
    ChanAmount  int64    `json:"chan_amount"`
    HandleStatus int     `json:"handle_status"` // 0-待处理 1-已处理 2-忽略
    HandleNote  string   `json:"handle_note"`
    ReconcileDate string `json:"reconcile_date"`
    CreatedAt   int64    `json:"created_at"`
}

// ============ 对账服务 ============

type ReconciliationService struct {
    systemBillRepo  SystemBillRepository
    channelParsers  map[string]ChannelBillParser
    diffRepo        DiffRepository
}

// Reconcile 执行对账
func (s *ReconciliationService) Reconcile(ctx context.Context, channel string, date string) (*ReconcileReport, error) {
    // 1. 获取系统账单
    sysBills, err := s.systemBillRepo.GetByDateAndChannel(ctx, date, channel)
    if err != nil {
        return nil, fmt.Errorf("获取系统账单失败: %w", err)
    }
    
    // 2. 下载并解析渠道账单
    parser, ok := s.channelParsers[channel]
    if !ok {
        return nil, fmt.Errorf("不支持的渠道: %s", channel)
    }
    chanBills, err := parser.Download(ctx, date)
    if err != nil {
        return nil, fmt.Errorf("下载渠道账单失败: %w", err)
    }
    
    // 3. 构建索引
    sysMap := make(map[string]*SystemBill, len(sysBills))
    for _, b := range sysBills {
        sysMap[b.TradeNo] = b
    }
    chanMap := make(map[string]*ChannelBill, len(chanBills))
    for _, b := range chanBills {
        chanMap[b.MerchantNo] = b
    }
    
    report := &ReconcileReport{
        Channel:       channel,
        Date:          date,
        SystemCount:   len(sysBills),
        ChannelCount:  len(chanBills),
    }
    
    // 4. 以系统账单为基准对账
    for tradeNo, sysBill := range sysMap {
        chanBill, exists := chanMap[tradeNo]
        if !exists {
            // 渠道缺失（掉单）
            diff := &DiffRecord{
                TradeNo:       tradeNo,
                DiffType:      DiffChannelMiss,
                SysAmount:     sysBill.Amount,
                ReconcileDate: date,
                CreatedAt:     time.Now().Unix(),
            }
            s.diffRepo.Save(ctx, diff)
            report.DiffCount++
            continue
        }
        
        // 金额比对
        if sysBill.Amount != chanBill.Amount {
            diff := &DiffRecord{
                TradeNo:       tradeNo,
                ChannelNo:     chanBill.ChannelNo,
                DiffType:      DiffAmountMiss,
                SysAmount:     sysBill.Amount,
                ChanAmount:    chanBill.Amount,
                ReconcileDate: date,
                CreatedAt:     time.Now().Unix(),
            }
            s.diffRepo.Save(ctx, diff)
            report.DiffCount++
        } else {
            report.MatchCount++
        }
        
        delete(chanMap, tradeNo)
    }
    
    // 5. 渠道有但系统没有（长款）
    for _, chanBill := range chanMap {
        diff := &DiffRecord{
            ChannelNo:     chanBill.ChannelNo,
            DiffType:      DiffSystemMiss,
            ChanAmount:    chanBill.Amount,
            ReconcileDate: date,
            CreatedAt:     time.Now().Unix(),
        }
        s.diffRepo.Save(ctx, diff)
        report.DiffCount++
    }
    
    report.MatchRate = float64(report.MatchCount) / float64(report.SystemCount) * 100
    return report, nil
}

type ReconcileReport struct {
    Channel      string  `json:"channel"`
    Date         string  `json:"date"`
    SystemCount  int     `json:"system_count"`
    ChannelCount int     `json:"channel_count"`
    MatchCount   int     `json:"match_count"`
    DiffCount    int     `json:"diff_count"`
    MatchRate    float64 `json:"match_rate"` // 百分比
}

// 接口定义
type SystemBillRepository interface {
    GetByDateAndChannel(ctx context.Context, date, channel string) ([]*SystemBill, error)
}
type ChannelBillParser interface {
    Download(ctx context.Context, date string) ([]*ChannelBill, error)
}
type DiffRepository interface {
    Save(ctx context.Context, diff *DiffRecord) error
}
```

```go
    ExpressNo    string       `json:"express_no"`    // 退货物流单号
    RefundNo     string       `json:"refund_no"`     // 渠道退款单号
    CreatedAt    int64        `json:"created_at"`
    UpdatedAt    int64        `json:"updated_at"`

// RefundService 退款服务
type RefundService struct {
    db         *gorm.DB
    paymentSvc PaymentService
    invSvc     *InventoryService

// CreateRefund 创建退款申请
func (s *RefundService) CreateRefund(ctx context.Context, req *CreateRefundReq) (*RefundOrder, error) {
    // 1. 校验订单状态（已支付以后才能退款）
    order, err := s.getOrder(ctx, req.OrderID)
    if err != nil {
        return nil, err
    }
    if order.Status < OrderStatusPaid {
        return nil, fmt.Errorf("订单状态不允许退款")
    }
    
    // 2. 校验退款金额不超过实付
    if req.Amount > order.PayAmount {
        return nil, fmt.Errorf("退款金额超过实付金额")
    }
    
    // 3. 创建退款单
    refund := &RefundOrder{
        RefundID:     generateRefundID(),
        OrderID:      req.OrderID,
        UserID:       req.UserID,
        Type:         req.Type,
        Reason:       req.Reason,
        RefundAmount: req.Amount,
        Status:       RefundCreated,
        CreatedAt:    time.Now().Unix(),
    }
    
    if err := s.db.Create(refund).Error; err != nil {
        return nil, err
    }
    return refund, nil
}

// ProcessRefund 执行退款（到支付渠道）
func (s *RefundService) ProcessRefund(ctx context.Context, refundID string) error {
    var refund RefundOrder
    if err := s.db.Where("refund_id = ?", refundID).First(&refund).Error; err != nil {
        return err
    }
    
    // 更新状态为处理中
    s.updateStatus(ctx, refundID, RefundProcessing)
    
    // 调用支付渠道退款
    refundNo, err := s.paymentSvc.Refund(ctx, refund.OrderID, refund.RefundAmount)
    if err != nil {
        s.updateStatus(ctx, refundID, RefundFailed)
        return err
    }
    
    // 退款成功
    s.db.Model(&RefundOrder{}).Where("refund_id = ?", refundID).Updates(map[string]interface{}{
        "status":     RefundSuccess,
        "refund_no":  refundNo,
        "updated_at": time.Now().Unix(),
    })
    
    // 回补库存（退货退款场景）
    if refund.Type == 2 {
        // 获取订单商品信息，回补库存
        s.rollbackInventory(ctx, refund.OrderID)
    }
    
    return nil
}

func (s *RefundService) updateStatus(ctx context.Context, refundID string, status RefundStatus) {
    s.db.Model(&RefundOrder{}).Where("refund_id = ?", refundID).
        Updates(map[string]interface{}{"status": status, "updated_at": time.Now().Unix()})
}
```

### 11.2 Go 实现：下单链路

```go
package checkout

import (
    "context"
    "fmt"
    "time"

    "gorm.io/gorm"
)

// ============ 结算服务 ============

type CheckoutService struct {
    db           *gorm.DB
    cartSvc      CartServiceI
    productSvc   ProductServiceI
    inventorySvc InventoryServiceI
    promotionSvc PromotionServiceI
    couponSvc    CouponServiceI
    riskSvc      RiskServiceI
    orderIDGen   *OrderIDGenerator
    timeoutSvc   *OrderTimeoutService
    mqProducer   MQProducerI
}

// ============ 请求/响应 ============

type SubmitOrderRequest struct {
    UserID      int64       `json:"user_id"`
    AddressID   int64       `json:"address_id"`
    CartItemIDs []int64     `json:"cart_item_ids"` // 购物车选中的SKU
    CouponID    int64       `json:"coupon_id"`
    Remark      string      `json:"remark"`
    Channel     string      `json:"channel"` // app/h5/mini
    ClientIP    string      `json:"client_ip"`
    DeviceID    string      `json:"device_id"`
}

type SubmitOrderResponse struct {
    OrderID    string `json:"order_id"`
    PayAmount  int64  `json:"pay_amount"`
    ExpireTime int64  `json:"expire_time"` // 支付截止时间
}

// SubmitOrder 提交订单(核心链路)
func (s *CheckoutService) SubmitOrder(ctx context.Context, req *SubmitOrderRequest) (*SubmitOrderResponse, error) {
    // ====== Step 1: 参数校验 ======
    if err := s.validateRequest(req); err != nil {
        return nil, fmt.Errorf("参数校验失败: %w", err)
    }
    
    // ====== Step 2: 风控校验 ======
    riskResult, err := s.riskSvc.Check(ctx, &RiskCheckReq{
        UserID:   req.UserID,
        ClientIP: req.ClientIP,
        DeviceID: req.DeviceID,
        Action:   "submit_order",
    })
    if err != nil || !riskResult.Pass {
        return nil, fmt.Errorf("风控校验未通过: %s", riskResult.Reason)
    }
    
    // ====== Step 3: 获取商品信息并校验 ======
    cartItems, err := s.cartSvc.GetSelectedItems(ctx, req.UserID, req.CartItemIDs)
    if err != nil {
        return nil, fmt.Errorf("获取购物车失败: %w", err)
    }
    
    skuInfos, err := s.productSvc.BatchGetSKU(ctx, extractSKUIDs(cartItems))
    if err != nil {
        return nil, fmt.Errorf("获取商品信息失败: %w", err)
    }
    
    // 校验商品状态和价格
    orderItems, err := s.buildOrderItems(cartItems, skuInfos)
    if err != nil {
        return nil, err
    }
    
    // ====== Step 4: 获取收货地址 ======
    address, err := s.getAddress(ctx, req.UserID, req.AddressID)
    if err != nil {
        return nil, fmt.Errorf("收货地址无效: %w", err)
    }
    
    // ====== Step 5: 计算价格(含优惠) ======
    priceResult, err := s.calculatePrice(ctx, req.UserID, orderItems, req.CouponID)
    if err != nil {
        return nil, fmt.Errorf("价格计算失败: %w", err)
    }
    
    // ====== Step 6: 锁定优惠券 ======
    if req.CouponID > 0 {
        if err := s.couponSvc.Lock(ctx, req.UserID, req.CouponID); err != nil {
            return nil, fmt.Errorf("优惠券锁定失败: %w", err)
        }
        // 失败需要释放
        defer func() {
            if err != nil {
                s.couponSvc.Unlock(ctx, req.UserID, req.CouponID)
            }
        }()
    }
    
    // ====== Step 7: 预扣库存 ======
    deductedSKUs := make([]int64, 0)
    for _, item := range orderItems {
        if err2 := s.inventorySvc.PreDeduct(ctx, item.SKUID, int64(item.Quantity), ""); err2 != nil {
            // 回补已预扣的库存
            for _, skuID := range deductedSKUs {
                s.inventorySvc.Rollback(ctx, skuID, int64(item.Quantity), "")
            }
            err = fmt.Errorf("库存不足: %s", item.Title)
            return nil, err
        }
        deductedSKUs = append(deductedSKUs, item.SKUID)
    }
    
    // ====== Step 8: 生成订单号 + 创建订单(事务) ======
    orderID := s.orderIDGen.Generate("OD")
    
    order := &Order{
        OrderID:        orderID,
        UserID:         req.UserID,
        Status:         OrderStatusCreated,
        TotalAmount:    priceResult.OriginalTotal,
        FreightAmount:  priceResult.FreightAmount,
        DiscountAmount: priceResult.DiscountTotal,
        PayAmount:      priceResult.PayTotal,
        CouponID:       req.CouponID,
        AddressJSON:    toJSON(address),
        Remark:         req.Remark,
        Channel:        req.Channel,
        CreatedAt:      time.Now().Unix(),
        UpdatedAt:      time.Now().Unix(),
        Items:          orderItems,
    }
    
    if err = s.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        // 创建主订单
        if err := tx.Create(order).Error; err != nil {
            return err
        }
        // 创建订单项
        for _, item := range orderItems {
            item.OrderID = orderID
            if err := tx.Create(item).Error; err != nil {
                return err
            }
        }
        return nil
    }); err != nil {
        // 事务失败，回补库存
        for _, item := range orderItems {
            s.inventorySvc.Rollback(ctx, item.SKUID, int64(item.Quantity), orderID)
        }
        return nil, fmt.Errorf("订单创建失败: %w", err)
    }
    
    // ====== Step 9: 注册超时取消 ======
    s.timeoutSvc.AddTimeout(ctx, orderID)
    
    // ====== Step 10: 清理购物车 ======
    s.cartSvc.RemoveItems(ctx, req.UserID, req.CartItemIDs)
    
    // ====== Step 11: 发送订单创建事件 ======
    s.mqProducer.Send(ctx, "order_created", toJSON(map[string]interface{}{
        "order_id": orderID,
        "user_id":  req.UserID,
        "amount":   priceResult.PayTotal,
    }))
    
    return &SubmitOrderResponse{
        OrderID:    orderID,
        PayAmount:  priceResult.PayTotal,
        ExpireTime: time.Now().Add(30 * time.Minute).Unix(),
    }, nil
}

// ====== 辅助方法 ======

func (s *CheckoutService) validateRequest(req *SubmitOrderRequest) error {
    if req.UserID <= 0 {
        return fmt.Errorf("用户ID无效")
    }
    if req.AddressID <= 0 {
        return fmt.Errorf("收货地址未选择")
    }
    if len(req.CartItemIDs) == 0 {
        return fmt.Errorf("未选择商品")
    }
    if len(req.CartItemIDs) > 100 {
        return fmt.Errorf("单次最多下单100件商品")
    }
    return nil
}

func (s *CheckoutService) buildOrderItems(cartItems []*CartItemVO, skuInfos map[int64]*SKUInfo) ([]*OrderItem, error) {
    items := make([]*OrderItem, 0, len(cartItems))
    for _, ci := range cartItems {
        sku, ok := skuInfos[ci.SKUID]
        if !ok || sku.Status != 1 {
            return nil, fmt.Errorf("商品[%s]已下架", ci.Title)
        }
        if sku.Stock < int64(ci.Quantity) {
            return nil, fmt.Errorf("商品[%s]库存不足", ci.Title)
        }
        items = append(items, &OrderItem{
            SKUID:       ci.SKUID,
            SPUID:       sku.SPUID,
            SellerID:    sku.SellerID,
            Title:       sku.Title,
            Image:       sku.Image,
            Price:       sku.Price,
            Quantity:    ci.Quantity,
            TotalAmount: sku.Price * int64(ci.Quantity),
            SpecJSON:    sku.SpecJSON,
        })
    }
    return items, nil
}

func (s *CheckoutService) calculatePrice(ctx context.Context, userID int64, items []*OrderItem, couponID int64) (*PriceResult, error) {
    var originalTotal int64
    for _, item := range items {
        originalTotal += item.TotalAmount
    }
    
    var discountTotal int64
    
    // 查询适用优惠
    if couponID > 0 {
        coupon, err := s.couponSvc.Get(ctx, userID, couponID)
        if err == nil && coupon.Status == 0 {
            if originalTotal >= coupon.Threshold {
                discountTotal = coupon.Amount
            }
        }
    }
    
    // 运费计算（简化）
    var freightAmount int64 = 0
    if originalTotal < 9900 { // 不满99包邮
        freightAmount = 1000 // 10元运费
    }
    
    payTotal := originalTotal - discountTotal + freightAmount
    if payTotal < 1 {
        payTotal = 1
    }
    
    return &PriceResult{
        OriginalTotal: originalTotal,
        DiscountTotal: discountTotal,
        FreightAmount: freightAmount,
        PayTotal:      payTotal,
    }, nil
}
```

| 步骤 | 失败时处理 | 补偿方式 |
|------|-----------|---------|
| 风控校验失败 | 直接返回 | 无需补偿 |
| 商品校验失败 | 直接返回 | 无需补偿 |

### 11.4 关键设计决策

| 决策点 | 方案选择 | 原因 |
|--------|---------|------|
| 库存扣减时机 | 下单预扣 | 平衡体验和超卖，超时自动回补 |
| 优惠券使用时机 | 下单锁定，支付后核销 | 防止未支付占用优惠券 |
| 价格计算 | 结算页+下单时双重计算 | 保证价格一致性 |
| 订单超时 | 30分钟，Redis ZSet | 可靠且性能好 |
| 数据一致性 | 最终一致性 | 使用补偿机制而非分布式事务 |

## 12. 面试题自查

### 12.1 治理补充（3条）

1. **幂等设计**：下单接口必须幂等，使用请求唯一ID(Token)防止重复提交，Token在Redis中一次性消费
2. **数据一致性**：电商系统不追求强一致性，采用最终一致性 + 对账兜底，关键路径使用事务消息/本地消息表
3. **链路追踪**：全链路TraceID透传，便于故障排查，关键节点记录业务日志(订单ID、金额、库存变更)

### 12.2 初级高频速刷（5题）

**Q1: SPU和SKU的区别？**
> SPU是标准产品单元，描述一类商品（如iPhone 15 Pro）；SKU是库存量单位，对应可售卖最小单元（如iPhone 15 Pro 黑色256G）。SPU和SKU是一对多关系。

**Q2: 购物车用Redis还是MySQL？**
> 推荐 Redis+MySQL 组合。Redis作主存储保证性能，异步同步到MySQL保证持久化。游客购物车仅存Redis（30天TTL），登录后合并到用户购物车。Redis使用Hash结构，field为skuID，value为JSON(数量、选中状态等)。

**Q3: 订单号为什么不用自增ID？**
> 1) 安全性：自增ID可被猜测订单量；2) 分库分表：自增ID无法跨库唯一；3) 可读性：好的订单号含时间和业务信息。推荐方案：业务前缀+时间戳+机器ID+序列号，或改良雪花算法。

**Q4: 如何防止超卖？**
> 核心：库存扣减原子性。低并发用DB乐观锁（`WHERE stock >= quantity`），高并发用Redis Lua脚本原子扣减。秒杀场景：Redis预扣+异步落DB。关键要保证Redis和DB的最终一致。

**Q5: 订单超时取消的实现方案？**
> 推荐Redis ZSet：score存过期时间戳，member存orderID。每秒轮询score<=当前时间的元素批量取消。优势：可靠、性能好、支持分布式。也可用RocketMQ延迟消息。

### 12.3 中级进阶（5题）

**Q1: 秒杀系统如何保证不超卖且高性能？**

> 三层架构：
> 1. **本地缓存层**：售罄标记 + 活动状态校验，过滤90%+无效请求
> 2. **Redis层**：Lua脚本原子操作（检查限购+扣减库存），QPS 10w+
> 3. **MQ异步**：库存扣减成功后发MQ，消费者异步创建订单
>
> 不超卖靠Redis Lua脚本的原子性保证。MQ发送失败时回补Redis库存。

**Q2: 电商系统分库分表如何设计？**

> - **订单表**：按user_id分片，用户维度查询友好；按时间归档历史订单
> - **商品表**：按spu_id/sku_id分片
> - **库存表**：按sku_id分片
> - 跨分片查询（如商家维度查订单）：异构索引表或ES
> - 分片键选择原则：高频查询维度 + 数据均匀分布

**Q3: 如何保证下单链路的数据一致性？**

> 不使用分布式事务（性能差），而是：
> 1. **本地事务**：订单创建在一个事务内
> 2. **补偿机制**：库存预扣失败回补，订单创建失败回补库存
> 3. **最终一致性**：本地消息表 + MQ 保证下游通知
> 4. **对账兜底**：定时对账发现不一致数据，自动修复或告警

**Q4: 大促保障怎么做？**

> 1. **容量规划**：全链路压测确定瓶颈和容量上限
> 2. **限流**：多级限流（Nginx→网关→服务），令牌桶算法
> 3. **降级预案**：关闭非核心功能（推荐、评论、积分），数据返回缓存
> 4. **弹性扩缩容**：K8s HPA 按CPU/QPS自动扩容
> 5. **监控告警**：全链路监控，关键指标(RT/QPS/错误率)设阈值告警

**Q5: 拆单逻辑和优惠分摊怎么做？**

> **拆单**：按商家→按仓库→按配送方式逐级拆分
> **优惠分摊**：按子单金额占比分摊。最后一个子单分摊剩余金额（避免精度问题）。
> 示例：总优惠30元，子单A占60%分18元，子单B占40%分12元。

> - 支付状态机：待支付→支付中→支付成功/失败
> - 支付流水：每笔操作记录流水，审计追踪
> - 渠道探活：定时检测支付渠道可用性，故障自动切换
> - 掉单处理：定时查询渠道补齐结果
> ```

**Q21: 如何处理热点商品的缓存击穿？**

> 1. **互斥锁**：Redis SETNX 加锁，只允许一个请求回源
> 2. **逻辑过期**：缓存不设TTL，value中存逻辑过期时间，过期后异步更新
> 3. **缓存预热**：大促前主动加载热点数据
> 4. **本地缓存**：热点商品加载到本地内存(sync.Map)，减少Redis压力
>
> ```go
> func (s *ProductCache) GetWithLock(ctx context.Context, skuID int64) (*Product, error) {
>     key := fmt.Sprintf("product:%d", skuID)
>     // 尝试从缓存获取
>     val, err := s.rdb.Get(ctx, key).Result()
>     if err == nil {
>         return unmarshalProduct(val), nil
>     }
>     // 加锁
>     lockKey := key + ":lock"
>     locked, _ := s.rdb.SetNX(ctx, lockKey, "1", 3*time.Second).Result()
>     if !locked {
>         time.Sleep(50 * time.Millisecond)
>         return s.GetWithLock(ctx, skuID) // 重试
>     }
>     defer s.rdb.Del(ctx, lockKey)
>     // 回源
>     product, err := s.productRepo.Get(ctx, skuID)
>     if err != nil { return nil, err }
>     data, _ := json.Marshal(product)
>     s.rdb.Set(ctx, key, data, 10*time.Minute)
>     return product, nil
> }
> ```

**Q24: 为什么电商系统金额要用分（整数）而不用float？**

> 浮点数在计算机中以IEEE 754标准存储，存在精度问题：
> ```go
> // 浮点数精度问题
> fmt.Println(0.1 + 0.2) // 0.30000000000000004
> 
> // 正确做法：使用分(int64)
> priceInCents := int64(9999) // 99.99元
> discount := int64(1000)     // 10.00元优惠
> pay := priceInCents - discount // 8999 = 89.99元 精确无误
> 
> // 展示时转换
> fmt.Sprintf("%.2f", float64(pay)/100) // "89.99"
> ```
> 全链路使用整数(分)计算，只在展示层转换为元。数据库字段用BIGINT存储。

### 开放式设计题

**D1：双十一大促预计流量是平时的50倍，你作为技术负责人如何做容量保障？**

**参考思路**：
- T-30天：历史数据估算峰值QPS、各服务容量评估（压测确认单机上限）、资源采购/扩容申请
- T-14天：全链路压测（影子流量/影子库）、发现瓶颈并修复、确认降级开关可用
- T-3天：预热缓存（商品/库存预加载）、CDN预推送静态资源、确认限流/熔断阈值
- D-Day：值班作战室、监控大盘实时观察、预案手册（流量超预期→一键限流、DB压力大→开启缓存降级）
- 关键原则：宁可过度准备不可不足、所有降级开关提前演练过、非核心功能直接关闭

**D2：秒杀活动结束后发现超卖了100件商品，如何处理和复盘？**

**参考思路**：
- 紧急处理：识别超卖订单（按时间排序，超出库存的后N单）→ 通知客服准备退款话术 → 补偿方案（优惠券/下期优先购）
- 根因分析：Redis预扣库存和DB扣减之间的时间窗口并发、Lua脚本原子性是否保证、是否有重试导致重复扣减
- 复盘修复：Redis Lua保证原子检查+扣减、DB层乐观锁兜底、异步扣减消费者幂等处理
- 预防：压测模拟超卖场景、库存扣减监控（扣到负数立即告警）、熔断保护
