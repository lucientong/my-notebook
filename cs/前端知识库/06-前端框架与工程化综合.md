## 1. Vue框架深入

### 1.1 Vue 3 vs Vue 2

| 特性 | Vue 2 | Vue 3 |
|------|-------|-------|
| **响应式原理** | Object.defineProperty | Proxy |
| **组合式API** | ❌ | ✅ Composition API |
| **性能** | 基准 | 快2倍、内存减半 |
| **Tree-shaking** | ❌ | ✅ |
| **TypeScript** | 支持较弱 | 完整支持 |

### 1.2 响应式原理

#### 1.2.1 Vue 2（Object.defineProperty）
```javascript
// Vue 2响应式实现
function defineReactive(obj, key, val) {
    const dep = new Dep();  // 依赖收集器
    
    Object.defineProperty(obj, key, {
        get() {
            if (Dep.target) {
                dep.depend();  // 收集依赖
            }
            return val;
        },
        set(newVal) {
            if (newVal !== val) {
                val = newVal;
                dep.notify();  // 通知更新
            }
        }
    });
}

// 缺陷：无法检测新增属性
const obj = { a: 1 };
defineReactive(obj, 'a', 1);
obj.b = 2;  // 不会触发更新，需要Vue.set()
```

#### 1.2.2 Vue 3（Proxy）
```javascript
// Vue 3响应式实现
function reactive(target) {
    return new Proxy(target, {
        get(target, key, receiver) {
            track(target, key);  // 收集依赖
            return Reflect.get(target, key, receiver);
        },
        set(target, key, value, receiver) {
            const result = Reflect.set(target, key, value, receiver);
            trigger(target, key);  // 触发更新
            return result;
        }
    });
}

// 优势：自动检测新增属性
const obj = reactive({ a: 1 });
obj.b = 2;  // 自动响应式
```

### 1.3 Composition API vs Options API

#### 1.3.1 Options API（Vue 2风格）
```vue
<script>
export default {
    data() {
        return {
            count: 0,
            user: null
        };
    },
    methods: {
        increment() {
            this.count++;
        },
        fetchUser() {
            // ...
        }
    },
    mounted() {
        this.fetchUser();
    }
};
</script>
```

**问题**：相关逻辑分散（count相关逻辑在data、methods、mounted中）

#### 1.3.2 Composition API（Vue 3推荐）
```vue
<script setup>
import { ref, onMounted } from 'vue';

// 计数器逻辑聚合
const count = ref(0);
const increment = () => count.value++;

// 用户逻辑聚合
const user = ref(null);
const fetchUser = async () => {
    user.value = await fetch('/api/user').then(r => r.json());
};
onMounted(fetchUser);
</script>
```

**优势**：
- 逻辑复用更容易（自定义Hook）
- TypeScript支持更好
- Tree-shaking友好

### 1.4 自定义Hooks（组合式函数）

```javascript
// useCounter.js
import { ref } from 'vue';

export function useCounter(initialValue = 0) {
    const count = ref(initialValue);
    const increment = () => count.value++;
    const decrement = () => count.value--;
    const reset = () => count.value = initialValue;
    
    return { count, increment, decrement, reset };
}

// 使用
<script setup>
import { useCounter } from './useCounter';

const { count, increment, decrement } = useCounter(10);
</script>

<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">+</button>
        <button @click="decrement">-</button>
    </div>
</template>
```

**常见Hooks**：
```javascript
// useFetch.js
import { ref, watchEffect } from 'vue';

export function useFetch(url) {
    const data = ref(null);
    const error = ref(null);
    const loading = ref(false);
    
    watchEffect(async () => {
        loading.value = true;
        try {
            const response = await fetch(url.value);
            data.value = await response.json();
        } catch (e) {
            error.value = e;
        } finally {
            loading.value = false;
        }
    });
    
    return { data, error, loading };
}
```

### 1.6 Pinia（Vue 3状态管理）

```javascript
// stores/user.js
import { defineStore } from 'pinia';

export const useUserStore = defineStore('user', {
    state: () => ({
        user: null,
        token: null
    }),
    getters: {
        isLoggedIn: (state) => !!state.token,
        userName: (state) => state.user?.name || 'Guest'

            const response = await fetch('/api/login', {
                method: 'POST',
                body: JSON.stringify({ username, password })

            this.token = data.token;

        logout() {
            this.user = null;
            this.token = null;
        }
    }
});
```

## 2. Next.js与SSR

### 2.1 渲染模式对比

| 模式 | 渲染位置 | 首屏加载 | SEO | 交互性 |
|------|----------|----------|-----|--------|
| **CSR** | 客户端 | 慢（需下载JS） | ❌ | 快 |
| **SSR** | 服务端 | 快（返回HTML） | ✅ | 需hydration |
| **SSG** | 构建时 | 最快（静态HTML） | ✅ | 需hydration |
| **ISR** | 按需生成 | 快 | ✅ | 需hydration |

### 2.2 Next.js核心概念

#### 2.2.1 页面路由
```javascript
// pages/index.js → /
// pages/about.js → /about
// pages/blog/[slug].js → /blog/:slug
// pages/api/user.js → /api/user（API路由）

// pages/blog/[slug].js
export default function BlogPost({ post }) {
    return <div>{post.title}</div>;
}

export async function getStaticPaths() {
    // 返回所有可能的路径
    const posts = await fetchAllPosts();
    return {
        paths: posts.map(p => ({ params: { slug: p.slug } })),
        fallback: false  // 404 for non-existent paths
    };
}

export async function getStaticProps({ params }) {
    // 获取单篇文章数据
    const post = await fetchPost(params.slug);
    return {
        props: { post },
        revalidate: 60  // ISR：60秒后重新生成
    };
}
```

#### 2.2.2 数据获取方式

**SSG（静态生成）**：
```javascript
// 构建时生成HTML

    const data = await fetch('https://api.example.com/data');

        props: { data }
    };
}
```

**SSR（服务端渲染）**：
```javascript
// 每次请求时生成HTML
export async function getServerSideProps(context) {
    const { req, res, query } = context;
    const data = await fetch(`https://api.example.com/data?id=${query.id}`);
    return {
        props: { data }
    };
}
```

**ISR（增量静态再生成）**：
```javascript
// 首次访问生成，后续按需更新
export async function getStaticProps() {
    const data = await fetch('https://api.example.com/data');
    return {
        props: { data },
        revalidate: 10  // 10秒后过期，后台重新生成
    };
}
```

## 3. 微前端架构

### 3.1 微前端解决的问题

**单体前端的问题**：
- 代码库过大，构建慢
- 技术栈统一，难以升级
- 团队协作冲突

**微前端方案**：
```
┌──────────────────────────────────────┐
│         主应用（Shell）                │
├──────────────────────────────────────┤
│ ┌──────┐  ┌──────┐  ┌──────┐        │
│ │子应用A│  │子应用B│  │子应用C│        │
│ │(React)│  │(Vue) │  │(Angular)│     │
│ └──────┘  └──────┘  └──────┘        │
└──────────────────────────────────────┘
```

### 3.2 实现方案

#### 3.2.1 iframe（最简单）
```html
<!-- 主应用 -->
<div id="app">
    <iframe src="https://子应用A.com" />
    <iframe src="https://子应用B.com" />
</div>
```

**优点**：
- 完全隔离（CSS、JS）
- 实现简单

**缺点**：
- SEO不友好
- 性能差（多个独立页面）
- 通信复杂（postMessage）
- URL同步困难

#### 3.2.2 single-spa（主流方案）
```javascript
// 主应用
import { registerApplication, start } from 'single-spa';

// 注册子应用
registerApplication({
    name: 'app1',
    app: () => System.import('https://app1.com/main.js'),
    activeWhen: '/app1',  // 路由匹配时加载
    customProps: { authToken: 'xxx' }  // 传递给子应用的数据
});

registerApplication({
    name: 'app2',
    app: () => System.import('https://app2.com/main.js'),
    activeWhen: '/app2'
});

start();

// 子应用（React）
import React from 'react';
import ReactDOM from 'react-dom';
import singleSpaReact from 'single-spa-react';
import App from './App';

const lifecycles = singleSpaReact({
    React,
    ReactDOM,
    rootComponent: App,
    errorBoundary: (err) => <div>Error: {err.message}</div>
});

export const { bootstrap, mount, unmount } = lifecycles;
```

#### 3.2.3 qiankun（阿里开源，基于single-spa）
```javascript

    {
```

**qiankun特性**：
- 自动样式隔离（ShadowDOM/Scoped CSS）
- JS沙箱（Proxy）
- 资源预加载
- 通信机制（全局状态）

## 8. 常见面试题

### Q1: Vue 3比Vue 2快在哪里？

**答案**：
1. **Proxy替代defineProperty**：性能更好，支持数组/对象新增属性
2. **编译优化**：静态提升、事件缓存
3. **Tree-shaking**：按需引入
4. **Fragment**：减少DOM节点

### Q2: SSR vs SSG的区别？

**答案**：
- **SSR**：每次请求时生成HTML（动态数据）
- **SSG**：构建时生成HTML（静态数据）
- **ISR**：结合两者，首次SSG，后续按需更新

### Q3: 微前端如何隔离样式和JS？

**答案**：
- **样式**：Shadow DOM（完全隔离）、Scoped CSS（添加属性选择器）
- **JS**：Proxy沙箱（拦截全局变量访问）

### Q4: Vite为什么比Webpack快？

**答案**：
- **开发模式**：Vite使用ESM，按需编译；Webpack打包所有模块
- **HMR**：Vite只更新变化的模块；Webpack重新打包
