# Web3知识库导航

> 这部分知识库原本从协议、智能合约、DeFi、Layer2 直接切入，适合已经在链上开发的人，但对新手不够友好。这个导航页重点补三件事：先建立概念地图、再安排阅读顺序、最后把开发和协议学习分开。

---

## 先回答：Web3 到底在学什么

Web3 很容易一上来就陷进框架、钱包、合约部署，但如果底层概念没立住，后面会越学越乱。建议先分清 4 层：

- `账本与共识层`：区块、交易、节点、共识、密码学，这决定链为什么可信
- `执行与合约层`：EVM、Gas、状态、Solidity，这决定链上程序怎么跑
- `协议与应用层`：DEX、借贷、稳定币、预言机、MEV，这决定业务怎么玩
- `工程与扩展层`：Hardhat、Foundry、钱包接入、索引、Layer2、跨链，这决定怎么真正开发和上线

如果连第 1 层都不稳，直接写合约通常会停留在“会部署，不懂系统”。

## 难度分层

### L1 入门层：先建立区块链世界观

目标：至少能回答“为什么不需要中心数据库，也能让账本一致”。

你应该先搞懂：

- 区块、交易、节点、账本分别是什么
- 哈希、数字签名、Merkle Tree 各自解决什么问题
- UTXO 和 Account 有什么差异
- PoW、PoS、BFT 在安全性和性能上怎么取舍
- 为什么区块链天然更慢、更贵，但更难篡改

建议阅读：

- [01-区块链核心原理.md](./01-区块链核心原理.md)

### L2 进阶层：理解以太坊和智能合约

目标：理解“链上程序”不是普通后端程序，所有状态变更都带成本和公开性。

你应该先搞懂：

- EVM 为什么是栈式虚拟机
- Storage、Memory、Stack 的区别
- Gas 为什么存在，谁来付，为什么能约束资源滥用
- CALL、DELEGATECALL、STATICCALL 的语义差异
- ERC 标准为什么重要，安全问题为什么频发

建议阅读：

- [02-以太坊与智能合约.md](./02-以太坊与智能合约.md)
- [06-Web3工程实践.md](./06-Web3工程实践.md) 中的测试、部署、升级部分

### L3 深水区：协议机制、扩容与复杂应用

目标：开始理解 DeFi 不是“几个合约拼起来”，而是带有经济激励和博弈约束的系统。

你应该先搞懂：

- AMM 为什么用 `x*y=k`，滑点和无常损失怎么来的
- 预言机为什么是 DeFi 的核心依赖和安全薄弱点
- 闪电贷、MEV、清算机制为什么会引入复杂攻击面
- Rollup 为什么能扩容，它的安全性依赖什么
- 跨链桥为什么高风险，风险来自验证和托管哪里

建议阅读：

- [04-DeFi与应用协议.md](./04-DeFi与应用协议.md)
- [05-Layer2与跨链技术.md](./05-Layer2与跨链技术.md)

## 新手建议这样学，不要这样学

### 建议这样学

1. 先理解一条交易是怎么广播、验证、打包、确认的。
2. 再理解以太坊怎么执行合约，为什么会有 Gas。
3. 然后用 Hardhat 或 Foundry 跑一个最小项目。
4. 最后再去研究 DeFi、Layer2、跨链和 Rust 生态。

### 不建议这样学

1. 一上来就抄 ERC20 / NFT 合约模板，但不知道 storage slot 和权限控制在干什么。
2. 先看 Layer2 和跨链桥，但对 L1 共识和数据可用性都没概念。
3. 先学钱包接入和前端交互，却不理解链上状态更新成本。
4. 把 DeFi 当成普通互联网业务，而忽略经济攻击面和合约安全。

## 阅读时最容易踩的坑

- 把“去中心化”理解成绝对没有中心。实际上很多系统在排序器、预言机、前端入口上仍有中心化程度差异。
- 把智能合约当成普通后端代码。链上代码更难升级、调试和回滚，错误代价更高。
- 把 Gas 只理解为手续费，而忽略它还是执行资源的定价机制。
- 把跨链桥想成“转账工具”，却忽略了它本质上是在做跨系统状态信任映射。
- 只会看 TVL、收益率，却不分析协议安全、清算机制、预言机依赖和治理风险。

## 读完后的检查标准

如果下面这些问题你能讲明白，说明这个知识库已经从“难以入手”变成“有主线可学”：

- 为什么区块链牺牲性能也要换取可验证性和抗篡改性？
- UTXO 和 Account 模型分别适合什么场景？
- 以太坊为什么需要 Gas，Gas 和合约设计有什么关系？
- DeFi 协议的主要风险为什么通常不是前端 bug，而是机制设计和权限边界？
- Rollup 和跨链桥的安全性分别依赖什么？
