## 1. 认证vs授权基础

### 1.1 核心概念对比

| 维度 | 认证（Authentication） | 授权（Authorization） |
|------|----------------------|---------------------|
| **核心问题** | 你是谁？ | 你能做什么？ |
| **英文缩写** | AuthN | AuthZ |
| **发生时机** | 先认证 | 后授权 |
| **典型实现** | 用户名密码、JWT、OAuth Token | RBAC、ABAC、ACL |
| **HTTP状态码** | 401 Unauthorized | 403 Forbidden |
| **类比** | 门禁刷卡确认身份 | 确认你有权进入哪些房间 |

### 1.2 认证方式演进

```
[HTTP Basic Auth] → [Session-Cookie] → [Token(JWT)] → [OAuth2.0/OIDC]
     ↓                    ↓                  ↓                ↓
  每次传密码         服务端有状态         服务端无状态       第三方委托授权
  明文/Base64        扩展性差             易于水平扩展       标准化协议
```

### 1.3 HTTP Basic Auth（最简单）

```
# 客户端发送
Authorization: Basic base64(username:password)

# 示例
Authorization: Basic YWRtaW46cGFzc3dvcmQ=  # admin:password
```

**问题**：每次请求都传密码，Base64不是加密只是编码，必须配合HTTPS使用。

### 1.4 认证授权完整流程

```
用户 → [1.认证] → 身份确认 → [2.授权] → 权限检查 → [3.访问资源]
 │                                                       │
 ├── 用户名+密码                                          ├── 200 OK
 ├── OAuth Token                                         ├── 401 未认证
 └── 证书/生物识别                                        └── 403 无权限
```

## 2. Session-Cookie认证

### 2.1 原理与流程

```
┌──────────┐                          ┌──────────┐
│  Client  │                          │  Server  │
└────┬─────┘                          └────┬─────┘
     │  1. POST /login                     │
     │     {username, password}            │
     │ ──────────────────────────────────► │
     │                                     │ 2. 验证密码
     │                                     │    生成 SessionID
     │                                     │    存储 Session 数据
     │  3. Set-Cookie: sid=abc123          │
     │ ◄────────────────────────────────── │
     │                                     │
     │  4. GET /api/user                   │
     │     Cookie: sid=abc123              │
     │ ──────────────────────────────────► │
     │                                     │ 5. 根据 SessionID
     │                                     │    查找 Session 数据
     │  6. 返回用户信息                      │
     │ ◄────────────────────────────────── │
```

### 2.2 Go实现Session-Cookie认证

```go
package main

import (
    "crypto/rand"
    "encoding/hex"
    "encoding/json"
    "net/http"
    "sync"
    "time"
)

// Session 数据结构
type Session struct {
    UserID    int64
    Username  string
    CreatedAt time.Time
    ExpiresAt time.Time
    Data      map[string]interface{}
}

// SessionStore 内存存储（生产环境用Redis）
type SessionStore struct {
    mu       sync.RWMutex
    sessions map[string]*Session
}

func NewSessionStore() *SessionStore {
    store := &SessionStore{
        sessions: make(map[string]*Session),
    }
    // 定期清理过期Session
    go store.cleanup()
    return store
}

func (s *SessionStore) cleanup() {
    ticker := time.NewTicker(5 * time.Minute)
    for range ticker.C {
        s.mu.Lock()
        for id, sess := range s.sessions {
            if time.Now().After(sess.ExpiresAt) {
                delete(s.sessions, id)
            }
        }
        s.mu.Unlock()
    }
}

// 生成安全的SessionID
func generateSessionID() string {
    b := make([]byte, 32) // 256-bit
    rand.Read(b)
    return hex.EncodeToString(b)
}

func (s *SessionStore) Create(userID int64, username string) string {
    sid := generateSessionID()
    s.mu.Lock()
    s.sessions[sid] = &Session{
        UserID:    userID,
        Username:  username,
        CreatedAt: time.Now(),
        ExpiresAt: time.Now().Add(24 * time.Hour),
        Data:      make(map[string]interface{}),
    }
    s.mu.Unlock()
    return sid
}

func (s *SessionStore) Get(sid string) (*Session, bool) {
    s.mu.RLock()
    defer s.mu.RUnlock()
    sess, ok := s.sessions[sid]
    if !ok || time.Now().After(sess.ExpiresAt) {
        return nil, false
    }
    return sess, true
}

func (s *SessionStore) Delete(sid string) {
    s.mu.Lock()
    delete(s.sessions, sid)
    s.mu.Unlock()
}

var store = NewSessionStore()

// 登录接口
func loginHandler(w http.ResponseWriter, r *http.Request) {
    var req struct {
        Username string `json:"username"`
        Password string `json:"password"`
    }
    json.NewDecoder(r.Body).Decode(&req)

    // 验证用户（示例，实际查库+bcrypt验证）
    if req.Username != "admin" || req.Password != "password" {
        http.Error(w, "Invalid credentials", http.StatusUnauthorized)
        return
    }

    // 创建Session
    sid := store.Create(1, req.Username)

    // 设置Cookie
    http.SetCookie(w, &http.Cookie{
        Name:     "session_id",
        Value:    sid,
        Path:     "/",
        HttpOnly: true,                    // 禁止JS访问
        Secure:   true,                    // 仅HTTPS传输
        SameSite: http.SameSiteStrictMode, // 防CSRF
        MaxAge:   86400,                   // 1天
    })

    json.NewEncoder(w).Encode(map[string]string{"message": "login success"})
}

// 认证中间件
func authMiddleware(next http.HandlerFunc) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        cookie, err := r.Cookie("session_id")
        if err != nil {
            http.Error(w, "Unauthorized", http.StatusUnauthorized)
            return
        }

        sess, ok := store.Get(cookie.Value)
        if !ok {
            http.Error(w, "Session expired", http.StatusUnauthorized)
            return
        }

        // 将用户信息放入请求上下文（实际用context.WithValue）
        r.Header.Set("X-User-ID", fmt.Sprintf("%d", sess.UserID))
        r.Header.Set("X-Username", sess.Username)
        next(w, r)
    }
}

// 登出
func logoutHandler(w http.ResponseWriter, r *http.Request) {
    cookie, err := r.Cookie("session_id")
    if err == nil {
        store.Delete(cookie.Value)
    }
    // 清除Cookie
    http.SetCookie(w, &http.Cookie{
        Name:   "session_id",
        Value:  "",
        Path:   "/",
        MaxAge: -1, // 立即过期
    })
    json.NewEncoder(w).Encode(map[string]string{"message": "logout success"})
}
```

### 2.3 分布式Session方案

**问题**：多台服务器部署时，Session存在单机内存中，负载均衡后请求可能到不同服务器。

```
┌──────────┐
│  Client  │
└────┬─────┘
     │ Cookie: sid=abc123
     ▼
┌──────────┐
│   Nginx  │ (负载均衡)
└────┬─────┘
   ┌─┴─┐
   ▼   ▼
┌────┐┌────┐
│Srv1││Srv2│ ← Session在Srv1内存中，Srv2找不到！
└────┘└────┘
```

**方案对比**：

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **Sticky Session** | Nginx按IP/Cookie固定路由 | 简单 | 单点故障、负载不均 |
| **Session复制** | 服务器之间同步Session | 一致 | 网络开销大、扩展性差 |
| **Redis集中存储⭐** | Session统一存Redis | 扩展好 | 依赖Redis可用性 |
| **JWT替代** | 无状态Token | 无需存储 | Token大、无法主动失效 |

**Redis存储Session（推荐方案）**：

```go
import (
    "context"
    "encoding/json"
    "time"

    "github.com/redis/go-redis/v9"
)

type RedisSessionStore struct {
    client *redis.Client
    prefix string
    ttl    time.Duration
}

func NewRedisSessionStore(client *redis.Client) *RedisSessionStore {
    return &RedisSessionStore{
        client: client,
        prefix: "session:",
        ttl:    24 * time.Hour,
    }
}

func (s *RedisSessionStore) Create(ctx context.Context, userID int64, username string) (string, error) {
    sid := generateSessionID()
    sess := &Session{
        UserID:    userID,
        Username:  username,
        CreatedAt: time.Now(),
        ExpiresAt: time.Now().Add(s.ttl),
    }
    data, _ := json.Marshal(sess)

    err := s.client.Set(ctx, s.prefix+sid, data, s.ttl).Err()
    if err != nil {
        return "", err
    }
    return sid, nil
}

func (s *RedisSessionStore) Get(ctx context.Context, sid string) (*Session, error) {
    data, err := s.client.Get(ctx, s.prefix+sid).Bytes()
    if err == redis.Nil {
        return nil, ErrSessionNotFound
    }
    if err != nil {
        return nil, err
    }

    var sess Session
    json.Unmarshal(data, &sess)
    return &sess, nil
}

func (s *RedisSessionStore) Delete(ctx context.Context, sid string) error {
    return s.client.Del(ctx, s.prefix+sid).Err()
}

// 续期（滑动过期）
func (s *RedisSessionStore) Refresh(ctx context.Context, sid string) error {
    return s.client.Expire(ctx, s.prefix+sid, s.ttl).Err()
}
```

### 2.4 Session vs Token对比

| 维度 | Session-Cookie | Token(JWT) |
|------|---------------|------------|
| **状态** | 服务端有状态 | 服务端无状态 |
| **存储** | 服务端存Session | 客户端存Token |
| **扩展性** | 需要共享Session | 天然支持分布式 |
| **主动失效** | 删除Session即可 | 困难（需黑名单） |
| **跨域** | Cookie受同源策略限制 | Token可跨域传输 |
| **CSRF** | 需要防护 | 天然免疫 |
| **XSS** | HttpOnly保护Cookie | localStorage易受攻击 |
| **适用场景** | 传统Web应用 | SPA/移动端/微服务 |

## 3. JWT认证⭐⭐⭐

### 3.1 JWT结构详解

JWT由三部分组成，用 `.` 连接：

```
xxxxx.yyyyy.zzzzz
  │      │      │
  │      │      └── Signature（签名）
  │      └── Payload（负载）
  └── Header（头部）
```

**Header（头部）**：
```json
{
  "alg": "HS256",   // 签名算法：HS256, RS256, ES256
  "typ": "JWT"      // Token类型
}
// Base64Url编码 → eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
```

**Payload（负载）**：
```json
{
  // 注册声明（Registered Claims）
  "iss": "auth-service",        // 签发者
  "sub": "1234567890",          // 主题（通常是用户ID）
  "aud": "my-app",              // 受众
  "exp": 1716239022,            // 过期时间（Unix时间戳）
  "nbf": 1716235422,            // 生效时间
  "iat": 1716235422,            // 签发时间
  "jti": "unique-token-id",     // JWT唯一标识

  // 自定义声明（Private Claims）
  "username": "admin",
  "role": "admin",
  "permissions": ["read", "write"]
}
// Base64Url编码 → eyJzdWIiOiIxMjM0NTY3ODkwIi4uLn0
```

**Signature（签名）**：
```
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)
```

**签名算法对比**：

| 算法 | 类型 | 密钥 | 适用场景 |
|------|------|------|----------|
| **HS256** | 对称加密 | 共享密钥 | 单体应用、内部服务 |
| **RS256** | 非对称加密 | 公私钥对 | 微服务（私钥签发，公钥验证） |
| **ES256** | 非对称加密（椭圆曲线） | 公私钥对 | 性能更好的非对称方案 |

### 3.2 JWT签名与验证原理

```
签发（Sign）：
┌──────────────────────────────────────────────────────┐
│ header = base64Url({"alg":"HS256","typ":"JWT"})      │
│ payload = base64Url({"sub":"123","exp":1716239022})   │
│ signature = HMAC-SHA256(header + "." + payload, key)  │
│ jwt = header + "." + payload + "." + base64Url(sig)   │
└──────────────────────────────────────────────────────┘

验证（Verify）：
┌──────────────────────────────────────────────────────┐
│ 1. 分割Token为 header.payload.signature              │
│ 2. 用相同密钥重新计算签名                               │
│    expected = HMAC-SHA256(header + "." + payload, key)│
│ 3. 比较签名是否一致                                    │
│ 4. 检查 exp 是否过期                                  │
│ 5. 检查 iss, aud 等声明                               │
└──────────────────────────────────────────────────────┘

注意：Payload是Base64编码，不是加密！任何人都能解码看到内容。
签名只保证完整性（未被篡改），不保证机密性。
敏感信息不要放在Payload中！
```

### 3.3 Go实现JWT中间件

```go
package auth

import (
    "context"
    "errors"
    "net/http"
    "strings"
    "time"

    "github.com/golang-jwt/jwt/v5"
)

var (
    ErrTokenExpired = errors.New("token expired")
    ErrTokenInvalid = errors.New("token invalid")
    jwtSecret       = []byte("your-256-bit-secret") // 生产环境从配置中读取
)

// 自定义Claims
type CustomClaims struct {
    UserID      int64    `json:"user_id"`
    Username    string   `json:"username"`
    Role        string   `json:"role"`
    Permissions []string `json:"permissions,omitempty"`
    jwt.RegisteredClaims
}

// 生成JWT Token
func GenerateToken(userID int64, username, role string) (string, error) {
    claims := CustomClaims{
        UserID:   userID,
        Username: username,
        Role:     role,
        RegisteredClaims: jwt.RegisteredClaims{
            Issuer:    "my-auth-service",
            Subject:   fmt.Sprintf("%d", userID),
            ExpiresAt: jwt.NewNumericDate(time.Now().Add(2 * time.Hour)),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
            NotBefore: jwt.NewNumericDate(time.Now()),
            ID:        uuid.New().String(), // jti
        },
    }

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(jwtSecret)
}

// 解析验证JWT Token
func ParseToken(tokenString string) (*CustomClaims, error) {
    token, err := jwt.ParseWithClaims(tokenString, &CustomClaims{},
        func(token *jwt.Token) (interface{}, error) {
            // 验证签名算法（防止alg替换攻击）
            if _, ok := token.Method.(*jwt.SigningMethodHMAC); !ok {
                return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
            }
            return jwtSecret, nil
        },
    )

    if err != nil {
        if errors.Is(err, jwt.ErrTokenExpired) {
            return nil, ErrTokenExpired
        }
        return nil, ErrTokenInvalid
    }

    if claims, ok := token.Claims.(*CustomClaims); ok && token.Valid {
        return claims, nil
    }
    return nil, ErrTokenInvalid
}

// 上下文Key
type contextKey string

const userClaimsKey contextKey = "user_claims"

// JWT认证中间件
func JWTAuthMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // 从Header获取Token
        authHeader := r.Header.Get("Authorization")
        if authHeader == "" {
            http.Error(w, `{"error":"missing authorization header"}`, http.StatusUnauthorized)
            return
        }

        // 提取Bearer Token
        parts := strings.SplitN(authHeader, " ", 2)
        if len(parts) != 2 || parts[0] != "Bearer" {
            http.Error(w, `{"error":"invalid authorization format"}`, http.StatusUnauthorized)
            return
        }

        // 解析验证Token
        claims, err := ParseToken(parts[1])
        if err != nil {
            if errors.Is(err, ErrTokenExpired) {
                http.Error(w, `{"error":"token expired"}`, http.StatusUnauthorized)
            } else {
                http.Error(w, `{"error":"invalid token"}`, http.StatusUnauthorized)
            }
            return
        }

        // 将Claims放入上下文
        ctx := context.WithValue(r.Context(), userClaimsKey, claims)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

// 从上下文获取用户信息
func GetUserFromContext(ctx context.Context) (*CustomClaims, bool) {
    claims, ok := ctx.Value(userClaimsKey).(*CustomClaims)
    return claims, ok
}

// 使用示例
func profileHandler(w http.ResponseWriter, r *http.Request) {
    claims, ok := GetUserFromContext(r.Context())
    if !ok {
        http.Error(w, "unauthorized", http.StatusUnauthorized)
        return
    }
    json.NewEncoder(w).Encode(map[string]interface{}{
        "user_id":  claims.UserID,
        "username": claims.Username,
        "role":     claims.Role,
    })
}
```

### 3.4 Access Token + Refresh Token 双Token方案⭐⭐⭐

**为什么需要双Token？**

```
单Token问题：
- Token有效期短（如15分钟）→ 用户频繁重新登录，体验差
- Token有效期长（如7天）→ 泄露后风险窗口大

双Token解决：
- Access Token：短有效期（15min-2h），用于访问资源
- Refresh Token：长有效期（7-30天），仅用于刷新Access Token
```

**流程图**：

```
┌──────────┐                          ┌──────────┐
│  Client  │                          │  Server  │
└────┬─────┘                          └────┬─────┘
     │  1. POST /login                     │
     │     {username, password}            │
     │ ──────────────────────────────────► │
     │                                     │ 2. 验证密码
     │  3. {access_token, refresh_token}   │    生成双Token
     │ ◄────────────────────────────────── │    Refresh Token存库
     │                                     │
     │  4. GET /api/data                   │
     │     Authorization: Bearer {AT}      │
     │ ──────────────────────────────────► │
     │                                     │ 5. 验证AT
     │  6. 返回数据                         │
     │ ◄────────────────────────────────── │
     │                                     │
     │  ....AT过期....                     │
     │                                     │
     │  7. GET /api/data → 401             │
     │ ◄────────────────────────────────── │
     │                                     │
     │  8. POST /auth/refresh              │
     │     {refresh_token}                 │
     │ ──────────────────────────────────► │
     │                                     │ 9. 验证RT
     │  10. {new_access_token,             │    生成新AT
     │       new_refresh_token}            │    Refresh Token轮换
     │ ◄────────────────────────────────── │
```

**Go实现**：

```go
package auth

import (
    "context"
    "crypto/rand"
    "encoding/hex"
    "time"
)

type TokenPair struct {
    AccessToken  string `json:"access_token"`
    RefreshToken string `json:"refresh_token"`
    ExpiresIn    int64  `json:"expires_in"` // Access Token剩余秒数
}

// RefreshToken存储（数据库/Redis）
type RefreshTokenRecord struct {
    Token     string
    UserID    int64
    ExpiresAt time.Time
    Used      bool // 是否已使用（用于Refresh Token轮换检测）
    Family    string // Token家族标识（用于检测盗用）
}

type AuthService struct {
    db          *sql.DB
    redisClient *redis.Client
}

// 登录：生成双Token
func (s *AuthService) Login(ctx context.Context, username, password string) (*TokenPair, error) {
    // 1. 验证用户密码
    user, err := s.verifyCredentials(ctx, username, password)
    if err != nil {
        return nil, ErrInvalidCredentials
    }

    // 2. 生成Token对
    return s.generateTokenPair(ctx, user.ID, user.Username, user.Role)
}

func (s *AuthService) generateTokenPair(ctx context.Context, userID int64, username, role string) (*TokenPair, error) {
    // 生成Access Token（JWT，短期）
    accessToken, err := GenerateToken(userID, username, role)
    if err != nil {
        return nil, err
    }

    // 生成Refresh Token（随机字符串，长期，存库）
    refreshTokenBytes := make([]byte, 32)
    rand.Read(refreshTokenBytes)
    refreshToken := hex.EncodeToString(refreshTokenBytes)

    family := hex.EncodeToString(refreshTokenBytes[:16])

    // 存储Refresh Token到Redis
    record := &RefreshTokenRecord{
        Token:     refreshToken,
        UserID:    userID,
        ExpiresAt: time.Now().Add(30 * 24 * time.Hour), // 30天
        Used:      false,
        Family:    family,
    }
    data, _ := json.Marshal(record)
    s.redisClient.Set(ctx, "refresh:"+refreshToken, data, 30*24*time.Hour)

    return &TokenPair{
        AccessToken:  accessToken,
        RefreshToken: refreshToken,
        ExpiresIn:    7200, // 2小时
    }, nil
}

// 刷新Token
func (s *AuthService) RefreshToken(ctx context.Context, refreshToken string) (*TokenPair, error) {
    // 1. 查找Refresh Token
    data, err := s.redisClient.Get(ctx, "refresh:"+refreshToken).Bytes()
    if err == redis.Nil {
        return nil, ErrRefreshTokenInvalid
    }

    var record RefreshTokenRecord
    json.Unmarshal(data, &record)

    // 2. 检查是否过期
    if time.Now().After(record.ExpiresAt) {
        s.redisClient.Del(ctx, "refresh:"+refreshToken)
        return nil, ErrRefreshTokenExpired
    }

    // 3. 检查是否已使用（Refresh Token Rotation检测盗用）
    if record.Used {
        // 可能被盗用！撤销该家族所有Token
        s.revokeTokenFamily(ctx, record.Family)
        return nil, ErrRefreshTokenReused
    }

    // 4. 标记当前Refresh Token为已使用
    record.Used = true
    data, _ = json.Marshal(record)
    s.redisClient.Set(ctx, "refresh:"+refreshToken, data, 30*24*time.Hour)

    // 5. 查询用户信息
    user, err := s.getUserByID(ctx, record.UserID)
    if err != nil {
        return nil, err
    }

    // 6. 生成新的Token对
    return s.generateTokenPair(ctx, user.ID, user.Username, user.Role)
}

// 撤销Token家族（检测到盗用时）
func (s *AuthService) revokeTokenFamily(ctx context.Context, family string) {
    // 扫描并删除该家族所有Refresh Token
    iter := s.redisClient.Scan(ctx, 0, "refresh:*", 100).Iterator()
    for iter.Next(ctx) {
        data, _ := s.redisClient.Get(ctx, iter.Val()).Bytes()
        var record RefreshTokenRecord
        json.Unmarshal(data, &record)
        if record.Family == family {
            s.redisClient.Del(ctx, iter.Val())
        }
    }
}

// 登出：撤销Refresh Token
func (s *AuthService) Logout(ctx context.Context, refreshToken string) error {
    // 删除Refresh Token
    s.redisClient.Del(ctx, "refresh:"+refreshToken)

    // 可选：将Access Token加入黑名单（如果需要立即失效）
    // s.redisClient.Set(ctx, "blacklist:"+accessToken, "1", 2*time.Hour)
    return nil
}
```

### 3.5 JWT安全注意事项⭐⭐

| 风险 | 说明 | 防护措施 |
|------|------|----------|
| **alg:none攻击** | 攻击者将算法改为none绕过签名验证 | 验证时明确指定允许的算法 |
| **密钥泄露** | HMAC密钥泄露可伪造任意Token | 使用强密钥（≥256bit）、定期轮换 |
| **Token泄露** | XSS窃取localStorage中的Token | 存HttpOnly Cookie或内存中 |
| **信息泄露** | Payload未加密，Base64可直接解码 | 敏感信息不放Payload，必要时用JWE |
| **Token过大** | 自定义声明太多导致Token很大 | 精简Payload，只放必要信息 |
| **无法撤销** | JWT无状态，签发后无法主动失效 | 短有效期+黑名单+Refresh Token |

```go
// 防止alg替换攻击：验证签名算法
token, err := jwt.ParseWithClaims(tokenString, &CustomClaims{},
    func(token *jwt.Token) (interface{}, error) {
        // ✅ 明确验证算法类型

            return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
        }
        return jwtSecret, nil
    },
)

// ❌ 错误做法：不验证算法
```

## 4. 密码安全

### 4.1 密码存储演进

```
明文存储     →  MD5/SHA哈希   →  加盐哈希      →  慢哈希（bcrypt/argon2）
  ❌              ❌               ⚠️                 ✅
脱库即泄露     彩虹表可破解     盐值泄露仍可暴力   GPU暴力破解也很慢
```

### 4.2 哈希算法对比

| 算法 | 类型 | 计算速度 | 内存消耗 | 安全性 | 推荐度 |
|------|------|----------|----------|--------|--------|
| **MD5/SHA256** | 快速哈希 | 极快（GPU加速） | 低 | ❌ 不安全 | 禁止用于密码 |
| **bcrypt** | 慢哈希（CPU密集） | 可调（cost参数） | 低 | ✅ | ⭐⭐⭐推荐 |
| **scrypt** | 慢哈希（内存密集） | 可调 | 高（可调） | ✅✅ | ⭐⭐ |
| **argon2id** | 慢哈希（CPU+内存） | 可调 | 高（可调） | ✅✅✅ 最强 | ⭐⭐⭐新项目首选 |

### 4.3 bcrypt原理

```
bcrypt结构：$2a$12$LJ3m4y9VpZL8YUsim2vid.kMGBnNdNbq0OE7NiO.FTQP3vp8mExa
             │   │  └─── 22字符的盐（Base64） + 31字符的哈希
             │   └─── cost因子（2^12 = 4096轮迭代）
             └─── 算法标识（2a = bcrypt）

特点：
1. 内置盐值（不需要单独存盐）
2. cost可调（每增加1，计算时间翻倍）
3. 输出固定60字符
4. 相同密码每次哈希结果不同（盐不同）
```

```go
package auth

import (
    "crypto/rand"
    "crypto/subtle"
    "encoding/base64"
    "fmt"
    "strings"

    "golang.org/x/crypto/bcrypt"
    "golang.org/x/crypto/argon2"

// ==================== bcrypt（推荐，最广泛使用） ====================

const bcryptCost = 12 // 推荐12-14，越大越安全但越慢

// 密码哈希
func HashPassword(password string) (string, error) {
    bytes, err := bcrypt.GenerateFromPassword([]byte(password), bcryptCost)
    if err != nil {
        return "", err
    }
    return string(bytes), nil
}

// 密码验证
func CheckPassword(password, hash string) bool {
    err := bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))
    return err == nil
}

// ==================== argon2id（新项目首选，更安全） ====================

type Argon2Params struct {
    Memory      uint32 // 内存消耗（KB）
    Iterations  uint32 // 迭代次数
    Parallelism uint8  // 并行线程数
    SaltLength  uint32 // 盐长度
    KeyLength   uint32 // 哈希输出长度
}

var DefaultArgon2Params = &Argon2Params{
    Memory:      64 * 1024, // 64MB
    Iterations:  3,
    Parallelism: 2,
    SaltLength:  16,
    KeyLength:   32,
}

// argon2id哈希
func HashPasswordArgon2(password string, p *Argon2Params) (string, error) {
    salt := make([]byte, p.SaltLength)
    if _, err := rand.Read(salt); err != nil {
        return "", err
    }

    hash := argon2.IDKey([]byte(password), salt, p.Iterations, p.Memory, p.Parallelism, p.KeyLength)

    // 编码为标准格式
    b64Salt := base64.RawStdEncoding.EncodeToString(salt)
    b64Hash := base64.RawStdEncoding.EncodeToString(hash)

    encoded := fmt.Sprintf("$argon2id$v=%d$m=%d,t=%d,p=%d$%s$%s",
        argon2.Version, p.Memory, p.Iterations, p.Parallelism, b64Salt, b64Hash)

    return encoded, nil
}

// argon2id验证
func CheckPasswordArgon2(password, encodedHash string) (bool, error) {
    p, salt, hash, err := decodeArgon2Hash(encodedHash)
    if err != nil {
        return false, err
    }

    otherHash := argon2.IDKey([]byte(password), salt, p.Iterations, p.Memory, p.Parallelism, p.KeyLength)

    // 使用恒等时间比较（防止时序攻击）
    if subtle.ConstantTimeCompare(hash, otherHash) == 1 {
        return true, nil
    }
    return false, nil
}

func decodeArgon2Hash(encodedHash string) (*Argon2Params, []byte, []byte, error) {
    parts := strings.Split(encodedHash, "$")
    if len(parts) != 6 {
        return nil, nil, nil, fmt.Errorf("invalid hash format")
    }

    var version int
    fmt.Sscanf(parts[2], "v=%d", &version)

    p := &Argon2Params{}
    fmt.Sscanf(parts[3], "m=%d,t=%d,p=%d", &p.Memory, &p.Iterations, &p.Parallelism)

    salt, _ := base64.RawStdEncoding.DecodeString(parts[4])
    hash, _ := base64.RawStdEncoding.DecodeString(parts[5])
    p.SaltLength = uint32(len(salt))
    p.KeyLength = uint32(len(hash))

    return p, salt, hash, nil
}

// ==================== 用户注册/登录示例 ====================

type UserService struct {
    db *sql.DB
}

func (s *UserService) Register(ctx context.Context, username, password string) error {
    // 1. 密码强度检查
    if err := validatePasswordStrength(password); err != nil {
        return err
    }

    // 2. 哈希密码
    hashedPassword, err := HashPassword(password)
    if err != nil {
        return err
    }

    // 3. 存入数据库（存的是哈希值，不是明文！）
    _, err = s.db.ExecContext(ctx,
        "INSERT INTO users (username, password_hash) VALUES ($1, $2)",
        username, hashedPassword,
    )
    return err
}

func (s *UserService) Login(ctx context.Context, username, password string) (*User, error) {
    var user User
    err := s.db.QueryRowContext(ctx,
        "SELECT id, username, password_hash, role FROM users WHERE username = $1",
        username,
    ).Scan(&user.ID, &user.Username, &user.PasswordHash, &user.Role)

    if err == sql.ErrNoRows {
        // 即使用户不存在也执行一次哈希比较（防止时序攻击泄露用户名是否存在）
        bcrypt.CompareHashAndPassword([]byte("$2a$12$dummy"), []byte(password))
        return nil, ErrInvalidCredentials
    }

    if !CheckPassword(password, user.PasswordHash) {
        return nil, ErrInvalidCredentials
    }

    return &user, nil
}

// 密码强度验证
func validatePasswordStrength(password string) error {
    if len(password) < 8 {
        return fmt.Errorf("password must be at least 8 characters")
    }
    var hasUpper, hasLower, hasDigit, hasSpecial bool
    for _, c := range password {
        switch {
        case 'A' <= c && c <= 'Z':
            hasUpper = true
        case 'a' <= c && c <= 'z':
            hasLower = true
        case '0' <= c && c <= '9':
            hasDigit = true
        default:
            hasSpecial = true
        }
    }
    if !hasUpper || !hasLower || !hasDigit || !hasSpecial {
        return fmt.Errorf("password must contain upper, lower, digit, and special characters")
    }
    return nil
}
```

### 5.1 核心角色

```
┌───────────────────────────────────────────────────────┐
│  Resource Owner（资源所有者）= 用户                      │
```

| **授权码+PKCE** | SPA/移动端 | ⭐⭐⭐ | ✅ |
| **隐式** | SPA（已废弃） | ⭐ | ❌ |
| **密码** | 自家可信应用 | ⭐⭐ | ✅ |
| **客户端凭证** | 服务间通信 | ⭐⭐ | ❌ |

### 5.3 PKCE（Proof Key for Code Exchange）

**问题**：SPA和移动端无法安全存储client_secret，授权码可能被拦截。

```
PKCE流程（在授权码模式基础上增加）：

1. 客户端生成随机 code_verifier（43-128字符）
2. 计算 code_challenge = BASE64URL(SHA256(code_verifier))
3. 授权请求带上 code_challenge 和 code_challenge_method=S256
4. 用code换token时带上 code_verifier
5. 服务器验证 SHA256(code_verifier) == code_challenge

即使攻击者截获了authorization code，没有code_verifier也无法换取token
```

```go
package oauth

import (
    "crypto/sha256"
    "encoding/base64"
    "crypto/rand"
)

// 生成PKCE参数
func GeneratePKCE() (verifier, challenge string) {
    // 生成code_verifier（43-128个字符的随机字符串）
    b := make([]byte, 32)
    rand.Read(b)
    verifier = base64.RawURLEncoding.EncodeToString(b)

    // 计算code_challenge = BASE64URL(SHA256(code_verifier))
    h := sha256.Sum256([]byte(verifier))
    challenge = base64.RawURLEncoding.EncodeToString(h[:])

    return verifier, challenge
}

// 验证PKCE
func VerifyPKCE(verifier, challenge string) bool {
    h := sha256.Sum256([]byte(verifier))
    expected := base64.RawURLEncoding.EncodeToString(h[:])
    return expected == challenge
}
```

```go
        return

        return

        return
```

### 5.5 OAuth2.0授权服务器实现（简化版）

```go
    "time"
)

type OAuthClient struct {

    RedirectURIs []string
    Scopes       []string
}

// 授权码
type AuthorizationCode struct {

    ClientID    string
    UserID      int64
    RedirectURI string
    Scopes      []string
    ExpiresAt   time.Time
    // PKCE
    CodeChallenge       string
    CodeChallengeMethod string
}

type OAuth2Server struct {
    mu      sync.RWMutex
    clients map[string]*OAuthClient      // client_id -> client
    codes   map[string]*AuthorizationCode // code -> auth code
    tokens  map[string]*TokenInfo         // access_token -> info
}

type TokenInfo struct {
    AccessToken  string
    TokenType    string
    ExpiresIn    int64
    RefreshToken string
    Scope        string
    UserID       int64
    ClientID     string
}

// 授权端点
func (s *OAuth2Server) AuthorizeHandler(w http.ResponseWriter, r *http.Request) {
    clientID := r.URL.Query().Get("client_id")
    redirectURI := r.URL.Query().Get("redirect_uri")
    responseType := r.URL.Query().Get("response_type")
    scope := r.URL.Query().Get("scope")
    state := r.URL.Query().Get("state")
    codeChallenge := r.URL.Query().Get("code_challenge")
    codeChallengeMethod := r.URL.Query().Get("code_challenge_method")

    // 验证客户端
    client, ok := s.clients[clientID]
    if !ok {
        http.Error(w, "Invalid client", http.StatusBadRequest)
        return
    }

    // 验证redirect_uri
    if !s.isValidRedirectURI(client, redirectURI) {
        http.Error(w, "Invalid redirect_uri", http.StatusBadRequest)
        return
    }

    if responseType != "code" {
        http.Error(w, "Unsupported response_type", http.StatusBadRequest)
        return
    }

    // 假设用户已登录并同意授权（实际需要展示授权页面）
    userID := int64(1) // 从session获取

    // 生成授权码
    codeBytes := make([]byte, 16)
    rand.Read(codeBytes)
    code := hex.EncodeToString(codeBytes)

    s.mu.Lock()
    s.codes[code] = &AuthorizationCode{
        Code:                code,
        ClientID:            clientID,
        UserID:              userID,
        RedirectURI:         redirectURI,
        Scopes:              strings.Split(scope, " "),
        ExpiresAt:           time.Now().Add(10 * time.Minute), // 授权码10分钟过期
        CodeChallenge:       codeChallenge,
        CodeChallengeMethod: codeChallengeMethod,
    }
    s.mu.Unlock()

    // 重定向回客户端
    redirectURL := fmt.Sprintf("%s?code=%s&state=%s", redirectURI, code, state)
    http.Redirect(w, r, redirectURL, http.StatusFound)
}

// Token端点
func (s *OAuth2Server) TokenHandler(w http.ResponseWriter, r *http.Request) {
    grantType := r.FormValue("grant_type")

    switch grantType {
    case "authorization_code":
        s.handleAuthCodeGrant(w, r)
    case "client_credentials":
        s.handleClientCredentialsGrant(w, r)
    case "refresh_token":
        s.handleRefreshTokenGrant(w, r)
    default:
        http.Error(w, "Unsupported grant_type", http.StatusBadRequest)
    }
}

func (s *OAuth2Server) handleAuthCodeGrant(w http.ResponseWriter, r *http.Request) {
    code := r.FormValue("code")
    clientID := r.FormValue("client_id")
    clientSecret := r.FormValue("client_secret")
    redirectURI := r.FormValue("redirect_uri")
    codeVerifier := r.FormValue("code_verifier")

    // 验证客户端凭证
    client, ok := s.clients[clientID]
    if !ok || client.ClientSecret != clientSecret {
        http.Error(w, "Invalid client credentials", http.StatusUnauthorized)
        return
    }

    // 验证授权码
    s.mu.Lock()
    authCode, ok := s.codes[code]
    if !ok || time.Now().After(authCode.ExpiresAt) || authCode.ClientID != clientID {
        s.mu.Unlock()
        http.Error(w, "Invalid authorization code", http.StatusBadRequest)
        return
    }
    // 授权码只能使用一次
    delete(s.codes, code)
    s.mu.Unlock()

    // 验证redirect_uri
    if authCode.RedirectURI != redirectURI {
        http.Error(w, "redirect_uri mismatch", http.StatusBadRequest)
        return
    }

    // 验证PKCE
    if authCode.CodeChallenge != "" {
        if !VerifyPKCE(codeVerifier, authCode.CodeChallenge) {
            http.Error(w, "PKCE verification failed", http.StatusBadRequest)
            return
        }
    }

    // 生成Token
    tokenInfo := s.generateTokenInfo(authCode.UserID, clientID, authCode.Scopes)

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(tokenInfo)
}
```

## 6. SSO单点登录⭐⭐

### 6.1 SSO核心原理

```
SSO（Single Sign-On）：用户在一个系统登录后，访问其他系统无需再次登录。

     用户一次登录
         │
    ┌────┴────┐
    ▼         ▼
 ┌──────┐ ┌──────┐ ┌──────┐
 │ App1 │ │ App2 │ │ App3 │  ← 所有应用共享登录状态
 └──┬───┘ └──┬───┘ └──┬───┘
    │        │        │
    └────┬───┘────┬───┘
         ▼        ▼
    ┌──────────────┐
    │  SSO Server  │  ← 统一认证中心
    └──────────────┘
```

### 6.2 CAS/SAML/OIDC对比

| 维度 | CAS | SAML2.0 | OIDC |
|------|-----|---------|------|
| **全称** | Central Authentication Service | Security Assertion Markup Language | OpenID Connect |
| **协议基础** | 自有协议 | XML/SOAP | OAuth2.0 + JSON |
| **Token格式** | Service Ticket（一次性） | XML Assertion | JWT(id_token) |
| **适用场景** | 企业内部Web SSO | 企业级跨域SSO | 互联网/移动端SSO |
| **复杂度** | 中等 | 高（XML复杂） | 低（基于OAuth2.0） |
| **数据格式** | URL参数 | XML | JSON |
| **移动端友好** | 一般 | 差 | ⭐好 |
| **典型使用者** | 高校/政务系统 | 企业IT（Okta,AD FS） | Google/微信/GitHub |
| **推荐度** | ⭐⭐遗留系统 | ⭐⭐企业级 | ⭐⭐⭐新项目首选 |

### 6.4 OIDC（OpenID Connect）⭐⭐⭐

```
OIDC = OAuth2.0 + 身份认证层

OAuth2.0只做授权（给你一个access_token可以访问资源）
OIDC在此基础上增加了认证（告诉你"用户是谁"）

核心增加：
- id_token：JWT格式，包含用户身份信息
- UserInfo Endpoint：获取用户详细信息的标准API
- 标准化的scope：openid, profile, email, phone, address
```

```go
func oidcCallbackHandler(w http.ResponseWriter, r *http.Request) {
    code := r.URL.Query().Get("code")

    // 用code换Token
    token, err := oidcConfig.Exchange(r.Context(), code)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // 从Token中获取id_token
    rawIDToken, ok := token.Extra("id_token").(string)
    if !ok {
        http.Error(w, "Missing id_token", http.StatusInternalServerError)
        return
    }

    // 验证id_token（需要用OIDC Provider的公钥验证签名）
    // 使用 github.com/coreos/go-oidc/v3 库
    provider, _ := oidc.NewProvider(r.Context(), "https://accounts.google.com")
    verifier := provider.Verifier(&oidc.Config{ClientID: oidcConfig.ClientID})

    idToken, err := verifier.Verify(r.Context(), rawIDToken)
    if err != nil {
        http.Error(w, "Invalid id_token", http.StatusUnauthorized)
        return
    }

    // 解析用户信息
    var claims struct {
        Sub           string `json:"sub"`
        Name          string `json:"name"`
        Email         string `json:"email"`
        EmailVerified bool   `json:"email_verified"`
        Picture       string `json:"picture"`
    }
    idToken.Claims(&claims)

    json.NewEncoder(w).Encode(claims)
}
```

### 6.5 SSO跨域问题与解决方案

```
问题：不同子域/域名之间Cookie不能共享

方案1：共享父域Cookie
- 适用：*.example.com 下的子域
- 设置 Cookie Domain=.example.com
- 限制：不同根域名无法使用

方案2：Token传递
- SSO Server签发JWT
- 通过URL参数或postMessage传递给各应用
- 各应用独立验证JWT

方案3：认证中心重定向
- 每个应用重定向到SSO Server
- SSO Server检查是否已登录（SSO的Cookie/Session）
- 已登录则直接重定向回并带上Token/Ticket
```

## 7. 多因素认证MFA

### 7.1 TOTP原理（基于时间的一次性密码）

```
TOTP = Time-based One-Time Password（RFC 6238）

原理：
1. 服务器和客户端共享一个密钥（Secret Key）
2. 将当前时间除以时间步长（默认30秒）得到计数器 T
3. 用密钥和T进行HMAC-SHA1运算
4. 从结果中截取6位数字作为验证码

T = floor(current_unix_time / 30)
TOTP = Truncate(HMAC-SHA1(secret, T)) mod 10^6

特点：
- 每30秒变化一次
- 离线可用（不需要网络）
- 服务端验证通常接受 ±1个时间步长（容错90秒窗口）
```

### 7.2 Go实现TOTP

```go
package mfa

import (
    "crypto/hmac"
    "crypto/rand"
    "crypto/sha1"
    "encoding/base32"
    "encoding/binary"
    "fmt"
    "math"
    "net/url"
    "time"
)

const (
    totpDigits   = 6
    totpPeriod   = 30 // 秒
    totpSkew     = 1  // 允许前后1个时间步长
    secretLength = 20 // 160-bit密钥
)

// 生成TOTP密钥
func GenerateSecret() (string, error) {
    secret := make([]byte, secretLength)
    _, err := rand.Read(secret)
    if err != nil {
        return "", err
    }
    return base32.StdEncoding.WithPadding(base32.NoPadding).EncodeToString(secret), nil
}

// 生成TOTP验证码
func GenerateTOTP(secret string, t time.Time) (string, error) {
    key, err := base32.StdEncoding.WithPadding(base32.NoPadding).DecodeString(secret)
    if err != nil {
        return "", err
    }

    // 计算时间计数器
    counter := uint64(t.Unix()) / totpPeriod

    // HMAC-SHA1
    buf := make([]byte, 8)
    binary.BigEndian.PutUint64(buf, counter)
    mac := hmac.New(sha1.New, key)
    mac.Write(buf)
    hash := mac.Sum(nil)

    // 动态截取
    offset := hash[len(hash)-1] & 0x0f
    code := int64(binary.BigEndian.Uint32(hash[offset:offset+4]) & 0x7fffffff)
    code = code % int64(math.Pow10(totpDigits))

    return fmt.Sprintf("%0*d", totpDigits, code), nil
}

// 验证TOTP
func VerifyTOTP(secret, code string) bool {
    now := time.Now()
    // 检查当前时间步长和前后各1个步长
    for i := -totpSkew; i <= totpSkew; i++ {
        t := now.Add(time.Duration(i*totpPeriod) * time.Second)
        expected, err := GenerateTOTP(secret, t)
        if err != nil {
            continue
        }
        if hmac.Equal([]byte(expected), []byte(code)) {
            return true
        }
    }
    return false
}

// 生成otpauth:// URI（用于二维码）
func GenerateOTPAuthURI(secret, issuer, account string) string {
    u := url.URL{
        Scheme: "otpauth",
        Host:   "totp",
        Path:   fmt.Sprintf("/%s:%s", issuer, account),
    }
    q := u.Query()
    q.Set("secret", secret)
    q.Set("issuer", issuer)
    q.Set("algorithm", "SHA1")
    q.Set("digits", fmt.Sprintf("%d", totpDigits))
    q.Set("period", fmt.Sprintf("%d", totpPeriod))
    u.RawQuery = q.Encode()
    return u.String()
}

// ==================== MFA服务集成 ====================

type MFAService struct {
    db *sql.DB
}

// 启用MFA
func (s *MFAService) EnableMFA(ctx context.Context, userID int64) (*MFASetupResponse, error) {
    // 生成密钥

    if err != nil {
        return nil, err
    }

    // 临时存储（等用户验证后再正式启用）
    _, err = s.db.ExecContext(ctx,
        "UPDATE users SET mfa_secret_pending = $1 WHERE id = $2",
        secret, userID,
    )
    if err != nil {
        return nil, err
    }

    // 生成二维码URI
    uri := GenerateOTPAuthURI(secret, "MyApp", fmt.Sprintf("user_%d", userID))

    return &MFASetupResponse{
        Secret: secret,
        QRCode: uri,
    }, nil
}

// 确认启用MFA（用户输入验证码验证）
func (s *MFAService) ConfirmMFA(ctx context.Context, userID int64, code string) error {
    var pendingSecret string
    err := s.db.QueryRowContext(ctx,
        "SELECT mfa_secret_pending FROM users WHERE id = $1",
        userID,
    ).Scan(&pendingSecret)
    if err != nil {
        return err
    }

    if !VerifyTOTP(pendingSecret, code) {
        return fmt.Errorf("invalid TOTP code")
    }

    // 正式启用
    _, err = s.db.ExecContext(ctx,
        "UPDATE users SET mfa_secret = $1, mfa_enabled = true, mfa_secret_pending = NULL WHERE id = $2",
        pendingSecret, userID,
    )
    return err
}

// 登录时验证MFA
func (s *MFAService) VerifyMFALogin(ctx context.Context, userID int64, code string) (bool, error) {
    var secret string
    var mfaEnabled bool
    err := s.db.QueryRowContext(ctx,
        "SELECT mfa_secret, mfa_enabled FROM users WHERE id = $1",
        userID,
    ).Scan(&secret, &mfaEnabled)
    if err != nil {
        return false, err
    }

    if !mfaEnabled {
        return true, nil // MFA未启用，直接通过
    }

    return VerifyTOTP(secret, code), nil
}
```

## 8. RBAC/ABAC权限模型⭐⭐⭐

### 8.1 权限模型对比

| 模型 | 全称 | 原理 | 适用场景 | 复杂度 |
|------|------|------|----------|--------|
| **ACL** | Access Control List | 直接给用户分配资源权限 | 简单系统 | 低 |

### 8.2 RBAC数据库设计

```
经典五表模型（RBAC0/RBAC1）：

│  users   │     │  user_roles  │     │  roles   │
├──────────┤     ├──────────────┤     ├──────────┤

                                     └──────┬───────┘
                                            │
                                     ┌──────┴───────┐
                                     │ permissions  │
                                     ├──────────────┤
                                     │ id           │
                                     │ resource     │
```

```sql
-- 用户表
CREATE TABLE users (
    id         BIGSERIAL PRIMARY KEY,
    username   VARCHAR(64) UNIQUE NOT NULL,
    email      VARCHAR(128) UNIQUE NOT NULL,
    password_hash VARCHAR(256) NOT NULL,
    status     SMALLINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 角色表
CREATE TABLE roles (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(64) UNIQUE NOT NULL,  -- admin, editor, viewer
    description TEXT,
    parent_id   BIGINT REFERENCES roles(id),  -- 角色继承（RBAC1）
    created_at  TIMESTAMP DEFAULT NOW()
);

-- 权限表
CREATE TABLE permissions (
    id       BIGSERIAL PRIMARY KEY,
    resource VARCHAR(128) NOT NULL,  -- article, user, order
    action   VARCHAR(32) NOT NULL,   -- create, read, update, delete
    description TEXT,
    UNIQUE(resource, action)
);

-- 用户-角色关联表
CREATE TABLE user_roles (
    user_id    BIGINT REFERENCES users(id) ON DELETE CASCADE,
    role_id    BIGINT REFERENCES roles(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT NOW(),
    PRIMARY KEY (user_id, role_id)
);

-- 角色-权限关联表
CREATE TABLE role_permissions (
    role_id       BIGINT REFERENCES roles(id) ON DELETE CASCADE,
    permission_id BIGINT REFERENCES permissions(id) ON DELETE CASCADE,
    PRIMARY KEY (role_id, permission_id)
);

-- 查询用户所有权限
SELECT DISTINCT p.resource, p.action
FROM users u
JOIN user_roles ur ON u.id = ur.user_id
JOIN roles r ON ur.role_id = r.id
JOIN role_permissions rp ON r.id = rp.role_id
JOIN permissions p ON rp.permission_id = p.id
WHERE u.id = 1;
```

### 8.3 Go实现RBAC权限中间件

```go
package rbac

import (
    "context"
    "net/http"
    "sync"
    "time"
)

// Permission 权限
type Permission struct {
    Resource string // article, user, order
    Action   string // create, read, update, delete
}

// PermissionService 权限查询服务
type PermissionService struct {
    db    *sql.DB
    cache sync.Map // 简单缓存，生产用Redis
}

// GetUserPermissions 获取用户所有权限
func (s *PermissionService) GetUserPermissions(ctx context.Context, userID int64) ([]Permission, error) {
    // 先查缓存
    cacheKey := fmt.Sprintf("perms:%d", userID)
    if cached, ok := s.cache.Load(cacheKey); ok {
        return cached.([]Permission), nil
    }

    rows, err := s.db.QueryContext(ctx, `
        SELECT DISTINCT p.resource, p.action
        FROM user_roles ur
        JOIN role_permissions rp ON ur.role_id = rp.role_id
        JOIN permissions p ON rp.permission_id = p.id
        WHERE ur.user_id = $1
    `, userID)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var perms []Permission
    for rows.Next() {
        var p Permission
        rows.Scan(&p.Resource, &p.Action)
        perms = append(perms, p)
    }

    // 缓存5分钟
    s.cache.Store(cacheKey, perms)
    go func() {
        time.Sleep(5 * time.Minute)
        s.cache.Delete(cacheKey)
    }()

    return perms, nil
}

// HasPermission 检查用户是否有指定权限
func (s *PermissionService) HasPermission(ctx context.Context, userID int64, resource, action string) (bool, error) {
    perms, err := s.GetUserPermissions(ctx, userID)
    if err != nil {
        return false, err
    }
    for _, p := range perms {
        if p.Resource == resource && p.Action == action {
            return true, nil
        }
        // 支持通配符 *
        if p.Resource == "*" && p.Action == "*" {
            return true, nil
        }
        if p.Resource == resource && p.Action == "*" {
            return true, nil
        }
    }
    return false, nil
}

// RequirePermission 权限校验中间件
func RequirePermission(permService *PermissionService, resource, action string) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // 从上下文获取用户信息（JWT中间件已设置）
            claims, ok := GetUserFromContext(r.Context())
            if !ok {
                http.Error(w, `{"error":"unauthorized"}`, http.StatusUnauthorized)
                return
            }

            // 检查权限
            hasPerms, err := permService.HasPermission(r.Context(), claims.UserID, resource, action)
            if err != nil {
                http.Error(w, `{"error":"internal server error"}`, http.StatusInternalServerError)
                return
            }
            if !hasPerms {
                http.Error(w, `{"error":"forbidden","required":"` +resource+ ":"+action+`"}`,
                    http.StatusForbidden)
                return
            }

            next.ServeHTTP(w, r)
        })
    }
}

// 使用示例
func setupRoutes(permService *PermissionService) {
    mux := http.NewServeMux()

    // 文章列表 - 需要 article:read 权限

    )
```

### 8.5 ABAC（基于属性的访问控制）

```go
// ABAC示例：基于属性的策略
type AccessRequest struct {
    Subject    SubjectAttrs    // 谁
    Resource   ResourceAttrs   // 访问什么
    Action     string          // 做什么
    Environment EnvironmentAttrs // 环境条件
}

type SubjectAttrs struct {
    UserID     int64
    Department string
    Role       string
    Level      int
}

type ResourceAttrs struct {
    Type       string // article, order
    Owner      int64
    Department string
    Sensitivity string // public, internal, confidential
}

type EnvironmentAttrs struct {
    Time     time.Time
    IP       string
    Location string
}

// ABAC策略引擎
type ABACEngine struct {
    policies []Policy
}

type Policy struct {
    Name      string
    Condition func(req *AccessRequest) bool
}

func NewABACEngine() *ABACEngine {
    engine := &ABACEngine{}

    // 策略1：用户只能编辑自己部门的文章
    engine.AddPolicy("department_edit", func(req *AccessRequest) bool {
        if req.Action == "edit" && req.Resource.Type == "article" {
            return req.Subject.Department == req.Resource.Department
        }
        return false
    })

    // 策略2：机密文档只有Level>=3的员工可查看
    engine.AddPolicy("confidential_access", func(req *AccessRequest) bool {
        if req.Resource.Sensitivity == "confidential" {
            return req.Subject.Level >= 3
        }
        return true // 非机密不限制
    })

    // 策略3：非工作时间禁止访问内部系统
    engine.AddPolicy("working_hours", func(req *AccessRequest) bool {
        hour := req.Environment.Time.Hour()
        return hour >= 8 && hour <= 22
    })

    return engine
}

func (e *ABACEngine) AddPolicy(name string, condition func(*AccessRequest) bool) {
    e.policies = append(e.policies, Policy{Name: name, Condition: condition})
}

func (e *ABACEngine) IsAllowed(req *AccessRequest) (bool, string) {
    for _, policy := range e.policies {
        if !policy.Condition(req) {
            return false, fmt.Sprintf("denied by policy: %s", policy.Name)
        }
    }
    return true, ""
}
```

### 9.1 HMAC签名认证

```
原理：客户端和服务端共享密钥，请求时用密钥对请求内容做HMAC签名，
      服务端用相同密钥验证签名，确保请求未被篡改。

签名流程：
1. 构造待签名字符串（方法+路径+时间戳+Body哈希）
2. 用密钥做HMAC-SHA256
3. 将签名放入请求头
```

```go
package apisecurity

import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "fmt"
    "net/http"
    "strconv"
    "time"
)

// 客户端：签名请求
func SignRequest(req *http.Request, apiKey, apiSecret string, body []byte) {
    timestamp := fmt.Sprintf("%d", time.Now().Unix())

    // 构造待签名字符串
    bodyHash := sha256.Sum256(body)
    stringToSign := fmt.Sprintf("%s\n%s\n%s\n%s",
        req.Method,
        req.URL.Path,
        timestamp,
        hex.EncodeToString(bodyHash[:]),
    )

    // HMAC-SHA256签名
    mac := hmac.New(sha256.New, []byte(apiSecret))
    mac.Write([]byte(stringToSign))
    signature := hex.EncodeToString(mac.Sum(nil))

    // 设置请求头
    req.Header.Set("X-API-Key", apiKey)
    req.Header.Set("X-Timestamp", timestamp)
    req.Header.Set("X-Signature", signature)
}

// 服务端：验证签名中间件
func HMACAuthMiddleware(getSecret func(apiKey string) (string, error)) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            apiKey := r.Header.Get("X-API-Key")
            timestamp := r.Header.Get("X-Timestamp")
            signature := r.Header.Get("X-Signature")

            if apiKey == "" || timestamp == "" || signature == "" {
                http.Error(w, "Missing auth headers", http.StatusUnauthorized)
                return
            }

            // 检查时间戳（防重放攻击，允许±5分钟）
            ts, _ := strconv.ParseInt(timestamp, 10, 64)
            if abs(time.Now().Unix()-ts) > 300 {
                http.Error(w, "Request expired", http.StatusUnauthorized)
                return
            }

            // 获取密钥
            secret, err := getSecret(apiKey)
            if err != nil {
                http.Error(w, "Invalid API key", http.StatusUnauthorized)
                return
            }

            // 读取body
            body, _ := io.ReadAll(r.Body)
            r.Body = io.NopCloser(bytes.NewBuffer(body))

            // 重新计算签名
            bodyHash := sha256.Sum256(body)
            stringToSign := fmt.Sprintf("%s\n%s\n%s\n%s",
                r.Method, r.URL.Path, timestamp,
                hex.EncodeToString(bodyHash[:]),
            )
            mac := hmac.New(sha256.New, []byte(secret))
            mac.Write([]byte(stringToSign))
            expected := hex.EncodeToString(mac.Sum(nil))

            // 恒等时间比较（防时序攻击）
            if !hmac.Equal([]byte(signature), []byte(expected)) {
                http.Error(w, "Invalid signature", http.StatusUnauthorized)
                return
            }

            next.ServeHTTP(w, r)
        })
    }
}
```

### 9.2 Rate Limiting（限流）

```go
package ratelimit

import (
    "net/http"
    "sync"
    "time"

    "golang.org/x/time/rate"
)

// 基于IP的限流器
type IPRateLimiter struct {
    mu       sync.RWMutex
    limiters map[string]*rate.Limiter
    rate     rate.Limit // 每秒允许的请求数
    burst    int        // 突发容量
}

func NewIPRateLimiter(r rate.Limit, burst int) *IPRateLimiter {
    rl := &IPRateLimiter{
        limiters: make(map[string]*rate.Limiter),
        rate:     r,
        burst:    burst,
    }
    // 定期清理过期限流器
    go rl.cleanup()
    return rl
}

func (rl *IPRateLimiter) GetLimiter(ip string) *rate.Limiter {
    rl.mu.Lock()
    defer rl.mu.Unlock()
    limiter, ok := rl.limiters[ip]
    if !ok {
        limiter = rate.NewLimiter(rl.rate, rl.burst)
        rl.limiters[ip] = limiter
    }
    return limiter
}

func (rl *IPRateLimiter) cleanup() {
    ticker := time.NewTicker(10 * time.Minute)
    for range ticker.C {
        rl.mu.Lock()
        // 简单策略：全部清除，让其自然重建
        rl.limiters = make(map[string]*rate.Limiter)
        rl.mu.Unlock()
    }
}

// 限流中间件
func RateLimitMiddleware(limiter *IPRateLimiter) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            ip := r.RemoteAddr
            if forwarded := r.Header.Get("X-Forwarded-For"); forwarded != "" {
                ip = forwarded
            }

            if !limiter.GetLimiter(ip).Allow() {
                w.Header().Set("Retry-After", "1")
                http.Error(w, `{"error":"rate limit exceeded"}`, http.StatusTooManyRequests)
                return
            }

            next.ServeHTTP(w, r)
        })
    }
}
```

```go
// CORS中间件
func CORSMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        origin := r.Header.Get("Origin")

        // 允许的源（生产环境用白名单）
        allowedOrigins := map[string]bool{
            "https://www.example.com": true,
            "https://app.example.com": true,
        }

        if allowedOrigins[origin] {
            w.Header().Set("Access-Control-Allow-Origin", origin)
            w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
            w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization, X-API-Key")
            w.Header().Set("Access-Control-Allow-Credentials", "true")
            w.Header().Set("Access-Control-Max-Age", "86400") // 预检缓存24小时
        }

        // 预检请求直接返回
        if r.Method == http.MethodOptions {
            w.WriteHeader(http.StatusNoContent)
            return
        }

        next.ServeHTTP(w, r)
    })
}
```

```go
package auth

import (
    "context"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    "github.com/golang-jwt/jwt/v5"
    "golang.org/x/crypto/bcrypt"

type AuthHandler struct {
    userRepo    UserRepository
    authService *AuthService
    mfaService  *MFAService
    permService *PermissionService
    oauthMgr    *OAuthManager
}

// 注册
func (h *AuthHandler) Register(w http.ResponseWriter, r *http.Request) {
    var req RegisterRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        writeError(w, http.StatusBadRequest, "invalid request body")
        return
    }

    // 1. 参数校验
    if err := validatePasswordStrength(req.Password); err != nil {
        writeError(w, http.StatusBadRequest, err.Error())
        return
    }

    // 2. 检查用户名/邮箱是否已存在
    exists, _ := h.userRepo.ExistsByUsernameOrEmail(r.Context(), req.Username, req.Email)
    if exists {
        writeError(w, http.StatusConflict, "username or email already exists")
        return
    }

    // 3. 哈希密码
    hashedPassword, err := bcrypt.GenerateFromPassword([]byte(req.Password), 12)
    if err != nil {
        writeError(w, http.StatusInternalServerError, "failed to hash password")
        return
    }

    // 4. 创建用户
    user, err := h.userRepo.Create(r.Context(), &User{
        Username:     req.Username,
        Email:        req.Email,
        PasswordHash: string(hashedPassword),
    })
    if err != nil {
        writeError(w, http.StatusInternalServerError, "failed to create user")
        return
    }

    // 5. 分配默认角色
    h.userRepo.AssignRole(r.Context(), user.ID, "viewer")

    // 6. 生成Token
    tokenPair, err := h.authService.generateTokenPair(r.Context(), user.ID, user.Username, "viewer")
    if err != nil {
        writeError(w, http.StatusInternalServerError, "failed to generate token")
        return
    }

    writeJSON(w, http.StatusCreated, map[string]interface{}{
        "user":  user,
        "token": tokenPair,
    })
}

// 登录（支持MFA）
func (h *AuthHandler) Login(w http.ResponseWriter, r *http.Request) {
    var req LoginRequest
    json.NewDecoder(r.Body).Decode(&req)

    // 1. 查找用户
    user, err := h.userRepo.FindByUsername(r.Context(), req.Username)
    if err != nil {
        // 防时序攻击
        bcrypt.CompareHashAndPassword([]byte("$2a$12$dummy"), []byte(req.Password))
        writeError(w, http.StatusUnauthorized, "invalid credentials")
        return
    }

    // 2. 验证密码
    if err := bcrypt.CompareHashAndPassword([]byte(user.PasswordHash), []byte(req.Password)); err != nil {
        h.recordLoginLog(r, user.ID, "password", false, "wrong password")
        writeError(w, http.StatusUnauthorized, "invalid credentials")
        return
    }

    // 3. 检查是否启用MFA
    if user.MFAEnabled {
        if req.MFACode == "" {
            // 需要MFA验证
            writeJSON(w, http.StatusOK, map[string]interface{}{
                "mfa_required": true,
                "message":      "please provide MFA code",
            })
            return
        }
        // 验证MFA
        if !VerifyTOTP(user.MFASecret, req.MFACode) {
            h.recordLoginLog(r, user.ID, "mfa", false, "invalid mfa code")
            writeError(w, http.StatusUnauthorized, "invalid MFA code")
            return
        }
    }

    // 4. 获取用户角色
    roles, _ := h.userRepo.GetUserRoles(r.Context(), user.ID)
    primaryRole := "viewer"
    if len(roles) > 0 {
        primaryRole = roles[0].Name
    }

    // 5. 生成Token对
    tokenPair, err := h.authService.generateTokenPair(r.Context(), user.ID, user.Username, primaryRole)
    if err != nil {
        writeError(w, http.StatusInternalServerError, "failed to generate token")
        return
    }

    // 6. 记录登录日志
    h.recordLoginLog(r, user.ID, "password", true, "")

    writeJSON(w, http.StatusOK, map[string]interface{}{
        "user":  user,
        "token": tokenPair,
    })
}

// OAuth第三方登录
func (h *AuthHandler) OAuthCallback(w http.ResponseWriter, r *http.Request) {
    provider := r.URL.Query().Get("provider") // github, wechat
    code := r.URL.Query().Get("code")
    state := r.URL.Query().Get("state")

    // 1. 验证state
    if !h.oauthMgr.ValidateState(state) {
        writeError(w, http.StatusBadRequest, "invalid state")
        return
    }

    // 2. 用code换Token
    oauthUser, err := h.oauthMgr.ExchangeAndGetUser(r.Context(), provider, code)
    if err != nil {
        writeError(w, http.StatusInternalServerError, "oauth exchange failed")
        return
    }

    // 3. 查找或创建用户
    user, err := h.userRepo.FindOrCreateByOAuth(r.Context(), provider, oauthUser)
    if err != nil {
        writeError(w, http.StatusInternalServerError, "failed to find or create user")
        return
    }

    // 4. 生成Token
    roles, _ := h.userRepo.GetUserRoles(r.Context(), user.ID)
    primaryRole := "viewer"
    if len(roles) > 0 {
        primaryRole = roles[0].Name
    }

    tokenPair, _ := h.authService.generateTokenPair(r.Context(), user.ID, user.Username, primaryRole)

    h.recordLoginLog(r, user.ID, "oauth_"+provider, true, "")

    writeJSON(w, http.StatusOK, map[string]interface{}{
        "user":  user,
        "token": tokenPair,
    })
}

// 辅助函数
func writeJSON(w http.ResponseWriter, status int, data interface{}) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    json.NewEncoder(w).Encode(data)
}

    writeJSON(w, status, map[string]string{"error": message})
```

```go
    mux.Handle("POST /api/user/mfa/enable",
        JWTAuthMiddleware(http.HandlerFunc(authHandler.EnableMFA)))
    mux.Handle("POST /api/user/mfa/confirm",
        JWTAuthMiddleware(http.HandlerFunc(authHandler.ConfirmMFA)))

    // === 需要认证+授权的接口 ===
    // 文章管理
    mux.Handle("GET /api/articles",
        JWTAuthMiddleware(RequirePermission(permService, "article", "read")(
            http.HandlerFunc(articleHandler.List))))
    mux.Handle("POST /api/articles",
        JWTAuthMiddleware(RequirePermission(permService, "article", "create")(
            http.HandlerFunc(articleHandler.Create))))

    // 管理后台
    mux.Handle("GET /api/admin/users",
        JWTAuthMiddleware(RequirePermission(permService, "user", "read")(
            http.HandlerFunc(adminHandler.ListUsers))))
    mux.Handle("POST /api/admin/roles",
        JWTAuthMiddleware(RequirePermission(permService, "system", "manage")(
            http.HandlerFunc(adminHandler.ManageRoles))))

    // 全局中间件链
    handler := CORSMiddleware(
        RateLimitMiddleware(rateLimiter)(
            LoggingMiddleware(
                RecoveryMiddleware(mux),
            ),
        ),
    )

    fmt.Println("Server running on :8080")
    http.ListenAndServe(":8080", handler)
```

## 11. 面试题自查

### 认证授权治理补充

- 认证和授权是两个独立关注点，不能混为一谈；Token管理不等于"能登录就行"，而是要设计完整的签发、验证、续期、撤销、黑名单生命周期。
- 权限模型的选择不是越复杂越好，RBAC覆盖大多数场景；引入ABAC前先确认RBAC真的不够用，否则策略维护成本远大于收益。
- API安全是体系化工程，不是"加了JWT就安全"；传输加密、签名防篡改、时间戳防重放、限流防滥用、日志审计一个都不能少。

### 初级高频速刷

**1. 认证（Authentication）和授权（Authorization）有什么区别？**

认证解决的是"你是谁"的问题，授权解决的是"你能做什么"的问题。认证在前授权在后，HTTP状态码上认证失败是401，授权失败是403。面试中经常混淆这两个概念，区分清楚是基础。

**2. Cookie和Session分别在哪里存储？为什么需要Session？**

Cookie存储在客户端浏览器中，Session存储在服务端。HTTP协议是无状态的，每次请求都是独立的，Session用来在服务端保持用户的登录状态，Cookie中只存SessionID作为关联凭证。

**3. JWT相比Session有什么优势和劣势？**

优势：无状态天然支持分布式扩展、不需要服务端存储、跨域友好、天然防CSRF。劣势：Token无法主动撤销（需要黑名单）、Token体积比SessionID大（每次请求都携带）、Payload不加密只有Base64编码。

**4. 为什么密码不能用MD5或SHA256直接哈希存储？**

因为MD5/SHA256是快速哈希函数，GPU加速下每秒可计算数十亿次，配合彩虹表可以快速碰撞破解。密码存储应该使用bcrypt/argon2这类"慢哈希"，通过可调的计算成本让暴力破解在时间上不可行。

**5. HTTPS为什么是API安全的基础？仅有HTTPS够不够？**

HTTPS通过TLS加密保证传输过程中数据不被窃听和篡改，是安全的基础层。但HTTPS只保护传输层，不能防止Token被XSS窃取、API被滥用调用、请求被重放等应用层问题，还需要认证授权、签名、限流等措施配合。

### 中级高频进阶

**6. OAuth2.0的授权码模式为什么比隐式模式安全？**

授权码模式中access_token的交换发生在后端服务器之间（用code+client_secret换token），token不会暴露在浏览器中。隐式模式直接在URL fragment中返回token，容易被中间人截获或浏览器历史记录泄露。

**7. JWT的Access Token和Refresh Token为什么要分开？**

Access Token短有效期（15min-2h），即使泄露危害窗口小；Refresh Token长有效期（7-30天）但只用来刷新AT，不直接访问资源，通常存在更安全的地方（HttpOnly Cookie）。分离后兼顾了安全性和用户体验。

**8. RBAC和ABAC在什么场景下该选择哪个？**

RBAC适合权限相对固定、基于角色就能满足的场景（大多数企业应用）。ABAC适合需要基于多种属性动态判断的场景（如根据部门、地理位置、时间、数据敏感度综合判断）。ABAC更灵活但策略维护复杂度高得多。

**9. 如何实现JWT的主动失效（用户修改密码后让所有Token失效）？**

三种方案：①Redis黑名单，将被撤销的Token加入黑名单，验证时先检查黑名单；②版本号机制，用户表存token_version，签发时写入JWT，验证时对比版本号；③短有效期+不刷新Refresh Token，修改密码时删除所有Refresh Token。

**10. PKCE是什么？为什么SPA和移动端需要它？**

PKCE（Proof Key for Code Exchange）解决SPA和移动端无法安全存储client_secret的问题。原理是客户端生成随机code_verifier并发送其SHA256哈希（code_challenge）给授权服务器，换token时发送原始code_verifier验证。即使authorization code被截获，没有code_verifier也无法换取token。

### 深入问答

#### Q1: JWT的三种签名算法HS256/RS256/ES256分别适用什么场景？切换算法时需要注意什么？

**答**：HS256（HMAC-SHA256）是对称加密，签发和验证用同一个密钥，适合单体应用或完全可信的内部服务间通信。RS256（RSA-SHA256）是非对称加密，私钥签发公钥验证，适合微服务架构（Auth服务持有私钥，其他服务只需公钥验证）。ES256（ECDSA-SHA256）同样非对称，密钥更短、签名更快，是RS256的性能优化替代。切换算法时必须注意：①验证端要明确指定允许的算法列表防止alg替换攻击；②密钥要同步更新；③过渡期需要同时支持新旧算法。

#### Q2: OAuth2.0的四种授权模式分别在什么场景下使用？为什么隐式模式被废弃？

**答**：授权码模式：有后端的Web应用（最安全最常用）。授权码+PKCE：SPA/移动端（替代隐式模式）。密码模式：自家完全可信的应用（如官方App）。客户端凭证：服务间通信（无用户参与的M2M）。隐式模式被废弃因为：①token直接暴露在URL中易泄露；②不支持refresh token；③PKCE方案完全替代了它的使用场景并且更安全。OAuth 2.1草案已经正式移除隐式模式。

#### Q3: 分布式Session有哪些方案？Redis存储Session时如何处理Redis故障？

**答**：方案有四种：①Sticky Session（Nginx按IP固定路由，简单但负载不均+单点故障）②Session复制（服务器间同步，网络开销大）③Redis集中存储（推荐，扩展性好）④JWT替代Session（无状态，但无法主动失效）。Redis故障处理：部署Redis Sentinel或Cluster保证高可用；实现降级逻辑，Redis不可用时临时切换到本地Session或允许重新登录；设置合理的连接超时避免阻塞请求。

#### Q4: 如何防止JWT被XSS窃取？localStorage和HttpOnly Cookie各有什么利弊？

**答**：localStorage中的JWT可以被任何JS代码读取，一旦页面存在XSS漏洞token立即泄露。HttpOnly Cookie中的JWT不能被JS访问，XSS无法直接窃取，但会自动随请求发送需要防CSRF。推荐方案：Access Token存HttpOnly Secure SameSite Cookie，配合CSRF Token（双重Cookie校验或自定义Header）。或者更激进的方案：AT存内存变量（刷新页面丢失），RT存HttpOnly Cookie用于自动续签。

#### Q5: 密码哈希的"加盐"是什么？为什么bcrypt比SHA256+盐更安全？

**答**：盐（Salt）是随机字符串，和密码拼接后再哈希。每个用户的盐不同，即使两个用户密码相同，哈希结果也不同，防止彩虹表攻击。bcrypt比SHA256+盐更安全因为：①bcrypt内置盐值管理不需要单独存储；②bcrypt有可调的cost因子（每增加1计算时间翻倍），可以对抗硬件提升；③SHA256是快速哈希，设计目标是快，而bcrypt是专为密码设计的慢哈希，GPU加速效果差。生产推荐bcrypt cost=12或argon2id。

#### Q6: TOTP（基于时间的一次性密码）为什么允许±1个时间步长的误差？不验证之前的code会有什么风险？

**答**：允许±1步长是因为客户端和服务端时钟可能有几秒偏差，严格要求当前30秒窗口会导致合法用户验证失败。风险是同一个code在90秒窗口内都有效，理论上可以被重放。防护措施：记录每个用户最后使用的TOTP计数器值，拒绝使用≤上次计数器值的code（防重放）。另外恢复码（Recovery Codes）要单独设计为一次性使用，并在启用MFA时同时生成提供给用户离线保存。

#### Q7: SSO实现中，CAS和OIDC的核心区别是什么？如何选择？

**答**：CAS是自有协议，基于Service Ticket（一次性票据），每次验证需要后端回调CAS Server验票，流程简单但功能有限。OIDC基于OAuth2.0，返回JWT格式的id_token，客户端可以本地验证不需要每次回调，支持丰富的用户属性声明。选择原则：新项目首选OIDC（标准化、移动端友好、生态丰富）；企业内部老系统用CAS够用；跨企业B2B场景可能需要SAML2.0（企业IT标准）。

#### Q8: OAuth2.0中的state参数为什么重要？不校验state会有什么安全问题？

**答**：state参数用于防止CSRF攻击。攻击场景：攻击者发起OAuth流程获取自己的authorization code，然后构造回调URL诱导受害者点击，受害者的账号就会绑定到攻击者的第三方账号。state的正确做法：①服务端生成随机字符串作为state②存入Session或签名加密③回调时验证state与之前生成的一致。不校验state等于OAuth流程完全暴露在CSRF攻击之下。

#### Q9: RBAC权限缓存应该设置多长的TTL？权限变更时如何保证及时生效？

**答**：TTL取决于业务对权限变更实时性的要求，通常5-15分钟是合理范围。保证及时生效的方案：①主动失效：权限变更时主动删除用户的权限缓存（精确失效）；②版本号：角色表维护version字段，缓存中存version，验证时对比；③广播机制：通过Redis Pub/Sub或消息队列通知所有服务节点清除缓存；④强制刷新：管理员操作后通过API触发特定用户的缓存刷新。生产环境通常用①+③组合。

#### Q10: 如何设计一个支持"用户在一台设备登录后其他设备被挤下线"的功能？

**答**：基于Token版本号方案：①用户表增加token_version字段，每次登录时递增；②JWT签发时将当前version写入Token；③验证Token时比较JWT中的version与数据库的version，不一致则拒绝；④新登录时递增version，旧Token自然失效。或者基于Redis：登录时在Redis中记录`user:{id}:current_token`，验证时检查当前Token是否匹配。后者可以做到更精细的多设备管理（如按设备类型限制同时在线数）。

#### Q11: Casbin的模型定义中matchers表达式有什么局限性？什么时候需要自己实现权限引擎？

**答**：Casbin的matchers基于字符串表达式，支持keyMatch、regexMatch等内置函数，但对复杂业务逻辑（如跨表查询、动态数据权限）支持有限，且调试困难。需要自己实现的场景：①数据级权限（"只能看自己部门的数据"需要查询数据归属）②复杂条件组合（涉及多个业务维度的动态判断）③性能要求极高（Casbin每次enforce有匹配开销，超大规模策略集可能成为瓶颈）。建议：简单RBAC用Casbin开箱即用，复杂场景在Casbin基础上扩展或自建。

#### Q12: API的HMAC签名方案中，时间戳和Nonce分别防什么攻击？只有一个够不够？

**答**：时间戳防重放攻击——拒绝超过时间窗口（如±5分钟）的请求。Nonce（随机数/唯一ID）防精确重放——即使在时间窗口内也不能用同一个请求重复调用。只有时间戳：时间窗口内可以重放。只有Nonce：需要服务端永久存储所有Nonce，存储压力大。两者结合：时间戳缩小窗口范围，Nonce只需要在窗口期内去重（过期Nonce可清除），兼顾安全性和存储开销。

#### Q13: 微服务架构中，认证应该在网关做还是各服务自己做？

**答**：推荐分层方案：**网关统一认证**（验证Token有效性、提取用户身份信息、基础限流）+ **各服务自主授权**（根据业务逻辑检查权限）。原因：①认证逻辑通用，重复实现浪费且易出错；②授权逻辑与业务强耦合，放网关会导致网关过于臃肿；③网关用RS256公钥验证JWT，不需要共享密钥。实现：网关验证JWT后将用户ID、角色等信息放入请求头（X-User-ID等）转发给下游服务，下游服务信任网关（内网通信）并据此做授权判断。

#### Q14: Refresh Token Rotation是什么？如何检测Token被盗用？

**答**：Refresh Token Rotation：每次使用RT换取新AT时，同时签发新的RT，旧RT失效（一次性使用）。检测盗用：给每组关联的RT分配一个family ID。如果一个已被标记为"已使用"的RT再次被使用，说明有两方同时持有该RT（合法用户和攻击者），立即撤销该family下所有RT，强制用户重新登录。这比传统的"RT可复用"方案安全得多，是OAuth 2.0安全最佳实践。

#### Q15: bcrypt的cost从12改为14，性能影响有多大？生产环境如何选择合适的cost？

**答**：bcrypt的cost每增加1，计算时间翻倍。cost=12约需250ms，cost=14约需1s。选择原则：在目标服务器上实测，选择使单次哈希耗时在250ms-500ms之间的cost值。太低不够安全（GPU可暴力破解），太高影响登录体验和服务器吞吐量。建议：①线上测试不同cost的实际耗时；②定期评估（硬件升级后应提高cost）；③数据库存储哈希结果本身包含cost信息，可以渐进升级（用户下次登录时用新cost重新哈希）。

#### Q16: OAuth2.0中客户端凭证模式和JWT Bearer断言有什么区别？服务间认证推荐哪种？

**答**：客户端凭证模式：客户端用client_id+client_secret向授权服务器换取access_token，每次请求都要经过授权服务器。JWT Bearer断言（RFC 7523）：客户端用私钥自签JWT作为凭证直接发送，资源服务器用公钥验证，无需每次调用授权服务器。推荐：高频服务间调用用JWT Bearer（减少对授权服务器的依赖，降低延迟）；低频或需要集中管控的场景用客户端凭证模式（授权服务器可以随时撤销权限）。

#### Q17: 设计一个支持多租户的认证授权系统，核心需要考虑哪些点？

**答**：核心考虑：①**租户隔离**：每个租户的用户、角色、权限独立，数据库层面用tenant_id做隔离（共享数据库+tenant_id字段或独立Schema）；②**租户级配置**：不同租户可以有不同的密码策略、MFA要求、OAuth Provider配置；③**跨租户管理**：平台管理员可以管理所有租户，租户管理员只能管理自己租户内的用户和权限；④**Token设计**：JWT中包含tenant_id，所有API请求都校验tenant隔离；⑤**性能**：权限缓存按tenant_id+user_id做key，租户间不互相影响。

#### Q18: 如何实现数据级权限控制（如"用户只能看到自己创建的订单"）？

**答**：数据级权限不能仅靠RBAC解决，常见方案：①**SQL条件注入**：在查询时自动拼接`WHERE created_by = :userId`条件（可通过ORM中间件或Interceptor实现）；②**数据所有者标记**：数据表增加owner_id字段，查询时校验；③**部门/组织层级**：先查用户的组织路径，再查询该路径下的数据；④**标签/规则引擎**：用ABAC策略表达数据可见性规则。Go实现：定义DataScope接口，在Repository层统一注入数据过滤条件，确保所有查询都经过数据权限过滤。

```go
// 数据级权限过滤器
type DataScope interface {
    Apply(query *gorm.DB, userID int64, role string) *gorm.DB
}

// "只看自己的"
type SelfDataScope struct{}
func (s *SelfDataScope) Apply(query *gorm.DB, userID int64, role string) *gorm.DB {
    if role == "admin" {
        return query // 管理员不过滤
    }
    return query.Where("created_by = ?", userID)
}

// "看本部门的"
type DeptDataScope struct{ deptRepo DeptRepository }
func (s *DeptDataScope) Apply(query *gorm.DB, userID int64, role string) *gorm.DB {
    deptIDs, _ := s.deptRepo.GetUserDeptAndChildren(userID)
    return query.Where("dept_id IN ?", deptIDs)
}
```

#### Q19: 前后端分离架构中，如何安全地实现"记住我"功能？

**答**：方案：使用长期有效的Refresh Token存在HttpOnly Secure SameSite Cookie中。勾选"记住我"时RT有效期设为30天，不勾选时设为当次会话（关闭浏览器Cookie失效）。流程：①页面加载时用RT自动获取新AT（静默刷新）②AT存内存变量（最安全）③AT过期时自动用RT续签④用户主动登出时删除RT。安全要点：RT必须是HttpOnly Cookie防XSS，配合SameSite=Strict防CSRF，服务端维护RT白名单支持主动撤销。

#### Q20: 如何应对Token泄露后的紧急响应？设计一套Token撤销机制。

**答**：紧急响应流程：①**发现泄露**（监控异常IP/设备登录、用户举报）→ ②**立即撤销**（清除用户所有Refresh Token、将用户所有Access Token加入Redis黑名单、递增用户token_version）→ ③**强制重认证**（下次请求返回401强制重新登录）→ ④**通知用户**（邮件/短信通知密码可能泄露，建议修改密码）→ ⑤**审计追查**（查登录日志定位泄露途径）。

```go
// Token撤销服务
type TokenRevocationService struct {
    redis *redis.Client
    db    *sql.DB
}

func (s *TokenRevocationService) RevokeAllUserTokens(ctx context.Context, userID int64) error {
    // 1. 递增token版本号（使所有JWT失效）
    s.db.ExecContext(ctx,
        "UPDATE users SET token_version = token_version + 1 WHERE id = $1", userID)

    // 2. 删除所有Refresh Token
    iter := s.redis.Scan(ctx, 0, fmt.Sprintf("refresh:*:user:%d", userID), 100).Iterator()
    for iter.Next(ctx) {
        s.redis.Del(ctx, iter.Val())
    }

    // 3. 记录撤销事件
    s.db.ExecContext(ctx,
        "INSERT INTO security_events (user_id, event_type) VALUES ($1, 'token_revocation')", userID)

    return nil
}

// JWT验证时检查版本号
func (s *TokenRevocationService) ValidateTokenVersion(ctx context.Context, userID int64, tokenVersion int) bool {
    var currentVersion int
    s.db.QueryRowContext(ctx,
        "SELECT token_version FROM users WHERE id = $1", userID,
    ).Scan(&currentVersion)
    return tokenVersion == currentVersion
}
```

#### Q21: 如何设计认证系统的监控和告警？哪些指标是关键的？

**答**：关键监控指标：①**认证成功率/失败率**（突然升高的失败率可能是攻击或系统故障）②**同一IP/用户的失败次数**（暴力破解检测，阈值触发临时封禁）③**Token签发QPS和延迟**（认证服务性能监控）④**Refresh Token使用频率**（异常高频刷新可能是Token被盗用轮换）⑤**OAuth回调成功率**（第三方服务可用性）⑥**MFA验证失败率**（区分用户操作失误和攻击行为）。告警规则：单IP 5分钟内登录失败>20次→封禁IP 30分钟；单用户1小时内失败>10次→锁定账户；认证服务P99延迟>2s→告警。

#### Q22: Session固定攻击（Session Fixation）是什么？如何防御？

**答**：攻击流程：①攻击者获取一个有效的SessionID（如访问目标站点获取）②将此SessionID通过URL参数或XSS注入到受害者的请求中③受害者使用这个SessionID登录④攻击者用同一个SessionID以受害者身份访问。防御措施：①**登录成功后重新生成SessionID**（最关键，使攻击者持有的旧SessionID无效）②不接受URL中的SessionID③设置Cookie的HttpOnly和Secure属性④设置合理的Session过期时间。Go中实现：登录成功后调用`store.Delete(oldSID)`并`store.Create()`生成新的SessionID。

#### Q23: 在零信任架构下，认证授权的设计理念有什么变化？

**答**：零信任的核心原则是"永不信任，始终验证"。变化：①**网络边界不再是信任边界**：内网请求也需要认证授权，不能因为在VPN内就免认证；②**每次请求都验证**：不是登录一次就全程信任，而是每个请求都验证Token和权限；③**最小权限原则**：Token携带最小必要权限，按需请求额外权限；④**设备信任评估**：不仅验证用户身份，还评估设备健康状态（是否有安全补丁、是否公司设备）；⑤**持续评估**：用户行为分析，异常行为触发重新认证或降级权限。技术实现上OIDC+mTLS+RBAC+行为分析是典型的零信任认证授权栈。

#### Q24: 如何防止OAuth2.0中的开放重定向（Open Redirect）攻击？

**答**：攻击原理：攻击者构造恶意redirect_uri（如`redirect_uri=https://evil.com`），用户授权后Token被发送到攻击者的服务器。防御措施：①**严格校验redirect_uri**：授权服务器注册客户端时记录合法的redirect_uri列表，授权请求时精确匹配（不允许通配符或子路径匹配）；②**不允许fragment**：redirect_uri中不允许#fragment；③**使用完整URL匹配**：不仅匹配域名还要匹配完整路径；④**PKCE配合**：即使redirect_uri被篡改，没有code_verifier也无法换取token。OAuth 2.1已经要求redirect_uri精确匹配为强制规则。

#### Q25: 如何优雅地实现JWT密钥轮换（Key Rotation）？保证轮换期间不影响已签发的Token？

**答**：方案：使用JWK（JSON Web Key）+ kid（Key ID）机制。①JWT Header中包含kid字段标识使用了哪个密钥签名；②服务端维护密钥列表（如Redis中存多个版本的密钥）；③签发新Token用最新密钥，验证时根据kid查找对应密钥；④新密钥生效后保留旧密钥一段时间（≥Access Token最大有效期），等旧Token自然过期；⑤过渡期结束后删除旧密钥。JWKS（JWK Set）端点`/.well-known/jwks.json`让所有服务可以自动获取最新的公钥列表（RS256/ES256场景）。

```go
// 密钥管理
type KeyManager struct {
    keys    map[string][]byte // kid -> secret
    current string            // 当前使用的kid
    mu      sync.RWMutex
}

func (km *KeyManager) SignToken(claims jwt.Claims) (string, error) {
    km.mu.RLock()
    kid := km.current
    secret := km.keys[kid]
    km.mu.RUnlock()

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    token.Header["kid"] = kid
    return token.SignedString(secret)
}

func (km *KeyManager) VerifyToken(tokenString string) (*jwt.Token, error) {
    return jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
        kid, ok := token.Header["kid"].(string)
        if !ok {
            return nil, fmt.Errorf("missing kid")
        }
        km.mu.RLock()
        defer km.mu.RUnlock()
        secret, ok := km.keys[kid]
        if !ok {
            return nil, fmt.Errorf("unknown kid: %s", kid)
        }
        return secret, nil
    })
}

// 轮换密钥
func (km *KeyManager) RotateKey() {
    newKid := fmt.Sprintf("key-%d", time.Now().Unix())
    newSecret := make([]byte, 32)
    rand.Read(newSecret)

    km.mu.Lock()
    km.keys[newKid] = newSecret
    km.current = newKid
    km.mu.Unlock()

    // 延迟删除旧密钥（等待旧Token过期）
    go func(oldKeys []string) {
        time.Sleep(24 * time.Hour)
        km.mu.Lock()
        for _, k := range oldKeys {
            if k != km.current {
                delete(km.keys, k)
            }
        }
        km.mu.Unlock()
    }(km.getOldKeys())
}
```

### 开放式设计题

**D1：设计一个支持多租户、多应用的统一身份认证平台（类似Okta/Auth0），核心模块是什么？**

**参考思路**：
- 核心模块：用户目录（LDAP/SCIM同步）、认证引擎（密码/MFA/SSO/Social Login）、授权引擎（RBAC+ABAC）、Token服务（JWT签发/刷新/吊销）
- 多租户隔离：租户级配置（允许的认证方式、密码策略、MFA强制）、数据隔离（Schema隔离 or 行级隔离）
- 协议支持：OAuth2.0/OIDC对外、SAML2.0对接企业IdP、SCIM用户同步
- 安全：登录异常检测（异地/设备指纹/频率）、Token黑名单（Redis Bloom Filter）、审计日志全量记录
- 关键SLA：认证延迟<200ms、可用性99.99%（登录服务挂了=所有业务挂）

**D2：JWT Token泄露了，如何快速止损？系统层面如何防止类似问题？**

**参考思路**：
- 即时止损：JWT无法单独吊销（无状态的代价）→ Token黑名单（Redis存被吊销的jti）→ 强制该用户所有Token失效（版本号机制）
- 缩小爆炸半径：Access Token短期（15min）+ Refresh Token长期（7d），泄露的Access Token很快过期
- 防止泄露：HTTPS Only、Secure/HttpOnly Cookie、不在URL中传Token、敏感日志脱敏
- 检测：异常设备/IP登录告警、Token使用地理位置异常检测
