## 1. 计算机体系结构概述

### 1.1 冯·诺依曼架构

```
冯·诺依曼架构（存储程序计算机）：

┌─────────────────────────────────────────────────────────────┐
│                        CPU                                  │
│  ┌─────────────────┐    ┌─────────────────┐                │
│  │   控制单元 CU    │    │   算术逻辑单元   │                │
│  │  - 指令解码     │    │      ALU        │                │
│  │  - 控制信号     │    │  - 加减乘除      │                │
│  └────────┬────────┘    └────────┬────────┘                │
│           └──────────┬───────────┘                          │
│              ┌───────┴───────┐                              │
│              │   寄存器组    │                              │
│              └───────┬───────┘                              │
└──────────────────────┼──────────────────────────────────────┘
                       │ 系统总线
        ┌──────────────┼──────────────┐
   ┌────┴────┐    ┌────┴────┐   ┌────┴────┐
   │  内存    │    │   I/O   │   │   I/O   │
   │ Memory  │    │  设备1   │   │  设备2   │
   └─────────┘    └─────────┘   └─────────┘

核心特征：
1. 程序和数据存储在同一内存中
2. 指令和数据以二进制表示
3. 顺序执行指令（可通过跳转改变）
```

**冯·诺依曼瓶颈**：CPU 处理速度远大于内存访问速度，总线带宽成为瓶颈。解决方案包括 Cache 层次结构、预取、乱序执行等。

### 1.2 性能指标

```
CPU 性能公式：

执行时间 = 指令数 × CPI × 时钟周期 = 指令数 × CPI / 时钟频率

- 指令数：由编译器和ISA决定
- CPI (Cycles Per Instruction)：每条指令平均周期数
- 时钟频率：工艺和设计决定
```

**Amdahl 定律**：加速比 = 1 / [(1-P) + P/S]，其中 P 是可并行化比例，S 是并行加速倍数。串行部分决定加速比上限。

## 2. RISC vs CISC 指令集架构

### 2.1 对比

| 特性 | CISC (x86) | RISC (ARM/RISC-V) |
|------|-----------|-------------------|
| **指令长度** | 可变（1-15字节） | 固定（32位） |
| **指令复杂度** | 复杂指令多 | 简单指令为主 |
| **寄存器数量** | 较少（16个） | 较多（31-32个） |
| **内存访问** | 任意指令可访存 | Load/Store架构 |
| **解码复杂度** | 高（需微码） | 低（硬连线） |
| **功耗** | 较高 | 较低 |

### 2.2 现代融合

现代 x86 处理器前端将 CISC 指令解码为微操作（μops），后端以 RISC 方式执行。ARM 也增加了复杂指令。

## 3. 指令流水线

### 3.1 五级流水线

```
IF (取指) → ID (译码) → EX (执行) → MEM (访存) → WB (写回)

理想情况：流水线满载后 CPI=1
```

### 3.2 流水线冒险

| 冒险类型 | 原因 | 解决方案 |
|---------|------|---------|
| 结构冒险 | 硬件资源冲突 | 资源复制（I-Cache/D-Cache分离） |
| 数据冒险 | 指令间数据依赖 | 转发（Forwarding）、流水线暂停 |
| 控制冒险 | 分支改变程序流 | 分支预测、推测执行 |

### 3.3 分支预测

**2-bit 饱和计数器**：强不跳转 ↔ 弱不跳转 ↔ 弱跳转 ↔ 强跳转

**现代预测器**：TAGE（Tagged Geometric），使用不同长度历史，准确率 95%+。

### 3.4 乱序执行

```
取指 → 译码 → 寄存器重命名 → 发射队列 → 执行 → 写回 → 提交

关键组件：
- 寄存器重命名：消除 WAR/WAW 依赖
- 保留站：乱序发射
- ROB（重排序缓冲区）：顺序提交，保证精确异常
```

## 4. Cache 缓存体系

### 4.1 存储层次

```
寄存器 (~1周期, KB) → L1 (~4周期, 32-64KB) → L2 (~12周期, 256KB-1MB) 
→ L3 (~40周期, 8-64MB) → 主内存 (~200周期, GB级)
```

### 4.2 Cache 组织

**N-way 组相联**：
- 地址分为 Tag | Index | Offset
- Index 定位 Set，并行比较 N 个 Way 的 Tag

**典型配置**：L1 8-way，L2 8-16way，L3 16-20way

### 4.3 Cache Line

```
Cache Line 大小：通常 64 字节

伪共享（False Sharing）：
不同核心访问同一 Cache Line 的不同变量，导致 Cache Line 在核心间反复失效
解决：填充对齐，确保热点变量在不同 Cache Line
```

### 4.4 写策略

| 策略 | 写命中 | 写缺失 |
|------|--------|--------|
| Write-Through | 同时写 Cache 和内存 | Write Allocate / No-Allocate |
| Write-Back | 只写 Cache，标记 Dirty | Write Allocate |

## 6. 虚拟内存与 TLB

### 6.1 页表

```
四级页表（x86_64）：

虚拟地址 48位：
[47:39] PML4 | [38:30] PDPT | [29:21] PD | [20:12] PT | [11:0] Offset

CR3 → PML4 → PDPT → PD → PT → 物理页帧
```

### 6.2 TLB

**TLB（Translation Lookaside Buffer）**：页表缓存，避免多次内存访问。

```
TLB 层次：
- L1 ITLB/DTLB：~64-128项，全相联
- L2 TLB：~1024-2048项，4-8way
```

**TLB Miss 代价**：需要进行页表遍历（Page Table Walk），可能 4 次内存访问。

### 6.3 大页

| 页大小 | TLB 覆盖范围（1024项） | 场景 |
|--------|----------------------|------|
| 4KB | 4MB | 通用 |
| 2MB | 2GB | 大内存应用 |
| 1GB | 1TB | HPC、数据库 |

### 8.1 多核架构

```
典型多核结构：

┌─────────────────────────────────────────────────────────┐
│  Core 0      Core 1      Core 2      Core 3            │
│ ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐            │
│ │L1 I/D │  │L1 I/D │  │L1 I/D │  │L1 I/D │            │
│ └───┬───┘  └───┬───┘  └───┬───┘  └───┬───┘            │
│     │          │          │          │                 │
│ ┌───┴───┐  ┌───┴───┐  ┌───┴───┐  ┌───┴───┐            │
│ │  L2   │  │  L2   │  │  L2   │  │  L2   │            │
│ └───┬───┘  └───┬───┘  └───┬───┘  └───┬───┘            │
│     └──────────┴──────────┴──────────┘                 │
│                     │                                   │
│              ┌──────┴──────┐                           │
│              │   L3 共享   │                           │

```

### 8.2 NUMA 架构

```
NUMA（Non-Uniform Memory Access）：

┌──────────────┐                    ┌──────────────┐
│   Socket 0   │                    │   Socket 1   │
│ ┌──────────┐ │     QPI/UPI       │ ┌──────────┐ │
│ │  Cores   │◄├───────────────────►│ │  Cores   │ │
│ └────┬─────┘ │                    │ └────┬─────┘ │
│      │       │                    │      │       │
│ ┌────┴─────┐ │                    │ ┌────┴─────┐ │
│ │ Local    │ │                    │ │ Local    │ │
│ │ Memory   │ │                    │ │ Memory   │ │
│ └──────────┘ │                    │ └──────────┘ │
└──────────────┘                    └──────────────┘

本地访问：~80ns
远程访问：~140ns（跨Socket）
```

**NUMA 优化**：使用 `numactl` 绑定进程到本地 Node，避免远程内存访问。

## 9. 内存一致性与内存序

### 9.1 Cache 一致性协议

**MESI 协议**：

| 状态 | 含义 | 本地读 | 本地写 | 远程读 | 远程写 |
|------|------|--------|--------|--------|--------|
| M (Modified) | 已修改，独占 | 命中 | 命中 | 需写回 | 需写回 |
| E (Exclusive) | 干净，独占 | 命中 | →M | →S | 失效 |
| S (Shared) | 干净，共享 | 命中 | 需失效其他 | 命中 | 失效 |
| I (Invalid) | 无效 | 缺失 | 缺失 | - | - |

### 9.2 内存序模型

| 模型 | 特点 | 代表架构 |
|------|------|---------|
| TSO (Total Store Order) | 强序，Store-Store 有序 | x86 |
| 弱序 (Weak Ordering) | 需显式屏障 | ARM, RISC-V |

**内存屏障**：

```c
// x86 (TSO, 较少需要屏障)
asm volatile("mfence" ::: "memory");  // 全屏障

// ARM (弱序，常需屏障)
asm volatile("dmb ish" ::: "memory"); // 数据屏障
asm volatile("dsb ish" ::: "memory"); // 数据同步屏障
asm volatile("isb" ::: "memory");     // 指令同步屏障
```

## 10. 面试题自查

**Q1: 什么是冯·诺依曼瓶颈？如何缓解？**

CPU处理速度远大于内存访问速度，总线带宽成为瓶颈。缓解方法：多级Cache、预取、乱序执行、哈佛架构变体（I-Cache/D-Cache分离）。

**Q2: RISC 和 CISC 的核心区别？现代 CPU 是如何融合的？**

RISC 指令简单固定长度，Load/Store架构；CISC 指令复杂可变长度。现代 x86 前端解码 CISC 为微操作，后端 RISC 方式执行。

**Q3: 什么是数据冒险？如何通过转发解决？**

后续指令依赖前序指令未写回的结果。转发直接将执行阶段结果传递给需要的指令，无需等待写回阶段。

**Q4: 分支预测失败的代价是什么？**

需要清空流水线中的错误指令（Flush），现代CPU流水线深度15-20级，分支预测失败代价约15-20周期。

**Q5: 解释 Cache 的组相联映射。**

地址分为 Tag、Index、Offset。Index 定位 Set，每个 Set 有 N 个 Way，并行比较所有 Way 的 Tag。平衡了直接映射的冲突问题和全相联的复杂度。

**Q6: 什么是伪共享（False Sharing）？如何避免？**

多核访问同一 Cache Line 的不同变量，导致 Cache Line 频繁失效。避免方法：填充对齐，确保热点变量在不同 Cache Line（通常 64 字节对齐）。

**Q7: Write-Back 和 Write-Through 策略的区别？**

Write-Through 同时写 Cache 和内存，一致性好但带宽大；Write-Back 只写 Cache 标记 Dirty，替换时才写回，带宽小但一致性复杂。

**Q8: TLB Miss 的代价是什么？**

需要进行页表遍历（Page Table Walk），四级页表可能需要 4 次内存访问，代价约 200-800 周期。

**Q9: 为什么使用大页（Huge Pages）可以提升性能？**

减少 TLB 项数需求，相同 TLB 大小可覆盖更大内存范围，减少 TLB Miss。2MB 大页覆盖范围是 4KB 页的 512 倍。

**Q10: 什么是 NUMA？如何优化 NUMA 系统性能？**

Non-Uniform Memory Access，多 Socket 系统访问本地内存快、远程内存慢。优化：用 numactl 绑定进程到本地 Node，内存分配优先本地。

**Q11: MESI 协议中各状态的含义？**

Modified：已修改独占；Exclusive：干净独占；Shared：干净共享；Invalid：无效。核心间通过总线监听维护一致性。

**Q12: x86 TSO 和 ARM 弱序模型的区别？**

x86 TSO 保证 Store-Store 有序、Load-Load 有序，只需较少屏障；ARM 弱序允许更多重排，需要显式 DMB/DSB 屏障保证顺序。

**Q13: 什么是乱序执行？为什么需要 ROB？**

乱序执行允许无依赖指令提前执行。ROB（重排序缓冲区）记录指令原始顺序，确保按程序顺序提交结果和处理异常。

**Q14: PCIe 5.0 x16 的带宽是多少？**

单 Lane 约 4GB/s，x16 约 64GB/s 双向（或 128GB/s 全双工）。

**Q15: 什么是 CXL？解决什么问题？**

Compute Express Link，基于 PCIe 物理层的新协议，支持 CPU 与加速器/内存的一致性访问，解决内存扩展和异构计算的互联问题。
