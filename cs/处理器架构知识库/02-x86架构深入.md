## 1. x86 架构演进

### 1.1 x86 发展历程

| 年代 | 处理器 | 位宽 | 关键特性 |
|------|--------|------|---------|
| 1978 | 8086 | 16位 | x86 起源，实模式 |
| 1985 | 80386 | 32位 | 保护模式，虚拟内存 |
| 2003 | AMD64/x86_64 | 64位 | 长模式，更多寄存器 |
| 2008 | Nehalem | 64位 | 集成内存控制器，QPI |
| 2013 | Haswell | 64位 | AVX2, TSX |
| 2020 | Ice Lake | 64位 | AVX-512, 10nm |
| 2023 | Sapphire Rapids | 64位 | AMX (矩阵加速) |

### 1.2 SIMD 扩展演进

```
x86 SIMD 扩展：

MMX (1997)
├── 64位寄存器 (MM0-MM7)
└── 整数 SIMD

SSE (1999)
├── 128位寄存器 (XMM0-XMM7)
└── 单精度浮点 SIMD

SSE2-4 (2001-2007)
├── 双精度浮点
└── 更多指令

AVX (2011)
├── 256位寄存器 (YMM0-YMM15)
└── 三操作数指令

AVX2 (2013)
└── 整数 256位 SIMD

AVX-512 (2017)
├── 512位寄存器 (ZMM0-ZMM31)
├── 掩码寄存器 (K0-K7)
└── 更多变体 (AVX-512F, BW, DQ, VL...)

AMX (2023)
├── 矩阵寄存器 (TILE)
└── 矩阵乘法加速
```

## 2. x86_64 寄存器模型

### 2.1 通用寄存器

```
x86_64 通用寄存器 (16个64位)：

64-bit    32-bit   16-bit   8-bit(H/L)
┌─────────┬────────┬────────┬────────┐
│  RAX    │  EAX   │   AX   │ AH│AL  │  累加器
│  RBX    │  EBX   │   BX   │ BH│BL  │  基址
│  RCX    │  ECX   │   CX   │ CH│CL  │  计数器
│  RDX    │  EDX   │   DX   │ DH│DL  │  数据
├─────────┼────────┼────────┼────────┤
│  RSI    │  ESI   │   SI   │   SIL  │  源索引
│  RDI    │  EDI   │   DI   │   DIL  │  目标索引
│  RBP    │  EBP   │   BP   │   BPL  │  栈基址
│  RSP    │  ESP   │   SP   │   SPL  │  栈指针
├─────────┼────────┼────────┼────────┤
│  R8     │  R8D   │  R8W   │   R8B  │  新增
│  R9     │  R9D   │  R9W   │   R9B  │  新增
│  ...    │  ...   │  ...   │   ...  │
│  R15    │  R15D  │ R15W   │  R15B  │  新增
└─────────┴────────┴────────┴────────┘
```

### 2.2 特殊寄存器

```
特殊寄存器：

RIP     - 指令指针（64位）
RFLAGS  - 标志寄存器
          CF (进位), ZF (零), SF (符号), OF (溢出)...

段寄存器（长模式下大部分不用）：
CS, DS, ES, SS, FS, GS

控制寄存器：
CR0  - 控制保护模式、分页等
CR2  - 页错误线性地址
CR3  - 页表基地址 (PML4)
CR4  - 扩展功能启用

MSR (Model Specific Register)：
IA32_EFER    - 长模式启用
IA32_SYSENTER_CS/EIP/ESP - 系统调用
IA32_STAR/LSTAR/CSTAR    - syscall
```

### 2.3 调用约定 (System V AMD64 ABI)

```
函数调用约定（Linux/macOS）：

参数传递（整数/指针）：
  第1-6个参数：RDI, RSI, RDX, RCX, R8, R9
  更多参数：压栈（从右向左）

参数传递（浮点）：
  第1-8个参数：XMM0-XMM7

返回值：
  整数：RAX (128位用 RAX:RDX)
  浮点：XMM0 (两个浮点用 XMM0:XMM1)

调用者保存：RAX, RCX, RDX, RSI, RDI, R8-R11, XMM0-XMM15
被调用者保存：RBX, RBP, R12-R15

栈对齐：调用前 RSP 必须 16 字节对齐
```

## 3. 指令编码与寻址模式

### 3.1 x86 指令格式

```
x86 指令编码（可变长1-15字节）：

┌─────────┬────────┬────────┬─────┬─────────────┬─────────────┐
│ Prefix  │ Opcode │ ModR/M │ SIB │ Displacement│  Immediate  │
│ 0-4字节 │ 1-3字节│ 0-1字节│0-1B │   0-4字节   │   0-8字节   │
└─────────┴────────┴────────┴─────┴─────────────┴─────────────┘

Prefix 类型：
- Legacy: LOCK, REP, 段覆盖
- REX: 64位扩展 (REX.W, REX.R, REX.X, REX.B)
- VEX/EVEX: AVX 指令前缀

ModR/M 字节 [7:6][5:3][2:0]：
- Mod: 寻址模式
- Reg/Opcode: 寄存器或扩展操作码
- R/M: 寄存器或内存操作数

SIB 字节 [7:6][5:3][2:0]：
- Scale: 比例因子 (1/2/4/8)
- Index: 索引寄存器
- Base: 基址寄存器
```

### 3.2 寻址模式

```
x86 寻址模式：

1. 立即数寻址
   MOV RAX, 0x1234

2. 寄存器寻址
   MOV RAX, RBX

3. 直接内存寻址
   MOV RAX, [0x1000]

4. 寄存器间接寻址
   MOV RAX, [RBX]

5. 基址+偏移
   MOV RAX, [RBX + 8]

6. 基址+索引*比例+偏移
   MOV RAX, [RBX + RCX*8 + 16]
   常用于数组访问：arr[i] = base + i * sizeof(element)

7. RIP相对寻址（x86_64特有）
   MOV RAX, [RIP + offset]
   用于位置无关代码(PIC)
```

## 4. x86 内存模型

### 4.1 分段机制

```
实模式（16位）：
物理地址 = 段基址 × 16 + 偏移
寻址范围：1MB

保护模式/长模式：
┌─────────────────────────────────────────────────────────────┐
│              段选择子 (16位)                                │
├─────────────────────────────────────────────────────────────┤
│  [15:3] Index │ [2] TI (GDT/LDT) │ [1:0] RPL               │
└───────┬───────┴─────────┬────────┴──────────────────────────┘
        │                 │
        ▼                 ▼
   ┌─────────────────────────────────────────────────────────┐
   │  GDT/LDT (描述符表)                                     │
   ├─────────────────────────────────────────────────────────┤
   │  段描述符（8字节）：                                     │
   │  - 基地址 (32位)                                        │
   │  - 限长 (20位)                                          │
   │  - 类型、特权级、标志等                                 │
   └─────────────────────────────────────────────────────────┘

x86_64 长模式：分段几乎不用（平坦模型），CS仅用于特权级
```

### 4.2 分页机制

```
x86_64 四级页表 (4KB页)：

虚拟地址 48位：
┌─────────┬─────────┬─────────┬─────────┬────────────────┐
│ [47:39] │ [38:30] │ [29:21] │ [20:12] │    [11:0]      │
│  PML4   │  PDPT   │   PD    │   PT    │   Offset       │
│  索引   │   索引  │   索引  │   索引  │  页内偏移      │
└────┬────┴────┬────┴────┬────┴────┬────┴────────────────┘
     │         │         │         │
     ▼         ▼         ▼         ▼
   ┌───┐    ┌───┐    ┌───┐    ┌───┐    ┌─────────────┐
CR3│PML4│───►│PDPT│───►│ PD │───►│ PT │───►│Physical Page│
   └───┘    └───┘    └───┘    └───┘    └─────────────┘

页表项格式 (64位)：
[63]    NX (不可执行)
[62:52] 保留/可用
[51:12] 物理页帧号
[11:9]  可用
[8]     G (全局)
[7]     PS (大页标志)
[6]     D (脏位)
[5]     A (访问位)
[4]     PCD (禁用缓存)
[3]     PWT (写透)
[2]     U/S (用户/管理员)
[1]     R/W (读/写)
[0]     P (存在位)

大页支持：
- 2MB 大页：PD 直接映射，跳过 PT
- 1GB 大页：PDPT 直接映射，跳过 PD 和 PT
```

### 4.3 五级页表 (LA57)

```
五级页表（57位虚拟地址，Ice Lake+）：

┌─────────┬─────────┬─────────┬─────────┬─────────┬──────────┐
│ [56:48] │ [47:39] │ [38:30] │ [29:21] │ [20:12] │ [11:0]   │
│  PML5   │  PML4   │  PDPT   │   PD    │   PT    │ Offset   │
└─────────┴─────────┴─────────┴─────────┴─────────┴──────────┘

虚拟地址空间：128 PB（2^57字节）
主要用途：大内存服务器、数据库
```

## 5. 特权级与保护模式

### 5.1 特权级 Ring

```
x86 特权级：

Ring 0 ────────── 内核模式（最高权限）
   │              - 可执行所有指令
   │              - 可访问所有内存
   │
Ring 1 ────────── 设备驱动（很少使用）
Ring 2 ────────── 设备驱动（很少使用）
   │
Ring 3 ────────── 用户模式（最低权限）
                  - 受限指令集
                  - 受限内存访问

特权级检查：
- CPL (Current Privilege Level): CS 的 RPL
- DPL (Descriptor Privilege Level): 段描述符中
- RPL (Requested Privilege Level): 段选择子中

访问规则：CPL ≤ DPL 才能访问
```

### 5.2 特权指令

```
特权指令（Ring 0）：

控制寄存器操作：
  MOV CR0/CR3/CR4, reg
  LIDT, LGDT, LLDT, LTR
  
I/O 操作：
  IN, OUT (除非 IOPL 允许)
  CLI, STI (中断使能)
  
系统管理：
  HLT (停机)
  INVD, WBINVD (缓存失效)
  INVLPG (TLB 失效)
  
MSR 操作：
  RDMSR, WRMSR
  
虚拟化相关：
  VMXON, VMXOFF, VMLAUNCH, VMRESUME
```

## 6. 系统调用机制

### 6.1 三种系统调用方式

```
1. INT 0x80（传统，32位Linux）
   用户态：
     MOV EAX, syscall_number
     MOV EBX, arg1
     MOV ECX, arg2
     INT 0x80
   
   内核态：通过 IDT 查找中断处理程序

2. SYSENTER/SYSEXIT（Intel，32位）
   预设 MSR：
     IA32_SYSENTER_CS  = 内核代码段
     IA32_SYSENTER_ESP = 内核栈指针
     IA32_SYSENTER_EIP = 入口点
   
   用户态：SYSENTER
   内核态：SYSEXIT 返回

3. SYSCALL/SYSRET（AMD64，64位）★主流
   预设 MSR：
     IA32_STAR   = CS/SS 选择子
     IA32_LSTAR  = 64位入口点
     IA32_FMASK  = RFLAGS 掩码
   
   用户态：
     MOV RAX, syscall_number
     MOV RDI, arg1
     MOV RSI, arg2
     MOV RDX, arg3
     MOV R10, arg4  # 注意：不是 RCX
     MOV R8, arg5
     MOV R9, arg6
     SYSCALL
   
   内核态：SYSRET 返回
```

### 6.2 SYSCALL 执行流程

```
SYSCALL 指令行为：

1. RCX ← RIP (保存返回地址)
2. R11 ← RFLAGS (保存标志)
3. RFLAGS ← RFLAGS AND NOT(IA32_FMASK)
4. CS ← IA32_STAR[47:32] (内核CS)
5. SS ← IA32_STAR[47:32] + 8 (内核SS)
6. RIP ← IA32_LSTAR (跳转到内核入口)
7. CPL ← 0 (进入Ring 0)

SYSRET 指令行为：
1. RIP ← RCX (恢复返回地址)
2. RFLAGS ← R11 (恢复标志)
3. CS ← IA32_STAR[63:48] + 16 (用户CS)
4. SS ← IA32_STAR[63:48] + 8 (用户SS)
5. CPL ← 3 (返回Ring 3)
```

## 7. 中断与异常处理

### 7.1 中断类型

| 类型 | 来源 | 示例 |
|------|------|------|
| 硬件中断 | 外部设备 | 键盘、网卡、时钟 |
| 软件中断 | INT 指令 | INT 0x80 |
| 异常 | CPU 内部 | 除零、页错误、GP |

### 7.2 IDT（中断描述符表）

```
IDT 结构：

┌─────────────────────────────────────────────────────────────┐
│  IDT Entry (16字节，64位模式)                              │
├─────────────────────────────────────────────────────────────┤
│  [127:96] 保留                                              │
│  [95:64]  Offset[63:32] 处理程序地址高32位                  │
│  [63:48]  Offset[31:16] 处理程序地址中16位                  │
│  [47]     P (存在位)                                        │
│  [46:45]  DPL (描述符特权级)                                │
│  [44:40]  Type (0xE=中断门, 0xF=陷阱门)                     │
│  [39:35]  IST (中断栈表索引)                                │
│  [34:32]  保留                                              │
│  [31:16]  段选择子                                          │
│  [15:0]   Offset[15:0] 处理程序地址低16位                   │
└─────────────────────────────────────────────────────────────┘

常见中断向量：
  0   - #DE 除零异常
  6   - #UD 无效操作码
  13  - #GP 通用保护错误
  14  - #PF 页错误
  32+ - 外部中断 (IRQ)
```

### 7.3 APIC

```
中断控制器演进：

8259 PIC (Legacy)
└── 级联两片，共 15 个 IRQ

Local APIC (每个 CPU 核心)
├── 本地中断处理
├── 处理器间中断 (IPI)
└── 定时器

I/O APIC (系统级)
├── 处理外部设备中断
└── 中断路由到各 CPU

MSI/MSI-X (现代)
├── 设备直接写内存触发中断
├── 无需 I/O APIC 路由
└── PCIe 设备标配
```

## 8. x86 虚拟化扩展

### 8.1 VT-x 概述

```
Intel VT-x 虚拟化：

┌─────────────────────────────────────────────────────────────┐
│                    VMM (Hypervisor)                        │
│                      VMX Root Mode                          │
├─────────────────────────────────────────────────────────────┤
│  VM 1        │   VM 2        │   VM 3                      │
│  (Guest)     │   (Guest)     │   (Guest)                   │
│  VMX Non-Root│   VMX Non-Root│   VMX Non-Root              │
└─────────────────────────────────────────────────────────────┘

关键指令：
  VMXON   - 进入 VMX 操作模式
  VMXOFF  - 退出 VMX 操作模式
  VMLAUNCH- 首次进入 Guest
  VMRESUME- 再次进入 Guest
  VMREAD  - 读取 VMCS 字段
  VMWRITE - 写入 VMCS 字段
```

### 8.2 VMCS 结构

```
VMCS (Virtual Machine Control Structure)：

┌─────────────────────────────────────────────────────────────┐
│  Guest-State Area                                          │
│  ├── CR0, CR3, CR4                                         │
│  ├── RSP, RIP, RFLAGS                                      │
│  ├── 段寄存器 (CS, SS, DS, ES, FS, GS)                    │
│  ├── GDTR, IDTR, LDTR, TR                                  │
│  └── MSR (EFER, PAT, ...)                                  │
├─────────────────────────────────────────────────────────────┤
│  Host-State Area                                           │
│  ├── CR0, CR3, CR4                                         │
│  ├── RSP, RIP                                              │
│  └── 段选择子                                              │
├─────────────────────────────────────────────────────────────┤
│  VM-Execution Control                                      │
│  ├── Pin-based: 外部中断、NMI                              │
│  ├── Primary: CR 访问、I/O、MSR                            │
│  ├── Secondary: EPT、VPID、虚拟化APIC                      │
│  └── Exception Bitmap                                      │
├─────────────────────────────────────────────────────────────┤
│  VM-Exit Control                                           │
│  └── 退出时保存的状态                                      │
├─────────────────────────────────────────────────────────────┤
│  VM-Entry Control                                          │
│  └── 进入时加载的状态                                      │
├─────────────────────────────────────────────────────────────┤
│  VM-Exit Information                                       │
│  ├── Exit Reason                                           │
│  ├── Exit Qualification                                    │
│  └── Guest Linear/Physical Address                         │
└─────────────────────────────────────────────────────────────┘
```

### 8.3 EPT（扩展页表）

```
EPT (Extended Page Tables) - 硬件辅助内存虚拟化：

无 EPT（影子页表）：
  Guest VA → Guest PA → Host PA
  需要 VMM 维护影子页表，开销大

有 EPT：
  Guest VA → Guest PA (Guest 页表)
           → Host PA  (EPT，硬件遍历)

EPT 页表结构（类似普通页表）：
  PML4 → PDPT → PD → PT → Host Physical Page

EPT 违例 (EPT Violation)：
  - Guest 访问未映射的 GPA
  - 权限检查失败
  触发 VM Exit，VMM 处理
```

## 9. 性能监控 PMU

### 9.1 PMU 概述

```
Performance Monitoring Unit (PMU)：

┌─────────────────────────────────────────────────────────────┐
│  PMC (Performance Monitoring Counter)                      │
│  ├── 固定计数器 (Fixed Counter)                            │
│  │   - 指令数 (Instructions Retired)                       │
│  │   - 核心周期 (Core Cycles)                              │
│  │   - 参考周期 (Reference Cycles)                         │
│  │                                                         │
│  └── 通用计数器 (General Purpose)                          │
│      - 可编程计数任意事件                                  │
│      - 数量因型号而异 (4-8个)                              │
└─────────────────────────────────────────────────────────────┘

常用性能事件：
  INST_RETIRED.ANY          - 指令数
  CPU_CLK_UNHALTED.THREAD   - CPU 周期
  BR_MISP_RETIRED.ALL_BRANCHES - 分支预测失败
  MEM_LOAD_RETIRED.L3_MISS  - L3 缓存缺失
  DTLB_LOAD_MISSES.MISS_CAUSES_A_WALK - DTLB 缺失
```

### 9.2 使用 PMU

```bash
# Linux perf 工具

# 查看可用事件
perf list

# 统计程序性能
perf stat ./program
# 输出：cycles, instructions, IPC, cache-misses...

# 采样分析
perf record -g ./program
perf report

# 实时监控
perf top

# 指定事件
perf stat -e cycles,instructions,cache-misses ./program
```

## 10. 面试题自查

**Q1: x86_64 有多少个通用寄存器？调用约定中参数如何传递？**

16个64位通用寄存器（RAX-RDX, RSI, RDI, RBP, RSP, R8-R15）。System V AMD64 ABI 中，前6个整数参数通过 RDI, RSI, RDX, RCX, R8, R9 传递，返回值在 RAX。

**Q2: x86 分段和分页的区别？x86_64 长模式下分段还有用吗？**

分段提供逻辑地址到线性地址的转换；分页提供线性地址到物理地址的转换。长模式下使用平坦模型，分段基本不用，但 CS 段仍用于特权级（CPL）。

**Q3: 解释 x86_64 的四级页表结构。**

48位虚拟地址分为：PML4索引[47:39] + PDPT索引[38:30] + PD索引[29:21] + PT索引[20:12] + 页内偏移[11:0]。CR3 指向 PML4 基地址，每级页表项8字节，指向下一级或物理页帧。

**Q4: SYSCALL 和 INT 0x80 的区别？为什么 SYSCALL 更快？**

INT 0x80 通过中断描述符表（IDT）进入内核，需要查表和栈切换；SYSCALL 直接从 MSR 读取目标地址，无需查表，且用寄存器保存返回地址（RCX）和标志（R11），更快。

**Q5: 什么是 Ring 0-3？用户程序运行在哪个 Ring？**

x86 四个特权级，Ring 0 最高（内核），Ring 3 最低（用户）。用户程序运行在 Ring 3，只能执行非特权指令，通过系统调用进入 Ring 0 执行内核代码。

**Q6: VT-x 的 VMCS 有哪些主要区域？**

Guest-State（Guest 寄存器状态）、Host-State（Host 寄存器状态）、VM-Execution Control（执行控制，如 EPT 启用）、VM-Exit Control、VM-Entry Control、VM-Exit Information（退出原因和信息）。

**Q7: 什么是 EPT？解决了什么问题？**

Extended Page Tables，硬件辅助的 Guest 物理地址到 Host 物理地址转换。解决了影子页表的高开销问题，Guest 页表修改无需 VM Exit，由硬件自动完成两级地址转换。

**Q8: x86 的内存序模型是什么？和 ARM 有什么区别？**

x86 使用 TSO（Total Store Order），保证 Store-Store 和 Load-Load 有序，只有 Store-Load 可能重排。ARM 使用弱内存序，几乎所有操作都可能重排，需要更多内存屏障。

**Q9: AVX-512 相比 AVX2 有什么改进？**

寄存器从 256位扩展到 512位（ZMM），寄存器数量从 16 增加到 32，新增掩码寄存器（K0-K7）支持谓词执行，更多指令变体。

**Q10: 什么是 MSI/MSI-X 中断？相比传统中断有什么优势？**

Message Signaled Interrupt，设备通过写特定内存地址触发中断，无需通过 I/O APIC 路由。优势：无共享中断线、更多中断向量、更低延迟、支持多队列。

**Q11: 页错误（#PF）异常时，CPU 会保存哪些信息？**

CR2 保存引发页错误的线性地址；错误码压栈，包含 P（页存在）、W/R（写/读）、U/S（用户/内核）、RSVD（保留位）、I/D（取指/数据）等标志。

**Q12: 什么是 PCID？有什么作用？**

Process-Context Identifier，CR3 低12位可存储 PCID（需 CR4.PCIDE=1）。不同进程使用不同 PCID，TLB 项带 PCID 标记，进程切换时无需刷新 TLB，提高性能。

**Q13: PMU 可以监控哪些性能指标？**

指令数、CPU 周期、IPC、分支预测失败、Cache 各级缺失、TLB 缺失、内存带宽等。通过 perf 工具或直接读取 MSR 获取。

**Q14: x86 指令编码为什么是可变长的？有什么优缺点？**

历史原因（向后兼容 8086），常用指令编码短，提高代码密度。缺点：解码复杂，需要多周期或微码辅助，影响流水线效率。

**Q15: 什么是 NX 位？有什么安全作用？**

No-Execute 位（页表项第63位），标记页面不可执行。防止代码注入攻击，栈和堆数据页设置 NX 后无法被执行，是 DEP（数据执行保护）的硬件基础。
