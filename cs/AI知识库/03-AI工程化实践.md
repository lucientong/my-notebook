## 开始前先看

### 为什么 AI 应用一定会进入工程化阶段

一个 demo 能跑，不代表系统能上线。AI 应用天然会遇到下面这些问题：

- `效果不稳定`：同一个问题，不同时间回答质量波动
- `成本不可控`：上下文一长，token 成本快速上升
- `延迟偏高`：工具调用、检索、模型生成串起来后响应变慢
- `安全风险`：提示注入、敏感信息泄露、工具越权调用
- `难以排障`：出了问题，很难判断是 Prompt、知识库、工具还是模型导致

工程化不是“模型做好以后再补的部分”，而是 AI 应用从第一天就要考虑的主线。

### 这篇的阅读目标

读完后你应该能回答：

- 一个 AI 应用上线前，除了模型本身还要设计什么？
- 为什么要做模型路由、缓存、限流、降级、评估和 tracing？
- AI 系统出问题时，应该怎么拆链路定位？

## 1. 先建立系统视角

### 1.0 一个最小 AI 系统长什么样

```text
用户请求
  ↓
API / 业务入口
  ↓
模型路由
  ↓
上下文构造
  ├─ Prompt
  ├─ RAG
  ├─ 用户上下文
  └─ 历史记忆
  ↓
模型调用
  ↓
工具调用 / 数据查询
  ↓
输出解析与安全过滤
  ↓
日志、指标、评估、反馈闭环
```

很多线上问题并不是“模型不够强”，而是上面某一层设计失衡。

### 1.0.1 先区分 4 类常见故障

| 现象 | 更可能的问题 | 常见修正方向 |
|------|-------------|-------------|
| 回答空泛、跑题 | Prompt / 上下文构造不好 | 重写提示、补结构化约束 |
| 回答像真的但事实不对 | 知识缺失 / 幻觉 | 上 RAG、加引用、加校验 |
| 回答慢且贵 | 路由、缓存、上下文过长 | 模型分级、缓存、裁剪 token |
| 工具调用失败或越权 | schema / 权限 / 重试策略不当 | 收紧权限、补参数校验、加降级 |

### 1.0.2 工程化的核心目标

AI 工程化主要不是追求“最先进”，而是追求 5 件事：

1. `可用`：大多数请求都能稳定返回。
2. `可控`：成本、延迟、权限、输出格式可控制。
3. `可评估`：能判断系统到底有没有变好。
4. `可排障`：坏结果能追到具体环节。
5. `可演进`：模型、Prompt、工具、知识库能持续灰度更新。

在这个视角下，后面的章节就不再是孤立知识点。

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import openai
import asyncio

app = FastAPI()

class LLMRequest(BaseModel):
    query: str
    user_id: str
    model: str = "gpt-4"

class LLMService:
    def __init__(self):
        self.router = ModelRouter()
        self.cache = CacheLayer()
        self.rate_limiter = RateLimiter()
        self.monitor = Monitor()
    
    async def generate(self, request: LLMRequest) -> str:
        """生成回答"""
        # 1. 监控：记录请求开始
        span = self.monitor.start_span("llm_generate")
        
        try:
            # 2. 策略：限流检查
            if not await self.rate_limiter.allow(request.user_id):
                raise HTTPException(429, "请求过于频繁")
            
            # 3. 策略：缓存检查
            cache_key = f"{request.model}:{request.query}"
            cached = await self.cache.get(cache_key)
            if cached:
                self.monitor.record_cache_hit()
                return cached
            
            # 4. 路由：选择模型
            model = self.router.select_model(request)
            
            # 5. LLM：生成回答
            response = await self.call_llm(model, request.query)
            
            # 6. 策略：写入缓存
            await self.cache.set(cache_key, response, ttl=3600)
            
            # 7. 监控：记录成功
            self.monitor.record_success(span)
            
            return response
            
        except Exception as e:
            # 8. 监控：记录失败
            self.monitor.record_error(span, e)
            
            # 9. 策略：降级处理
            return await self.fallback(request.query)
    
    async def call_llm(self, model: str, query: str) -> str:
        """调用LLM"""
        response = await openai.ChatCompletion.acreate(
            model=model,
            messages=[{"role": "user", "content": query}],
            temperature=0.7
        )
        return response.choices[0].message.content
    
    async def fallback(self, query: str) -> str:
        """降级处理"""
        return "服务暂时不可用，请稍后再试。"

# API端点
service = LLMService()

@app.post("/chat")
async def chat(request: LLMRequest):
    response = await service.generate(request)
    return {"response": response}
```

### 1.2 模型路由

**策略**：
```python
class ModelRouter:
    def __init__(self):
        self.models = {
            "simple": "gpt-3.5-turbo",    # 简单问答
            "complex": "gpt-4",            # 复杂推理
            "code": "claude-3-opus",       # 代码生成
            "long_context": "claude-3"     # 长文本
        }
    
    def select_model(self, request: LLMRequest) -> str:
        """根据请求选择模型"""
        query = request.query
        
        # 1. 用户指定模型
        if request.model:
            return request.model
        
        # 2. 代码相关 → Claude
        if any(kw in query.lower() for kw in ["代码", "code", "编程", "bug"]):
            return self.models["code"]
        
        # 3. 长文本 → Claude (200K上下文)
        if len(query) > 5000:
            return self.models["long_context"]
        
        # 4. 复杂推理 → GPT-4
        if self.is_complex(query):
            return self.models["complex"]
        
        # 5. 默认 → GPT-3.5 (便宜快速)
        return self.models["simple"]
    
    def is_complex(self, query: str) -> bool:
        """判断是否复杂问题"""
        # 启发式规则
        complex_keywords = ["分析", "推理", "设计", "规划", "比较"]
        return any(kw in query for kw in complex_keywords)
```

### 1.3 灰度发布

```python
class GrayReleaseRouter:
    def __init__(self):
        self.config = {
            "model_a": "gpt-4-0613",  # 旧版本
            "model_b": "gpt-4-1106",  # 新版本
            "gray_percentage": 10      # 10%流量到新版本
        }
    
    def select_model(self, user_id: str) -> str:
        """灰度路由"""
        # 根据user_id哈希决定路由
        hash_value = int(hashlib.md5(user_id.encode()).hexdigest(), 16)
        percentage = hash_value % 100
        
        if percentage < self.config["gray_percentage"]:
            return self.config["model_b"]  # 新版本
        else:
            return self.config["model_a"]  # 旧版本
```

```python
class PromptManager:
    def __init__(self, base_dir="prompts"):
        self.base_dir = base_dir
        self.cache = {}
    
    def get_prompt(self, name: str, version: str = "current") -> str:
        """获取Prompt模板"""
        cache_key = f"{version}:{name}"
        
        if cache_key in self.cache:
            return self.cache[cache_key]
        
        # 读取文件
        path = os.path.join(self.base_dir, version, f"{name}.txt")
        with open(path, 'r', encoding='utf-8') as f:
            template = f.read()
        
        self.cache[cache_key] = template
        return template
    
    def render(self, name: str, **kwargs) -> str:
        """渲染Prompt"""
        template = self.get_prompt(name)
        return template.format(**kwargs)

# 使用
pm = PromptManager()
prompt = pm.render(
    "code_review",
    language="Python",
    code="def add(a, b): return a + b"
)
```

```python
class PromptABTest:
    def __init__(self, llm):
        self.llm = llm
        self.results = {"A": [], "B": []}
    
    async def run_test(self, query: str, prompt_a: str, prompt_b: str, n=10):
        """运行A/B测试"""
        for i in range(n):
            # 测试A
            response_a = await self.llm.generate(prompt_a.format(query=query))
            score_a = await self.evaluate(response_a, query)

            self.results["B"].append(score_b)
        
        # 统计结果
        avg_a = sum(self.results["A"]) / len(self.results["A"])
        avg_b = sum(self.results["B"]) / len(self.results["B"])
        
        print(f"Prompt A平均分：{avg_a:.2f}")
        print(f"Prompt B平均分：{avg_b:.2f}")
        print(f"B相比A提升：{(avg_b - avg_a) / avg_a * 100:.1f}%")
    
    async def evaluate(self, response: str, query: str) -> float:

        """
```

## 3. 评估体系

### 3.1 评估指标

**分类**：
| 类别 | 指标 | 描述 | 计算方法 |
|------|------|------|----------|
| **准确性** | Accuracy | 答案正确率 | 正确数 / 总数 |
| **相关性** | Relevance | 与问题的相关程度 | 人工评分 / 自动评分 |
| **完整性** | Completeness | 信息覆盖度 | 关键点覆盖率 |
| **幻觉率** | Hallucination Rate | 编造信息的比例 | 幻觉数 / 总回答数 |
| **文本质量** | BLEU/ROUGE | 与参考答案的相似度 | N-gram重叠 |
| **语义相似度** | Embedding Similarity | 语义层面相似度 | Cosine相似度 |

**Python实现**：
```python
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

class LLMEvaluator:
    def __init__(self, embedding_model):
        self.embedding_model = embedding_model
    
    def evaluate_accuracy(self, predictions, references):
        """准确性"""
        correct = sum(p == r for p, r in zip(predictions, references))
        return correct / len(predictions)
    
    def evaluate_relevance(self, response: str, query: str) -> float:
        """相关性（基于Embedding）"""
        # 向量化
        query_emb = self.embedding_model.encode(query)
        response_emb = self.embedding_model.encode(response)
        
        # Cosine相似度
        similarity = cosine_similarity(
            [query_emb], [response_emb]
        )[0][0]
        
        return similarity
    
    def evaluate_hallucination(self, response: str, context: str) -> bool:
        """幻觉检测"""
        # 简单策略：检查回答中的事实是否出现在上下文中
        facts = self.extract_facts(response)
        
        for fact in facts:
            if fact not in context:
                return True  # 存在幻觉
        
        return False
    
    def evaluate_bleu(self, prediction: str, reference: str) -> float:
        """BLEU分数"""
        from nltk.translate.bleu_score import sentence_bleu
        
        pred_tokens = prediction.split()
        ref_tokens = [reference.split()]
        
        return sentence_bleu(ref_tokens, pred_tokens)
    
    def evaluate_rouge(self, prediction: str, reference: str) -> dict:
        """ROUGE分数"""
        from rouge import Rouge
        
        rouge = Rouge()
        scores = rouge.get_scores(prediction, reference)[0]
        
        return {
            "rouge-1": scores["rouge-1"]["f"],
            "rouge-2": scores["rouge-2"]["f"],
            "rouge-l": scores["rouge-l"]["f"]
        }
```

```python
class CostAwareRouter:
    def __init__(self):
        self.models = {
            "cheap": {
                "name": "gpt-3.5-turbo",
                "input_cost": 0.001,
                "output_cost": 0.002
            },
            "balanced": {
                "name": "gpt-4-turbo",
                "input_cost": 0.01,
                "output_cost": 0.03
            },
            "premium": {
                "name": "gpt-4",
                "input_cost": 0.03,
                "output_cost": 0.06
            }
        }
    
    def select_model(self, query: str, max_cost: float = None) -> str:
        """根据预算选择模型"""
        # 估算token数
        estimated_tokens = len(query) / 4 * 1.5  # 输入+输出
        
        # 计算成本
        for tier, model in self.models.items():
            cost = estimated_tokens * (model["input_cost"] + model["output_cost"]) / 1000
            
            if max_cost is None or cost <= max_cost:
                return model["name"]
        
        # 超预算，使用最便宜的模型
        return self.models["cheap"]["name"]
```

### 4.2 缓存策略

**多级缓存**：
```python
class MultiLevelCache:
    def __init__(self):
        self.l1_cache = {}  # 内存缓存（小且热）
        self.l2_cache = redis.Redis()  # Redis缓存（中等）
        self.l3_cache = {}  # 数据库缓存（大且冷）
    
    async def get(self, key: str) -> str:
        """获取缓存"""
        # L1: 内存
        if key in self.l1_cache:
            return self.l1_cache[key]
        
        # L2: Redis
        value = await self.l2_cache.get(key)
        if value:
            self.l1_cache[key] = value  # 回写L1
            return value
        
        # L3: 数据库
        value = await self.db.get(key)
        if value:
            await self.l2_cache.set(key, value, ex=3600)  # 回写L2
            self.l1_cache[key] = value  # 回写L1
            return value
        
        return None
    
    async def set(self, key: str, value: str, ttl: int = 3600):
        """设置缓存"""
        # 写入所有层级
        self.l1_cache[key] = value
        await self.l2_cache.set(key, value, ex=ttl)
        await self.db.set(key, value)
```

```python
class SemanticCache:
    def __init__(self, embedding_model, threshold=0.95):
        self.embedding_model = embedding_model
        self.threshold = threshold
        self.cache = []  # [(embedding, response)]
    
    async def get(self, query: str) -> str:
        """基于语义相似度的缓存"""
        query_emb = self.embedding_model.encode(query)
        
        for cached_emb, response in self.cache:
            similarity = cosine_similarity([query_emb], [cached_emb])[0][0]
            
            if similarity >= self.threshold:
                return response  # 缓存命中
        
        return None
    
    async def set(self, query: str, response: str):
        """添加到缓存"""
        query_emb = self.embedding_model.encode(query)
        self.cache.append((query_emb, response))
        
        # 限制缓存大小
        if len(self.cache) > 1000:
            self.cache.pop(0)
```

```python
class PredictiveLoader:
    def __init__(self):
        self.patterns = {}
    
    def record(self, query: str, next_query: str):
        """记录查询模式"""
        if query not in self.patterns:
            self.patterns[query] = {}
        
        if next_query not in self.patterns[query]:
            self.patterns[query][next_query] = 0
        
        self.patterns[query][next_query] += 1
    
    def predict(self, query: str) -> str:
        """预测下一个查询"""
        if query in self.patterns:
            return max(self.patterns[query], key=self.patterns[query].get)
        return None
    
    async def preload(self, query: str):
        """预加载可能的下一个查询"""
        next_query = self.predict(query)
        if next_query:
            # 后台预生成
            asyncio.create_task(llm.generate(next_query))
```

```python
class PromptInjectionDetector:
    def __init__(self):
        self.dangerous_patterns = [
            r"ignore previous instructions",
            r"忽略之前的指令",
            r"system:",
            r"<script>",
            r"rm -rf",
            r"DROP TABLE"
        ]
    
    def detect(self, query: str) -> bool:
        """检测Prompt注入"""
        for pattern in self.dangerous_patterns:
            if re.search(pattern, query, re.IGNORECASE):
                return True
        return False
    
    def sanitize(self, query: str) -> str:
        """清理输入"""
        # 移除特殊字符
        sanitized = re.sub(r'[<>{}]', '', query)
        return sanitized
```

### 9.1 LangChain

**核心概念**：
- **Chain**：串联多个处理步骤

- 快速原型开发
- RAG应用
- 简单的Agent场景

### 9.2 LangGraph

**核心概念**：
- **State**：全局状态
- **Node**：处理节点
- **Edge**：条件边（决定流转）

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict

# 定义状态
class AgentState(TypedDict):
    messages: list
    next_action: str

# 创建图
workflow = StateGraph(AgentState)

# 添加节点
workflow.add_node("analyze", analyze_node)
workflow.add_node("search", search_node)
workflow.add_node("generate", generate_node)

# 添加边
workflow.add_edge("analyze", "search")
workflow.add_conditional_edges(
    "search",
    should_continue,  # 条件函数
    {"continue": "search", "generate": "generate"}
)
workflow.add_edge("generate", END)

# 编译运行
app = workflow.compile()
result = app.invoke({"messages": ["用户问题"], "next_action": ""})
```

**适用场景**：
- 复杂的多步骤Agent
- 需要条件分支的流程
- 状态管理复杂的应用

### 9.3 框架对比

| 特性 | LangChain | LangGraph | 直接调用API |
|------|-----------|-----------|-------------|
| 复杂度 | 中等 | 高 | 低 |
| 灵活性 | 一般 | 高 | 最高 |
| 学习曲线 | 中等 | 陡峭 | 平缓 |
| 适用场景 | 标准RAG、简单Agent | 复杂工作流 | 简单应用 |
| 可调试性 | 一般 | 好 | 最好 |
| 生产就绪 | 是 | 是 | 是 |

**选型建议**：
- 简单应用/学习 → 直接调用API
- 标准RAG → LangChain
- 复杂Agent/工作流 → LangGraph
- 极致性能要求 → 自研框架

这里重点不是“模型答得像不像”，而是“这套应用上线后能不能持续稳定地产生业务价值”。因此评估必须覆盖质量、系统和成本三层。

**评估指标**：
| 指标 | 计算方法 | 适用场景 |
|------|----------|----------|
| 准确性 | 正确数/总数 | 有标准答案 |
| 相关性 | Embedding相似度或 Judge 评分 | 检索、对话 |
| 幻觉率 | 幻觉数/总数 | 事实类问答 |
| 任务完成率 | 完成任务数/总任务数 | Agent、工作流 |
| 延迟与失败率 | P95/P99、错误数/总数 | 线上服务 |
| 单请求成本 | token/工具/缓存成本 | 生产运营 |
| 用户满意度 | 人工评分/行为反馈 | 产品体验 |

**评估方法**：
1. **自动化基准测试**：标准数据集 + 自动评分，用于日常回归。
2. **LLM-as-Judge + 人工抽检**：提升覆盖面，同时控制评审漂移。
3. **线上 A/B 测试**：验证真实用户价值，而不是只看离线分数。
4. **版本化追踪**：按 Prompt、模型、检索版本分别看效果，避免问题无法归因。

如果你只回答准确率而不回答延迟、成本和失败率，这题在工程场景里通常不算回答完整。

### Q6: 在生产环境里，Prompt 注入防护为什么不能只靠 Prompt？如何做多层防线？

**答案**：
如果基础文档讲的是“Prompt Injection 是什么”，工程文档就要回答“为什么线上不能只靠一句系统提示挡住它”。核心原因是：攻击入口不只来自用户输入，还可能来自网页抓取结果、RAG 文档、工具返回值和多 Agent 中间消息，所以必须把防护做成系统能力。

1. **输入层**：识别高风险模式（越权、绕过系统指令、诱导泄露），命中后进入拦截或人工复核。
2. **上下文层**：系统指令、业务规则、用户输入严格分区，禁止用户内容覆盖系统策略。
3. **执行层**：工具按最小权限开放，写操作强制二次确认，外部调用设置超时与配额。
4. **输出层**：对敏感字段、越权结果做脱敏与阻断，保留审计日志。

**落地要点**：
- 把注入防护接入网关和编排层，不依赖某个 Prompt 模板。
- 建立红队用例集（越权提权、数据外带、策略绕过）并纳入回归测试。
- 给每次拦截打标签，持续迭代规则，避免“上线后再补漏洞”。
- 把拦截命中率、误杀率、越权调用数纳入安全看板，而不是只在事故后回看日志。

### Q8: 语义缓存是什么？如何实现？

**答案**：
**语义缓存**：基于查询语义相似度的缓存，相似查询可复用结果。

**实现**：
```python
class SemanticCache:
    def __init__(self, threshold=0.95):
        self.cache = []  # [(embedding, response)]
        self.threshold = threshold
    
    def get(self, query):
        query_emb = embed(query)
        
        for cached_emb, response in self.cache:
            similarity = cosine_similarity(query_emb, cached_emb)
            if similarity >= self.threshold:
                return response  # 命中
        
        return None
```

**优势**：
- "什么是AI"和"AI是什么"可以命中同一缓存
- 大幅提高缓存命中率

**注意**：
- 阈值太低会导致错误命中
- 需要高质量Embedding模型
- 定期清理过期缓存

### Q11: 如何实现LLM-as-Judge自动评估？

```python
async def llm_judge(query: str, response: str) -> dict:
```

### Q13: 如何进行LLM应用的AB测试？

**答案**：

**测试对象**：
- 不同Prompt版本
- 不同模型
- 不同参数（temperature等）

**实现**：
```python
class ABTest:
    def __init__(self, variants, metric_fn):
        self.variants = variants  # {"A": prompt_a, "B": prompt_b}
        self.metric_fn = metric_fn
        self.results = {"A": [], "B": []}
    
    async def run(self, queries, n_per_query=5):
        for query in queries:
            for variant, prompt in self.variants.items():
                for _ in range(n_per_query):
                    response = await llm.generate(prompt.format(query=query))
                    score = await self.metric_fn(query, response)
                    self.results[variant].append(score)
        
        # 统计显著性
        return self.analyze()
```

**分析**：
- 计算各变体平均分
- t检验判断显著性
- 样本量要足够（至少30次/变体）

### Q15: 在生产环境里，如何把幻觉治理做成可观测、可发布、可回滚的工程闭环？

**答案**：
基础文档里可以回答“RAG、工具、Prompt 约束能减幻觉”，但到了工程落地，关键不是技巧清单，而是能不能把幻觉治理纳入日常发布和质量体系。

1. **定义可观测口径**：把幻觉拆成可测指标（事实错误率、无依据断言率、引用缺失率），按场景分别统计。
2. **前置约束**：检索增强、引用必填、不可回答时标准拒答模板，减少模型自由发挥空间。
3. **后置校验**：关键场景加规则校验或二次验证模型，对数字、时间、实体名做一致性检查。
4. **发布治理**：把“幻觉率上限”写入灰度门禁，超阈值自动回滚，不让问题版本扩散。
5. **反馈闭环**：线上 badcase 自动入库，定期回放评测集，驱动 Prompt/检索/模型联动迭代。

一句话总结：这里关注的是“长期可控地治理幻觉”，而不是只列基础缓解手段。

### Q16: LLM 服务雪崩时，降级顺序应该怎么定？

**答案**：

关键是提前定义分级开关和回滚条件，避免事故中临场拍脑袋。

### Q18: 如何定义 LLM 应用的 SLO，避免只看接口 200 成功率？

### Q19: 向量检索上线后效果波动大，先调模型还是先查数据？

**答案**：
先查数据，再调模型。多数波动来自数据链路：切分策略变化、索引过期、脏数据混入、元数据过滤失效。建议先做：
1. 检索样本回放（query -> topK）对比。
2. 索引新鲜度和覆盖率检查。
3. chunk 与 embedding 版本一致性校验。

数据闭环稳定后，再进行模型和 rerank 调优。

### Q20: AI 工程里如何设计“可解释的例外发布策略”？

**答案**：
当业务强推上线但风险偏高时，可解释策略应包含：
1. 例外范围：仅限必要能力和人群。
2. 风险补偿：更严格灰度门禁和自动回滚阈值。
3. 审批留痕：记录责任人、风险说明、复盘项。
4. 退出机制：例外发布结束后恢复常规门禁。

例外策略的目标是“可追溯、可审计、可回退”，而不是特殊情况下绕流程。
