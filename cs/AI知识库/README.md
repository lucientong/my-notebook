# AI知识库导航

> 这部分知识库原本更偏“直接上手做 LLM 应用”，对已经做过项目的人有用，但对基础不牢的人不够友好。这个导航页的目标是补上进入路径，把“概念理解”和“工程落地”拆开，并给出难度递增顺序。

---

## 这套知识库适合谁

- 想系统理解 LLM / Agent / MCP，但目前只接触过 API 调用和简单 demo
- 已经做过一点 AI 应用开发，但对模型、训练、评估、系统架构缺乏整体认知
- 需要把“会调接口”升级为“知道系统为什么这样设计、出了问题该怎么拆”

---

## 先解决一个问题：AI 知识库到底在学什么

很多人刚接触 AI 会把下面几件事混在一起：

- `模型原理`：LLM 为什么能生成文本，Transformer、Token、上下文窗口、训练目标是什么
- `应用开发`：怎么写 Prompt、怎么做 RAG、怎么调用工具、怎么做 Agent
- `系统工程`：怎么控成本、控延迟、做评估、做观测、做权限隔离
- `模型训练`：怎么微调、对齐、量化、部署

这四层不是一回事。建议按下面顺序理解：

1. 先知道模型是什么，回答“它为什么能工作”。
2. 再学应用编排，回答“怎么把模型接到真实业务里”。
3. 再学工程化，回答“怎么把 demo 变成稳定系统”。
4. 最后再进入训练优化，回答“怎么改模型本身，而不只是调用模型”。

## 难度分层

### L1 入门层：先看懂关键概念，不追求细节

目标：能用自己的话解释这些概念，不再只会背术语。

- `Token` 是什么，为什么不是“一个字等于一个 token”
- `Transformer` 干了什么，注意力机制解决了什么问题
- `上下文窗口`、`KV Cache`、`温度`、`top-p` 分别影响什么
- `Prompt`、`Tool Calling`、`Agent`、`RAG` 的边界是什么
- `微调` 和 `RAG` 的区别是什么

建议阅读：

- [01-LLM与Agent基础.md](./01-LLM与Agent基础.md)
- [03-AI工程化实践.md](./03-AI工程化实践.md) 中的“LLM应用架构 / 评估体系”

### L2 进阶层：理解一套 AI 应用如何落地

目标：知道一个 AI 系统不是“调一次 API”这么简单，而是完整链路工程。

- 请求如何进入模型路由层
- 什么时候查缓存，什么时候降级
- 工具调用如何做 schema 和权限控制
- RAG 为什么会召回错、引用错、答非所问
- 评估为什么必须拆成离线评估、在线指标、人工抽检

建议阅读：

- [03-AI工程化实践.md](./03-AI工程化实践.md)
- [02-MCP与工具集成.md](./02-MCP与工具集成.md)

### L3 深水区：开始碰训练、部署、性能与算法

目标：能看懂训练与部署链路，知道常见技术的适用边界。

- LoRA / QLoRA 适合什么场景
- RLHF、PPO、DPO 分别解决什么问题
- 量化为什么能降成本，代价是什么
- 分布式训练为什么需要数据并行、模型并行、ZeRO
- 推荐模型和 LLM 在训练范式上有什么相同和不同

建议阅读：

- [04-AI模型训练与优化.md](./04-AI模型训练与优化.md)
- [05-推荐模型与训练.md](./05-推荐模型与训练.md)

## 阅读时最容易踩的坑

- 把 `Agent` 理解成“更聪明的模型”。实际上 Agent 更像“带规划、记忆、工具和流程控制的系统”。
- 把 `RAG` 当成万能解法。RAG 只能缓解知识缺失，不能自动解决推理和事实校验问题。
- 把 `MCP` 当成某个产品私有能力。它本质是工具和上下文集成协议，不等于某个具体客户端。
- 一上来就研究 LoRA、量化、蒸馏，但连基础推理链路都没搭清楚。这会导致知识结构倒挂。
- 看到工程文档就直接照搬代码，而不先搞清楚流量、成本、权限、失败兜底这些约束。

## 读完后的检查标准

如果下面这些问题你能讲清楚，说明这个知识库已经不再“难以下手”：

- 为什么 LLM 适合做生成，但不天然擅长长期精确记忆？
- Prompt、RAG、Tool Calling、Agent 分别解决什么问题？
- 一个 AI 应用为什么需要模型路由、缓存、限流、评估和可观测性？
- 微调适合什么时候做，什么时候不该做？
- MCP 给 AI 系统带来的核心价值是什么？

## 面试题风格约定

为保证跨文档可读性与可检索性，AI 知识库面试题部分统一采用：

- 标题格式：`### Qx: 问题`
- 回答起始：`**答案**：`
- 分隔样式：题目之间统一使用 `---`
- 内容定位：基础篇偏原理解释，工程篇偏治理与发布，专题篇偏场景决策

如果不同文档出现相同主题（例如幻觉、注入防护、评估），优先做“视角区分”而不是机械删除：

- `01-LLM与Agent基础`：解释机制与原理
- `03-AI工程化实践`：强调可观测、门禁、回滚和线上闭环
- 其他专题篇：强调该专题下的落地策略
