## 模型训练基础

### 训练流程

**标准训练流程**：

```python
# 1. 准备数据
train_dataloader = DataLoader(train_dataset, batch_size=32, shuffle=True)

# 2. 初始化模型
model = GPT2LMHeadModel.from_pretrained('gpt2')
model.to(device)

# 3. 配置优化器
optimizer = AdamW(model.parameters(), lr=5e-5, weight_decay=0.01)
scheduler = get_linear_schedule_with_warmup(
    optimizer, 
    num_warmup_steps=100, 
    num_training_steps=len(train_dataloader) * epochs
)

# 4. 训练循环
for epoch in range(epochs):
    model.train()
    for batch in train_dataloader:
        # Forward pass
        outputs = model(**batch)
        loss = outputs.loss
        
        # Backward pass
        loss.backward()
        
        # 梯度裁剪（防止梯度爆炸）
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        
        # 参数更新
        optimizer.step()
        scheduler.step()
        optimizer.zero_grad()
    
    # 验证
    model.eval()
    with torch.no_grad():
        for batch in val_dataloader:
            outputs = model(**batch)
            # 计算验证指标
```

### 损失函数

**语言模型的损失函数**：

```python
# Cross-Entropy Loss（交叉熵损失）
# 目标：预测下一个token

# 示例：预测 "The cat sits on the mat"
# 输入：["The", "cat", "sits", "on", "the"]
# 标签：["cat", "sits", "on", "the", "mat"]

def compute_loss(logits, labels):
    """
    logits: (batch_size, seq_len, vocab_size)
    labels: (batch_size, seq_len)
    """
    # Shift：预测下一个token
    shift_logits = logits[..., :-1, :].contiguous()  # (batch, seq-1, vocab)
    shift_labels = labels[..., 1:].contiguous()      # (batch, seq-1)
    
    # 计算交叉熵
    loss_fct = nn.CrossEntropyLoss(ignore_index=-100)
    loss = loss_fct(
        shift_logits.view(-1, shift_logits.size(-1)),  # (batch*(seq-1), vocab)
        shift_labels.view(-1)                           # (batch*(seq-1))
    )
    return loss

# 困惑度（Perplexity）：评估语言模型性能
perplexity = torch.exp(loss)
# 困惑度越低，模型越好（理想值接近1）
```

### 优化器

**常用优化器对比**：

| 优化器 | 特点 | 适用场景 |
|--------|------|----------|
| **SGD** | 简单、内存占用小 | 小模型 |
| **Adam** | 自适应学习率、常用 | 大部分场景 |
| **AdamW** | Adam + 权重衰减 | Transformer模型（推荐） |
| **Adafactor** | 内存高效（不存储一阶动量） | 超大模型 |

**AdamW实现**：

```python
class AdamW(Optimizer):
    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999), eps=1e-8, weight_decay=0.01):
        self.lr = lr
        self.beta1, self.beta2 = betas
        self.eps = eps
        self.weight_decay = weight_decay
        
        # 一阶动量（梯度的指数移动平均）
        self.m = [torch.zeros_like(p) for p in params]
        # 二阶动量（梯度平方的指数移动平均）
        self.v = [torch.zeros_like(p) for p in params]
        self.t = 0
    
    def step(self, grads):
        self.t += 1
        for i, (param, grad) in enumerate(zip(self.params, grads)):
            # 更新一阶动量
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * grad
            # 更新二阶动量
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * grad ** 2
            
            # 偏差修正
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            v_hat = self.v[i] / (1 - self.beta2 ** self.t)
            
            # 参数更新
            param -= self.lr * m_hat / (torch.sqrt(v_hat) + self.eps)
            
            # 权重衰减（与L2正则化不同，直接衰减参数）
            param -= self.lr * self.weight_decay * param
```

### 学习率调度

**常用调度策略**：

```python
# 1. 线性预热 + 线性衰减
from transformers import get_linear_schedule_with_warmup

scheduler = get_linear_schedule_with_warmup(
    optimizer,
    num_warmup_steps=1000,   # 预热步数
    num_training_steps=10000  # 总训练步数
)

# 学习率变化：
# 0 → 1000步：线性增长到 lr
# 1000 → 10000步：线性衰减到 0

# 2. 余弦退火
from torch.optim.lr_scheduler import CosineAnnealingLR

scheduler = CosineAnnealingLR(optimizer, T_max=10000, eta_min=1e-6)

# 学习率按余弦函数衰减

# 3. 阶梯式衰减
from torch.optim.lr_scheduler import StepLR

scheduler = StepLR(optimizer, step_size=1000, gamma=0.1)

# 每1000步，学习率乘以0.1
```

**为什么需要预热（Warmup）？**

```
问题：训练初期，模型参数随机初始化，梯度可能很大

解决：
1. 学习率从小开始（如1e-7）
2. 逐步增大到目标学习率（如5e-5）
3. 避免训练初期参数更新过大导致不稳定

示例：
Step 0:    lr = 1e-7
Step 500:  lr = 2.5e-5
Step 1000: lr = 5e-5  ← 预热结束
```

### LoRA

**定义**：Low-Rank Adaptation，低秩适配。

**核心思想**：
- 冻结预训练模型参数
- 只训练少量低秩矩阵

```
原始矩阵更新：
W' = W + ΔW
ΔW ∈ R^(d×d)  # 需要训练 d×d 个参数

LoRA：
W' = W + BA
B ∈ R^(d×r), A ∈ R^(r×d)  # 只需要训练 2×d×r 个参数

当 r << d 时，参数量大幅减少
示例：d=4096, r=8
原始：4096×4096 = 16,777,216 参数
LoRA：2×4096×8 = 65,536 参数（减少256倍）
```

**实现**：

```python
import torch.nn as nn

class LoRALayer(nn.Module):
    def __init__(self, in_features, out_features, rank=8, alpha=16):
        super().__init__()
        self.rank = rank
        self.alpha = alpha
        
        # 低秩矩阵
        self.lora_A = nn.Parameter(torch.randn(in_features, rank) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))
        
        # 缩放因子
        self.scaling = alpha / rank
    
    def forward(self, x):
        # LoRA输出
        lora_output = (x @ self.lora_A @ self.lora_B) * self.scaling
        return lora_output

# 使用LoRA微调
from peft import get_peft_model, LoraConfig, TaskType

peft_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,                    # 秩
    lora_alpha=16,          # 缩放因子
    lora_dropout=0.1,       # Dropout
    target_modules=["q_proj", "v_proj"]  # 应用LoRA的模块
)

model = AutoModelForCausalLM.from_pretrained("gpt2")
model = get_peft_model(model, peft_config)

# 只有LoRA参数会被训练
model.print_trainable_parameters()
# 输出：trainable params: 294,912 || all params: 124,439,808 || trainable%: 0.24
```

**LoRA优势**：
- ✅ 参数量减少100-1000倍
- ✅ 显存占用减少
- ✅ 训练速度快
- ✅ 可以保存多个LoRA版本（不同任务）

### Prefix Tuning

**定义**：在输入前添加可训练的前缀。

```python
# 原始输入
input_ids: [101, 2023, 2003, ...]  # "This is ..."

# Prefix Tuning
prefix: [trainable_1, trainable_2, ..., trainable_n]  # n个可训练token
input_ids: [prefix, 101, 2023, 2003, ...]

# 只训练prefix参数，模型参数冻结
```

**实现**：

```python
class PrefixEncoder(nn.Module):
    def __init__(self, prefix_length, hidden_size):
        super().__init__()
        self.prefix_length = prefix_length
        
        # 可训练的prefix embedding
        self.prefix_embedding = nn.Parameter(
            torch.randn(prefix_length, hidden_size)
        )
    
    def forward(self, batch_size):
        # 扩展到batch维度
        prefix = self.prefix_embedding.unsqueeze(0).expand(batch_size, -1, -1)
        return prefix

# 使用
prefix_encoder = PrefixEncoder(prefix_length=10, hidden_size=768)
prefix = prefix_encoder(batch_size=32)  # (32, 10, 768)

# 拼接到输入
input_embeds = model.get_input_embeddings()(input_ids)  # (32, seq_len, 768)
input_embeds = torch.cat([prefix, input_embeds], dim=1)  # (32, 10+seq_len, 768)

# 前向传播
outputs = model(inputs_embeds=input_embeds)
```

## RLHF强化学习

### RLHF流程

**定义**：Reinforcement Learning from Human Feedback，从人类反馈中强化学习。

**三阶段流程**：

```
Stage 1: 预训练（Pretraining）
- 大规模无标注数据训练
- 目标：语言建模能力

Stage 2: 监督微调（SFT, Supervised Fine-Tuning）
- 高质量问答对训练
- 目标：回答问题的能力

Stage 3: 强化学习（RL）
- 人类偏好数据训练奖励模型
- 用奖励模型指导策略优化
- 目标：生成符合人类偏好的回答
```

### PPO算法

**定义**：Proximal Policy Optimization，近端策略优化。

**核心思想**：
- 限制策略更新幅度（防止训练崩溃）
- 使用重要性采样（利用旧策略的数据）

**算法流程**：

```python
# 1. 采样：使用当前策略生成回答
prompts = ["What is AI?", "Explain quantum computing", ...]
responses = policy_model.generate(prompts)

# 2. 计算奖励：使用奖励模型评分
rewards = reward_model(prompts, responses)

# 3. 计算优势函数（Advantage）
advantages = compute_advantages(rewards, values)

# 4. PPO损失
def ppo_loss(old_log_probs, new_log_probs, advantages, clip_epsilon=0.2):
    # 重要性采样比率
    ratio = torch.exp(new_log_probs - old_log_probs)
    
    # 裁剪目标
    clipped_ratio = torch.clamp(ratio, 1 - clip_epsilon, 1 + clip_epsilon)
    
    # PPO损失（取最小值，保守更新）
    loss = -torch.min(
        ratio * advantages,
        clipped_ratio * advantages
    ).mean()
    
    return loss

# 5. 更新策略
for epoch in range(ppo_epochs):
    new_log_probs = policy_model.get_log_probs(prompts, responses)
    loss = ppo_loss(old_log_probs, new_log_probs, advantages)
    loss.backward()
    optimizer.step()
```

**为什么要裁剪比率？**

```
问题：策略更新太大 → 训练不稳定 → 性能崩溃

解决：限制比率在 [1-ε, 1+ε] 范围内（如 [0.8, 1.2]）
- 如果新策略比旧策略好太多 → 裁剪到1.2
- 如果新策略比旧策略差太多 → 裁剪到0.8
- 确保平滑更新
```

### DPO算法

**定义**：Direct Preference Optimization，直接偏好优化。

**创新**：
- 不需要训练奖励模型
- 直接从偏好数据优化策略

**训练数据格式**：
```python
# 偏好数据：(prompt, chosen_response, rejected_response)
data = [
    {
        "prompt": "What is AI?",
        "chosen": "AI is artificial intelligence, which...",    # 人类偏好
        "rejected": "AI is a robot that can think."            # 不偏好
    },
    ...
]
```

**DPO损失**：

```python
def dpo_loss(policy_model, ref_model, prompt, chosen, rejected, beta=0.1):
    # 策略模型的log概率
    policy_chosen_logp = policy_model.get_log_prob(prompt, chosen)
    policy_rejected_logp = policy_model.get_log_prob(prompt, rejected)
    
    # 参考模型的log概率（冻结）
    ref_chosen_logp = ref_model.get_log_prob(prompt, chosen)
    ref_rejected_logp = ref_model.get_log_prob(prompt, rejected)
    
    # DPO损失
    loss = -torch.log(torch.sigmoid(
        beta * (policy_chosen_logp - ref_chosen_logp) -
        beta * (policy_rejected_logp - ref_rejected_logp)
    )).mean()
    
    return loss

# 目标：
# - 增大 chosen 的概率
# - 减小 rejected 的概率
# - 不偏离参考模型太远（防止过拟合）
```

**DPO vs PPO**：

| 特性 | PPO | DPO |
|------|-----|-----|
| **需要奖励模型** | ✅ 需要 | ❌ 不需要 |
| **训练复杂度** | 高（两个模型） | 低（一个模型） |
| **稳定性** | 较不稳定 | 较稳定 |
| **效果** | 稍好 | 接近 |

## 模型量化

### 量化原理

**定义**：将高精度权重（FP32/FP16）转换为低精度（INT8/INT4）。

**动机**：
- 减少模型大小（4倍~16倍）
- 降低推理时间（2倍~4倍）
- 降低显存占用

**量化公式**：

```
量化：
x_quantized = round((x - zero_point) / scale)

反量化：
x_dequantized = x_quantized * scale + zero_point

参数计算：
scale = (x_max - x_min) / (q_max - q_min)
zero_point = q_min - x_min / scale

示例：FP32 → INT8
x_min = -10.0, x_max = 10.0
q_min = -128, q_max = 127

scale = (10 - (-10)) / (127 - (-128)) = 20 / 255 ≈ 0.0784
zero_point = -128 - (-10) / 0.0784 ≈ 0

x = 5.0:
  量化：round((5.0 - 0) / 0.0784) = 64
  反量化：64 * 0.0784 + 0 = 5.018  # 略有误差
```

---

### PTQ训练后量化

**定义**：Post-Training Quantization，模型训练完成后再量化。

**步骤**：
1. 收集激活值分布（Calibration）
2. 计算量化参数（scale, zero_point）
3. 量化权重和激活值

```python
import torch.quantization as quantization

# 1. 加载模型
model = MyModel()
model.eval()

# 2. 配置量化
model.qconfig = quantization.get_default_qconfig('fbgemm')  # x86 CPU
quantization.prepare(model, inplace=True)

# 3. 校准（Calibration）：收集激活值分布
with torch.no_grad():
    for batch in calibration_dataloader:
        model(batch)

# 4. 转换为量化模型
quantization.convert(model, inplace=True)

# 5. 推理
output = model(input)

# 模型大小对比
# FP32: 100MB
# INT8: 25MB（减少4倍）
```

**优点**：
- ✅ 简单（不需要重新训练）
- ✅ 快速

**缺点**：
- ⚠️ 精度损失（1-5%）

---

### QAT量化感知训练

**定义**：Quantization-Aware Training，训练时模拟量化。

**原理**：
- 前向传播：量化→反量化（模拟量化误差）
- 反向传播：FP32梯度（正常更新）

```python
import torch.quantization as quantization

# 1. 配置量化
model.qconfig = quantization.get_default_qat_qconfig('fbgemm')
quantization.prepare_qat(model, inplace=True)

# 2. 训练（模拟量化）
model.train()
for epoch in range(epochs):
    for batch in train_dataloader:
        outputs = model(batch)  # 内部进行 量化→反量化
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

# 3. 转换为真正的量化模型
model.eval()
quantization.convert(model, inplace=True)

# 4. 推理
output = model(input)
```

**优点**：
- ✅ 精度损失小（< 1%）
- ✅ 适合对精度敏感的任务

**缺点**：
- ⚠️ 需要重新训练（时间长）

### AWQ

**定义**：Activation-aware Weight Quantization，激活感知的权重量化。

**核心思想**：

- 重要通道的权重使用高精度
- 不重要通道的权重激进量化

### KV Cache

**问题**：自回归生成时，每次都要重新计算之前token的K和V。

```python
# 生成 "Hello world!"
# Step 1: 输入 "Hello"
#   计算 K_hello, V_hello
#   输出 "world"

# Step 2: 输入 "Hello world"
#   重新计算 K_hello, V_hello  ← 浪费！
#   计算 K_world, V_world
#   输出 "!"

# Step 3: 输入 "Hello world !"
#   重新计算 K_hello, V_hello, K_world, V_world  ← 浪费！
#   计算 K_!
#   输出 <EOS>
```

**KV Cache优化**：
```python
# 缓存已计算的K和V
cache = {
    "past_key_values": []  # 存储每层的 (K, V)
}

# Step 1: 输入 "Hello"
outputs = model(input_ids=[101], past_key_values=None)
cache["past_key_values"] = outputs.past_key_values  # 保存 K_hello, V_hello

# Step 2: 输入 "world" （只计算新token）
outputs = model(input_ids=[102], past_key_values=cache["past_key_values"])
cache["past_key_values"] = outputs.past_key_values  # 更新缓存

# Step 3: 输入 "!"
outputs = model(input_ids=[103], past_key_values=cache["past_key_values"])

# 加速：3倍（避免重复计算）
```

**KV Cache的显存占用**：
```
每层的KV Cache大小 = 2 * batch_size * seq_len * hidden_size * 2字节（FP16）

示例：Llama-2-7B（32层）
- batch_size=1, seq_len=2048, hidden_size=4096
- KV Cache = 2 * 1 * 2048 * 4096 * 32 * 2 = 1GB

生成长文本时，KV Cache可能占用大量显存！
```

## 分布式训练

### 数据并行

**定义**：每个GPU有完整模型副本，处理不同数据。

```python
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

# 初始化分布式环境
dist.init_process_group(backend="nccl")

# 创建模型
model = MyModel().to(device)
model = DDP(model, device_ids=[local_rank])

# 训练
for batch in train_dataloader:
    outputs = model(batch)
    loss = outputs.loss
    loss.backward()
    
    # 自动同步梯度（all-reduce）
    optimizer.step()
    optimizer.zero_grad()
```

**通信模式**：
```
All-Reduce（梯度同步）：
GPU 0: grad_0
GPU 1: grad_1
GPU 2: grad_2
GPU 3: grad_3
    ↓
每个GPU：(grad_0 + grad_1 + grad_2 + grad_3) / 4
```

---

### 模型并行

**定义**：模型分割到多个GPU，每个GPU处理模型的一部分。

```python
# 示例：将模型分割到2个GPU
class ModelParallel(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(1000, 1000).to('cuda:0')
        self.layer2 = nn.Linear(1000, 1000).to('cuda:0')
        self.layer3 = nn.Linear(1000, 1000).to('cuda:1')  # 放到GPU 1
        self.layer4 = nn.Linear(1000, 10).to('cuda:1')
    
    def forward(self, x):
        x = x.to('cuda:0')
        x = self.layer1(x)
        x = self.layer2(x)
        
        x = x.to('cuda:1')  # 数据移动到GPU 1
        x = self.layer3(x)
        x = self.layer4(x)
        return x
```

**问题**：GPU利用率低（前一层计算时，后一层空闲）

### 流水线并行

**定义**：将batch分成多个micro-batch，流水线执行。

```
流水线并行（Pipeline Parallelism）：
Time → 
GPU 0: [Micro-batch 1] [Micro-batch 2] [Micro-batch 3]
GPU 1:                 [Micro-batch 1] [Micro-batch 2] [Micro-batch 3]

提高GPU利用率！
```

```python
from torch.distributed.pipeline.sync import Pipe

# 将模型分成多个阶段
model = nn.Sequential(
    nn.Linear(1000, 1000),  # Stage 0
    nn.ReLU(),
    nn.Linear(1000, 1000),  # Stage 1
    nn.ReLU(),
    nn.Linear(1000, 10)     # Stage 2
)

# 流水线并行
model = Pipe(model, chunks=8)  # 将batch分成8个micro-batch

# 训练
outputs = model(inputs)
```

### ZeRO优化

**定义**：Zero Redundancy Optimizer，零冗余优化器。

**核心思想**：分割优化器状态、梯度、参数到多个GPU。

**三个阶段**：

```
ZeRO Stage 1：分割优化器状态
- 每个GPU只保存部分优化器状态（m, v）
- 显存减少：4倍（对于Adam）

ZeRO Stage 2：分割梯度
- 每个GPU只保存部分梯度
- 显存减少：8倍

ZeRO Stage 3：分割参数
- 每个GPU只保存部分参数
- 前向传播时，动态收集参数
- 反向传播后，立即释放参数
- 显存减少：N倍（N=GPU数量）

示例：8个GPU训练GPT-3（175B参数）
- 无ZeRO：每个GPU需要 700GB 显存（训练不了）
- ZeRO Stage 3：每个GPU需要 88GB 显存（可训练）
```

```python
from deepspeed import zero

# 配置
ds_config = {
    "zero_optimization": {
        "stage": 3,                      # ZeRO Stage 3
        "offload_optimizer": {
            "device": "cpu",             # 优化器状态卸载到CPU
            "pin_memory": True
        },
        "offload_param": {
            "device": "cpu",             # 参数卸载到CPU
            "pin_memory": True
        }
    }
}

# 初始化
model_engine, optimizer, _, _ = deepspeed.initialize(
    model=model,
    optimizer=optimizer,
    config=ds_config
)

# 训练
for batch in train_dataloader:
    loss = model_engine(batch)
    model_engine.backward(loss)
    model_engine.step()
```

## 常见面试题

### 1. LoRA为什么能减少参数量？

**答**：
- LoRA将权重更新分解为低秩矩阵：ΔW = BA
- B∈R^(d×r), A∈R^(r×d)，参数量：2dr
- 完整更新：ΔW∈R^(d×d)，参数量：d²
- 当r << d时，2dr << d²（减少100-1000倍）

---

### 2. RLHF为什么需要奖励模型？

**答**：
- 人类偏好数据是稀疏的（只有对比结果，没有具体分数）
- 奖励模型将偏好转换为连续的奖励信号
- 用于指导策略优化（PPO）

---

### 3. 量化为什么能加速推理？

**答**：
1. **减少内存带宽**：INT8比FP32小4倍，读取更快
2. **硬件加速**：INT8有专门的指令集（如VNNI、Tensor Core）
3. **缓存友好**：模型更小，更多参数可以放入缓存

---

### 4. KV Cache如何节省计算？

**答**：
- 自回归生成时，之前token的K和V不变
- 缓存已计算的K和V，只计算新token
- 避免重复计算（加速3倍）

---

### 5. ZeRO Stage 3如何减少显存？

**答**：
- 将参数分割到多个GPU
- 前向传播时，动态收集需要的参数
- 反向传播后，立即释放参数
- 显存减少：N倍（N=GPU数量）
