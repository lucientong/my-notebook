### 1.2 残差连接原理 ⭐⭐⭐

```
传统网络:  x → [Conv → BN → ReLU → Conv → BN] → H(x)

残差网络:  x → [Conv → BN → ReLU → Conv → BN] → F(x) + x → ReLU
                    │                                 ↑
                    └─────── Shortcut Connection ─────┘

核心思想: 学习残差 F(x) = H(x) - x 比直接学习 H(x) 更容易
- 如果最优映射接近恒等映射，F(x)→0 比 H(x)→x 更容易优化
- 解决了深层网络的梯度消失/退化问题
```

```python
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, in_channels, out_channels, stride=1):
        super().__init__()
        self.conv1 = nn.Conv2d(in_channels, out_channels, 3, stride, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.conv2 = nn.Conv2d(out_channels, out_channels, 3, 1, 1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # Shortcut: 维度不匹配时用1×1卷积调整
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels, 1, stride, bias=False),
                nn.BatchNorm2d(out_channels)
            )
    
    def forward(self, x):
        residual = self.shortcut(x)
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += residual  # 残差连接
        return torch.relu(out)
```

## 2. Vision Transformer

### 2.1 ViT 原理 ⭐⭐⭐

```
输入图像 (H×W×C)
    │
    ▼
Patch Embedding: 切分为 N 个 patch → 线性投影为 D 维向量
    │  N = (H/P) × (W/P)，P=patch大小(通常16)
    │  例: 224×224 图像，P=16 → N=196 个 patch
    ▼
添加 [CLS] token + 位置编码 (Position Embedding)
    │  输入序列: [CLS, patch_1, patch_2, ..., patch_N] ∈ R^{(N+1)×D}
    ▼
L 层 Transformer Encoder
    │  每层: LayerNorm → MSA → Residual → LayerNorm → MLP → Residual
    ▼
[CLS] token 输出 → 分类头 (MLP Head)
    │
    ▼
类别预测
```

**数学表达**：
```
Patch Embedding:  z_0 = [x_cls; x_1E; x_2E; ...; x_NE] + E_pos
                  E ∈ R^{(P²·C)×D}, E_pos ∈ R^{(N+1)×D}

Transformer:      z'_l = MSA(LN(z_{l-1})) + z_{l-1}
                  z_l  = MLP(LN(z'_l)) + z'_l

Output:           y = MLP_head(LN(z_L^0))  # 取CLS token
```

### 2.2 ViT 实现核心 ⭐⭐⭐

```python
import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    """将图像切分为patch并投影到embedding空间"""
    def __init__(self, img_size=224, patch_size=16, in_channels=3, embed_dim=768):
        super().__init__()
        self.num_patches = (img_size // patch_size) ** 2
        # 用卷积实现patch切分+线性投影（等价于reshape+linear）
        self.projection = nn.Conv2d(
            in_channels, embed_dim, 
            kernel_size=patch_size, stride=patch_size
        )
    
    def forward(self, x):
        # x: (B, C, H, W) → (B, embed_dim, H/P, W/P) → (B, num_patches, embed_dim)
        x = self.projection(x)  # (B, D, H/P, W/P)
        x = x.flatten(2).transpose(1, 2)  # (B, N, D)
        return x


class MultiHeadAttention(nn.Module):
    def __init__(self, embed_dim=768, num_heads=12, dropout=0.0):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads
        self.scale = self.head_dim ** -0.5
        
        self.qkv = nn.Linear(embed_dim, embed_dim * 3)
        self.proj = nn.Linear(embed_dim, embed_dim)
        self.attn_dropout = nn.Dropout(dropout)
    
    def forward(self, x):
        B, N, D = x.shape
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, self.head_dim)
        qkv = qkv.permute(2, 0, 3, 1, 4)  # (3, B, heads, N, head_dim)
        q, k, v = qkv.unbind(0)
        
        attn = (q @ k.transpose(-2, -1)) * self.scale  # (B, heads, N, N)
        attn = attn.softmax(dim=-1)
        attn = self.attn_dropout(attn)
        
        x = (attn @ v).transpose(1, 2).reshape(B, N, D)
        return self.proj(x)


class ViT(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_channels=3,
                 num_classes=1000, embed_dim=768, depth=12, num_heads=12):
        super().__init__()
        self.patch_embed = PatchEmbedding(img_size, patch_size, in_channels, embed_dim)
        num_patches = self.patch_embed.num_patches
        
        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))
        self.pos_embed = nn.Parameter(torch.zeros(1, num_patches + 1, embed_dim))
        
        self.blocks = nn.ModuleList([
            TransformerBlock(embed_dim, num_heads) for _ in range(depth)
        ])
        self.norm = nn.LayerNorm(embed_dim)
        self.head = nn.Linear(embed_dim, num_classes)
    
    def forward(self, x):
        B = x.shape[0]
        x = self.patch_embed(x)  # (B, N, D)
        
        cls_tokens = self.cls_token.expand(B, -1, -1)
        x = torch.cat([cls_tokens, x], dim=1)  # (B, N+1, D)
        x = x + self.pos_embed
        
        for block in self.blocks:
            x = block(x)
        
        x = self.norm(x)
        return self.head(x[:, 0])  # CLS token
```

### 2.3 ViT vs CNN 对比 ⭐⭐⭐

| 维度 | CNN | ViT |
|------|-----|-----|
| **归纳偏置** | 局部性 + 平移等变性（内置） | 几乎无（需要大量数据学习） |
| **感受野** | 逐层扩大（局部→全局） | 第一层就是全局注意力 |
| **数据需求** | 中小数据集即可 | 需要大数据（ImageNet-21K+）或数据增强 |
| **计算复杂度** | O(K²·C²·HW) | O(N²·D)，N=patch数 |
| **可解释性** | 特征图可视化 | Attention Map |
| **扩展性** | 受限于架构设计 | 随模型/数据规模提升明显 |
| **小数据表现** | ✅ 更好 | ❌ 容易过拟合 |
| **大数据表现** | ❌ 饱和 | ✅ Scaling Law明显 |

### 2.4 DeiT 数据高效训练

**核心创新**：无需ImageNet-21K预训练，仅用ImageNet-1K即可训练ViT

- **知识蒸馏**：用CNN教师模型（如RegNet）指导ViT学生模型
- **蒸馏Token**：在CLS token旁添加一个distillation token
- **数据增强**：RandAugment、Mixup、CutMix、Random Erasing

### 2.5 Swin Transformer ⭐⭐⭐

```
Swin Transformer 关键创新:

1. 层级特征图（类似CNN的多尺度）
   Stage 1: H/4 × W/4    (细粒度)
   Stage 2: H/8 × W/8
   Stage 3: H/16 × W/16
   Stage 4: H/32 × W/32  (粗粒度)

2. 窗口注意力（Window Attention）
   - 将特征图划分为不重叠的 M×M 窗口
   - 注意力只在窗口内计算: O(M²·N) 替代 O(N²)
   
3. 移位窗口（Shifted Window）
   Layer L:   [W0|W1|W2|W3]  → 窗口内注意力
   Layer L+1: 窗口移位M/2   → 跨窗口信息交互

原始ViT: O(N²) = O((H/P × W/P)²)   N=196时: 38,416
Swin:    O(M²·N) = O(49·N)           M=7时: 9,604 (节省4倍)
```

## 3. 多模态大模型

### 3.1 CLIP（Contrastive Language-Image Pre-training）⭐⭐⭐

```
CLIP 双塔架构:

Image Encoder (ViT/ResNet)          Text Encoder (Transformer)
       │                                     │
   image_embed                           text_embed
       │                                     │
       └───────── 对比学习 ──────────────────┘
           (余弦相似度矩阵 + InfoNCE Loss)

训练: 
- 正样本: 匹配的(image, text)对，拉近距离
- 负样本: batch内其他不匹配的组合，推远距离
- 损失: 对称的InfoNCE = L_img→text + L_text→img
```

**Zero-Shot 分类**：
```python
import clip
import torch
from PIL import Image

model, preprocess = clip.load("ViT-B/32", device="cuda")

image = preprocess(Image.open("cat.jpg")).unsqueeze(0).to("cuda")
text = clip.tokenize(["a photo of a cat", "a photo of a dog", "a photo of a car"]).to("cuda")

with torch.no_grad():
    image_features = model.encode_image(image)
    text_features = model.encode_text(text)
    
    # 归一化
    image_features /= image_features.norm(dim=-1, keepdim=True)
    text_features /= text_features.norm(dim=-1, keepdim=True)
    
    # 计算相似度
    similarity = (100.0 * image_features @ text_features.T).softmax(dim=-1)
    print(similarity)  # tensor([[0.95, 0.03, 0.02]])
```

### 4.1 融合策略对比 ⭐⭐⭐

| 策略 | 融合时机 | 优点 | 缺点 | 代表模型 |
|------|---------|------|------|---------|

### 4.2 Cross-Attention 机制 ⭐⭐⭐

```
Cross-Attention:
Q 来自文本, K/V 来自图像 → 文本"关注"图像的相关区域

Text Tokens → Q = W_q × text_hidden
Image Tokens → K = W_k × image_hidden
               V = W_v × image_hidden

Attention = softmax(Q × K^T / √d_k) × V
```

```python
class CrossAttention(nn.Module):
    def __init__(self, text_dim, image_dim, num_heads=8):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = text_dim // num_heads
        
        self.q_proj = nn.Linear(text_dim, text_dim)
        self.k_proj = nn.Linear(image_dim, text_dim)
        self.v_proj = nn.Linear(image_dim, text_dim)
        self.out_proj = nn.Linear(text_dim, text_dim)
    
    def forward(self, text_hidden, image_hidden):
        B, T, D = text_hidden.shape
        _, N, _ = image_hidden.shape
        
        q = self.q_proj(text_hidden).reshape(B, T, self.num_heads, self.head_dim).transpose(1, 2)
        k = self.k_proj(image_hidden).reshape(B, N, self.num_heads, self.head_dim).transpose(1, 2)
        v = self.v_proj(image_hidden).reshape(B, N, self.num_heads, self.head_dim).transpose(1, 2)
        
        attn = (q @ k.transpose(-2, -1)) / (self.head_dim ** 0.5)
        attn = attn.softmax(dim=-1)
        
        out = (attn @ v).transpose(1, 2).reshape(B, T, D)
        return self.out_proj(out)
```

### 4.3 Q-Former（BLIP-2）⭐⭐⭐

```
Q-Former 架构:

Learnable Queries (32个) ──→ ┌──────────────┐
                              │   Q-Former    │
Image Features (ViT) ──────→ │ (Cross-Attn + │ ──→ 32个Visual Tokens → LLM
                              │  Self-Attn)   │
                              └──────────────┘

核心思想:
- 用固定数量(32)的可学习Query向量从图像中"提取"信息
- 将任意长度的视觉特征压缩为固定32个Token
- 大幅减少送入LLM的Token数（ViT输出257 → 32）

三阶段预训练:
1. Image-Text Contrastive Learning (ITC)
2. Image-grounded Text Generation (ITG)  
3. Image-Text Matching (ITM)
```

### 4.4 Perceiver 架构

```
Perceiver: 通用的模态无关架构

任意模态输入 (N个token) → Cross-Attention → Latent Array (M个, M<<N) → Self-Attention → 输出

优势: 输入长度N可以很大（百万像素/长音频），计算量只与Latent数M有关
```

## 5. 图像生成

### 5.1 Diffusion Model 原理 ⭐⭐

```
前向过程（加噪）:
x_0 → x_1 → x_2 → ... → x_T (纯高斯噪声)
     +ε_1    +ε_2         +ε_T

q(x_t | x_{t-1}) = N(x_t; √(1-β_t)·x_{t-1}, β_t·I)

反向过程（去噪）:
x_T → x_{T-1} → ... → x_1 → x_0 (生成图像)
  模型预测每一步的噪声 ε_θ(x_t, t)

训练目标: L = E[||ε - ε_θ(x_t, t)||²]
即: 模型学习预测在时间步t添加到x_0上的噪声
```

**DDPM vs DDIM**：

| 方面 | DDPM | DDIM |
|------|------|------|
| 采样步数 | ~1000步 | ~50步 |
| 采样过程 | 随机（马尔可夫） | 确定性（非马尔可夫） |
| 质量 | 高 | 相当 |
| 速度 | 慢 | 快20倍 |
| 可逆性 | 不可逆 | 可逆（可做图像编辑） |

### 5.2 Stable Diffusion 架构 ⭐⭐

```
Stable Diffusion = VAE + U-Net + CLIP Text Encoder

文本 "a cat on the moon"
  │
  ▼
[CLIP Text Encoder] → Text Embeddings (77×768)
                           │
                    Cross-Attention
                           │
[Random Noise z_T] → [U-Net Denoiser] → [Predicted z_0] → [VAE Decoder] → Image
  (latent space)     (迭代T步去噪)       (latent space)     (4×64×64→3×512×512)

关键: 在Latent Space(4×64×64)而非Pixel Space(3×512×512)做扩散
```

### 10.1 图像搜索（CLIP Embedding）

```python
import clip
import torch
import numpy as np
from PIL import Image

model, preprocess = clip.load("ViT-B/32", device="cuda")

# 索引阶段：提取所有图片的embedding
def extract_image_embedding(image_path):
    image = preprocess(Image.open(image_path)).unsqueeze(0).to("cuda")
    with torch.no_grad():
        embedding = model.encode_image(image)
        embedding /= embedding.norm(dim=-1, keepdim=True)
    return embedding.cpu().numpy().flatten()

# 查询阶段：文本搜索图片
def search_by_text(query_text, image_embeddings, top_k=5):
    text = clip.tokenize([query_text]).to("cuda")
    with torch.no_grad():
        text_embedding = model.encode_text(text)
        text_embedding /= text_embedding.norm(dim=-1, keepdim=True)
    
    similarities = np.dot(image_embeddings, text_embedding.cpu().numpy().T).flatten()
    top_indices = np.argsort(similarities)[::-1][:top_k]
    return top_indices, similarities[top_indices]
```

```python
class ContentModerator:
    def __init__(self):
        self.clip_model, self.preprocess = clip.load("ViT-L/14")
        self.unsafe_categories = [
            "violent content", "explicit content", "hate speech imagery",
            "self-harm", "illegal activity"
        ]
        self.text_features = self._encode_categories()
    
    def _encode_categories(self):
        texts = clip.tokenize([f"a photo of {c}" for c in self.unsafe_categories])
        with torch.no_grad():
            features = self.clip_model.encode_text(texts)
            features /= features.norm(dim=-1, keepdim=True)
        return features
    
    def check_image(self, image_path, threshold=0.3):
        image = self.preprocess(Image.open(image_path)).unsqueeze(0)
        with torch.no_grad():
            image_features = self.clip_model.encode_image(image)
            image_features /= image_features.norm(dim=-1, keepdim=True)
        
        similarities = (image_features @ self.text_features.T).squeeze()
        
        violations = []
        for i, (category, score) in enumerate(zip(self.unsafe_categories, similarities)):
            if score > threshold:
                violations.append({"category": category, "score": score.item()})
        
        return {"safe": len(violations) == 0, "violations": violations}
```

## 11. 实战案例

### 构建多模态 RAG 系统

```
用户查询 (文本/图片)
      │
      ▼
┌───────────────┐
│ Query Encoder │ → Query Embedding
│ (CLIP/BGE-M3) │
└───────┬───────┘
        │
        ▼
┌───────────────────┐
│  Hybrid Retrieval  │
│ 文本: BM25 + Dense │ → Top-K 文档(含图文)
│ 图片: CLIP向量     │
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│  Multimodal LLM   │
│ (LLaVA/GPT-4V)   │ → 结合检索结果生成回答
│ Context: 文本+图片 │
└───────────────────┘
```

```python
class MultimodalRAG:
    def __init__(self):
        self.clip_model, self.preprocess = clip.load("ViT-L/14")
        self.text_embedder = SentenceTransformer("BAAI/bge-m3")
        self.vector_store = MilvusClient("milvus.db")
        self.llm = LLaVAModel("llava-1.5-13b")
    
    def index_document(self, doc):
        """索引文档（支持图文混合）"""
        # 文本embedding
        if doc.text:
            text_emb = self.text_embedder.encode(doc.text)
            self.vector_store.insert("text_collection", {
                "embedding": text_emb, "content": doc.text, "doc_id": doc.id
            })
        
        # 图片embedding
        if doc.images:
            for img_path in doc.images:
                img_emb = self._encode_image(img_path)
                self.vector_store.insert("image_collection", {
                    "embedding": img_emb, "image_path": img_path, "doc_id": doc.id
                })
    
    def query(self, question, image=None):
        """多模态查询"""
        # 文本检索
        query_emb = self.text_embedder.encode(question)
        text_results = self.vector_store.search("text_collection", query_emb, top_k=5)
        
        # 图片检索（如果有图片查询）
        image_results = []
        if image:
            img_emb = self._encode_image(image)
            image_results = self.vector_store.search("image_collection", img_emb, top_k=3)
        
        # 构建多模态Context
        context_texts = [r.content for r in text_results]
        context_images = [r.image_path for r in image_results]
        
        # LLM生成
        prompt = self._build_prompt(question, context_texts, context_images)
        response = self.llm.generate(prompt, images=context_images)
        
        return response
    
    def _encode_image(self, image_path):
        image = self.preprocess(Image.open(image_path)).unsqueeze(0)
        with torch.no_grad():
            features = self.clip_model.encode_image(image)
            features /= features.norm(dim=-1, keepdim=True)
        return features.cpu().numpy().flatten()
    
    def _build_prompt(self, question, texts, images):
        context = "\n".join([f"[文档{i+1}] {t}" for i, t in enumerate(texts)])
        img_refs = "\n".join([f"<image_{i+1}>" for i in range(len(images))])
        return f"""基于以下参考资料回答问题。

参考文本:
{context}

参考图片:
{img_refs}

问题: {question}
回答:"""
```

## 12. 面试题自查

### 多模态治理补充

- 视觉编码器的选择直接决定了多模态模型的上限，但连接方式和训练策略同样关键——CLIP ViT-L配简单Linear Projection在LLaVA中已经取得了很好的效果。
- 多模态系统的评估比纯文本更困难：需要同时评估感知能力（图片理解）和推理能力（基于图片的逻辑推断），不能只看单一benchmark。
- 部署多模态模型时，视觉编码器的推理开销往往被忽视——一张图片经ViT-L产生256个token，相当于增加了大量的LLM输入，对延迟和成本影响显著。

### 初级高频速刷

**1. ViT和CNN的核心区别是什么？ViT为什么需要大量数据？**

CNN内置了局部性和平移等变性的归纳偏置，小数据集也能学到有用特征；ViT的Self-Attention是全局操作，没有这些先验，需要从大量数据中学习这些模式。所以ViT在ImageNet-1K上不如CNN，但在ImageNet-21K+上反超。

**2. CLIP是什么？为什么它能做Zero-Shot分类？**

CLIP用对比学习在4亿图文对上训练了一个图像编码器和文本编码器，使匹配的图文对在embedding空间中接近。Zero-Shot时，将类别名转为文本embedding，与图像embedding计算相似度，选最相似的类别。不需要任何目标数据集的标注。

**3. Stable Diffusion为什么在Latent Space做扩散而不是Pixel Space？**

Pixel Space的图片维度是3×512×512=786,432，Latent Space压缩为4×64×64=16,384，减少了约48倍。在Latent Space做扩散的计算量和内存需求大幅降低，同时VAE保证了Latent Space和Pixel Space的高质量映射。

**4. 什么是Patch Embedding？为什么ViT要把图片切成Patch？**

Transformer本身处理的是序列，图像需要转化为token序列。Patch Embedding将图像切成16×16的小块，每个块通过线性投影变为一个token。224×224的图像产生196个token，这个长度Transformer可以高效处理。

**5. 多模态大模型中的视觉编码器和LLM是怎么连接的？**

主流有三种方式：(1) Linear/MLP Projection直接映射（LLaVA）；(2) Cross-Attention让LLM通过注意力机制查询视觉特征（Flamingo）；(3) Q-Former用可学习Query压缩视觉特征（BLIP-2）。方案1最简单但Token数多，方案3最高效但训练复杂。

### 中级高频进阶

**6. Swin Transformer相比ViT解决了什么问题？代价是什么？**

解决了两个问题：(1) 计算复杂度从O(N²)降到O(M²·N)，使高分辨率输入可行；(2) 引入层级特征图，适合检测/分割等密集预测任务。代价是引入了窗口大小这个超参数，且Shifted Window的实现增加了工程复杂度，同时失去了全局注意力的全程覆盖。

**7. 对比学习(Contrastive Learning)的InfoNCE Loss是什么？温度参数的作用？**

InfoNCE Loss = -log(exp(sim(i,j)/τ) / Σexp(sim(i,k)/τ))，最大化正样本相似度、最小化负样本相似度。温度τ控制分布的锐度：τ小时分布更尖锐（对负样本惩罚更强），τ大时分布更平滑。CLIP默认τ=0.07，是可学习参数。

**8. Diffusion Model的Classifier-Free Guidance(CFG)是什么？**

CFG在推理时同时预测有条件和无条件的噪声，然后用差值放大条件引导：`ε_guided = ε_uncond + w·(ε_cond - ε_uncond)`。w是guidance scale（通常7-15），w越大生成结果越符合文本描述但多样性降低。训练时随机以一定概率丢弃条件（用空文本替代）来同时学习有/无条件生成。

**9. Q-Former和直接Linear Projection相比，各自的优劣？**

Q-Former：优点是将任意长度视觉特征压缩为固定32个token，大幅减少LLM输入长度和推理成本；缺点是训练复杂（三阶段预训练），压缩可能丢失细节信息。Linear Projection：优点是简单直接，保留所有视觉信息（256个token）；缺点是token数多增加LLM推理开销。实践中LLaVA的简单方案效果反而很好，说明视觉编码器质量比连接方式更重要。

**10. 如何评估多模态大模型的能力？有哪些常用benchmark？**

常用benchmark按能力分：(1) 感知类：VQAv2（视觉问答）、TextVQA（图中文字理解）、GQA（组合推理）；(2) 推理类：MMMU（大学考试）、MathVista（数学推理）、ScienceQA（科学推理）；(3) 综合类：MMBench、MME、SEED-Bench（多维度评估）；(4) 幻觉类：POPE（对象幻觉检测）。单一benchmark不能反映全貌，需要综合看。

### 深入问答

#### Q1: ViT的位置编码为什么用可学习的而不是Sinusoidal？对不同分辨率输入怎么处理？

**答**：ViT用可学习位置编码是因为图像的2D空间关系比1D序列复杂，Sinusoidal编码难以直接表达2D位置。实验表明两者在ImageNet上差距很小，但可学习编码稍优。对于不同分辨率：训练时固定分辨率（如224），推理时如果输入336，patch数从196变为441，需要对位置编码做2D双线性插值。DeiT-III和FlexiViT进一步研究了分辨率自适应。

#### Q2: CLIP训练用了4亿图文对，如何理解其数据飞轮效应？数据质量和数量哪个更重要？

**答**：数据飞轮：CLIP→Zero-Shot标注新数据→过滤提升数据质量→训练更好的CLIP。研究表明：(1) 数据质量比数量重要——LAION-400M(噪声大)训练效果不如CLIP原始数据；(2) DataComp研究发现，用CLIP模型本身做数据过滤(CLIP Score过滤低质量图文对)可以大幅提升下一代模型质量；(3) SigLIP用Sigmoid Loss替代Softmax，减少了对大batch size的依赖。

#### Q3: Stable Diffusion的CFG Scale设太高或太低会怎样？如何选择合适的值？

**答**：CFG Scale(w)=1时等于无引导，生成结果随机性高但可能不符合描述；w=7-12是通常推荐范围；w>15时会出现过饱和、色彩异常、细节丢失（因为引导信号过强，生成偏离自然图像分布）。选择策略：(1) 写实风格用7-9；(2) 动漫/插画用9-12；(3) 严格遵循文本描述用12-15。另外可以用Dynamic CFG：前期用高w（构图准确），后期用低w（细节自然）。

#### Q4: 如何在生产环境中部署一个多模态搜索系统？延迟和成本如何优化？

**答**：(1) **架构**：CLIP Embedding离线预计算+Milvus/Faiss向量检索；(2) **Embedding缓存**：热门查询的embedding缓存到Redis；(3) **模型优化**：ViT用TensorRT/ONNX量化到INT8，推理延迟从50ms降到15ms；(4) **分级召回**：先用轻量CLIP-B/32粗筛Top100，再用CLIP-L/14精排Top20；(5) **异步Pipeline**：图片上传时异步计算embedding，不阻塞写入；(6) **成本控制**：批量推理(batch=32)比逐条推理吞吐提升10倍以上。

#### Q5: LLaVA为什么用两阶段训练而不是端到端一起训？

**答**：两阶段的设计逻辑：(1) Stage 1只训练Projection，相当于学一个"翻译器"把视觉信号映射到LLM的语义空间，此时ViT和LLM的知识都保持不变；(2) Stage 2解冻LLM（或用LoRA），让模型在对齐的基础上学习指令跟随能力。如果端到端一起训：ViT和LLM可能互相干扰导致灾难性遗忘，且需要更大的数据量才能稳定收敛。两阶段训练用更少数据（600K+158K）就达到了很好的效果。

#### Q6: 多模态模型的幻觉问题和纯文本LLM有什么不同？如何缓解？

**答**：多模态幻觉的独特性：(1) **对象幻觉**：描述图中不存在的物体（如把猫说成狗）；(2) **属性幻觉**：颜色/数量/位置描述错误；(3) **关系幻觉**：物体间关系描述错误。缓解方法：(1) 训练时加入对比否定样本（"图中没有X"）；(2) 高分辨率输入减少感知误差；(3) 推理时用POPE-style验证（先问"图中有X吗？"再描述）；(4) 多次采样+投票；(5) 外部视觉验证工具（目标检测模型交叉验证）。

#### Q7: ControlNet是怎么在不破坏原模型的情况下添加条件控制的？

**答**：ControlNet的核心设计是"零初始化"：(1) 复制SD U-Net的Encoder部分作为可训练副本；(2) 条件图（边缘/深度等）经过零初始化的卷积层输入副本；(3) 副本的输出通过零初始化的线性层加到原U-Net对应层。由于零初始化，训练开始时ControlNet的输出为零，不影响原模型行为，随着训练逐渐学习条件控制。这样原SD的生成质量完全保留，只增加了条件引导能力。

#### Q8: 视频多模态模型的Token数如何控制？32帧视频全部送入LLM可行吗？

**答**：不可行。32帧×256 token/帧=8192 token仅视觉部分就很大。控制方法：(1) **帧采样**：均匀采样8-16帧，关键帧检测进一步筛选；(2) **时间池化**：相邻帧的token做平均池化或取代表性token；(3) **Q-Former压缩**：每帧256→32 token，32帧→1024 token；(4) **分层理解**：先逐片段总结，再整体分析（类似MapReduce）；(5) **稀疏注意力**：只在关键时间步做完整注意力。实际系统通常控制视觉token在500-2000之间。

#### Q9: Diffusion Model如何实现图片编辑（如替换背景）而不重新生成整张图？

**答**：核心技术是DDIM Inversion + Prompt-to-Prompt：(1) **DDIM Inversion**：将原图通过确定性的DDIM反向过程映射回噪声空间（x_0 → x_T），得到对应的latent；(2) 修改文本条件，从x_T重新去噪；(3) **Attention Control**：在去噪过程中，保持Self-Attention的结构信息（维持布局），只修改Cross-Attention中与新prompt对应的部分；(4) **Inpainting**：用mask指定编辑区域，mask内用新prompt生成，mask外保持原图latent。SDEdit则更简单：给原图加部分噪声，用新prompt去噪。

#### Q10: CLIP的局限性有哪些？SigLIP和EVA-CLIP是怎么改进的？

**答**：CLIP局限：(1) 需要超大batch（32K）才能有足够负样本，GPU内存需求高；(2) 对细粒度理解弱（"红色杯子左边的蓝色盒子"）；(3) 长尾概念理解差；(4) 对抗样本脆弱。改进：**SigLIP**用Sigmoid Loss替代Softmax，每个图文对独立计算二分类loss，不需要大batch的负样本，batch=16K即可达到CLIP batch=32K的效果。**EVA-CLIP**用更大的ViT（1B参数）+更好的预训练策略（EVA预训练+CLIP微调），在保持效率的同时提升了表征质量。

#### Q11: 如何从头训练一个领域专用的多模态模型（如医疗影像理解）？

**答**：(1) **数据准备**：收集医疗影像+诊断报告对（X光/CT/MRI+放射科报告），规模建议>100K对；(2) **视觉编码器**：以BiomedCLIP或通用CLIP ViT为起点，在医疗影像上做领域适应微调；(3) **对齐训练**：类似LLaVA Stage 1，冻结ViT和LLM，训练Projection将医疗视觉特征对齐到LLM空间；(4) **指令微调**：构造医疗VQA数据（"这张X光显示什么异常？"），可用GPT-4V辅助生成；(5) **关键注意**：医疗领域对幻觉零容忍，必须加入Uncertainty Estimation和拒绝回答机制（"不确定时说不知道"），评估时需要临床专家参与。

#### Q12: 图像生成的评估指标FID和CLIP Score分别衡量什么？

**答**：**FID (Frechet Inception Distance)**：衡量生成图像分布与真实图像分布的距离，用Inception网络提取特征后计算两个特征分布的Frechet距离。FID越低越好，衡量的是"整体质量和多样性"。**CLIP Score**：衡量生成图像与文本描述的匹配度，用CLIP计算图文相似度。CLIP Score越高表示越符合文本。两者互补：一个评估图像质量，一个评估文本匹配度。另外IS（Inception Score）评估质量和多样性，LPIPS评估感知相似度。

#### Q13: 未来多模态AI的发展方向是什么？

**答**：(1) **原生多模态**：从设计之初就是多模态（如Gemini），而非在LLM上"接"视觉模块，模态间理解更深入；(2) **任意模态生成**：统一的模型同时理解和生成文本/图像/音频/视频（Unified-IO, CoDi）；(3) **具身智能**：多模态理解+机器人控制（RT-2, PaLM-E）；(4) **长视频理解**：突破Token长度限制，理解小时级视频；(5) **高效化**：MoE结构、稀疏注意力、更小但更强的视觉编码器；(6) **世界模型**：从图像/视频中学习物理规律和因果关系（Sora的方向）。

#### Q14: 如何做多模态模型的模型压缩和部署优化？

**答**：(1) **视觉编码器**：INT8量化（TensorRT）可减少50%推理时间且精度损失<1%；也可以用蒸馏把ViT-L蒸馏到ViT-B；(2) **LLM部分**：GPTQ/AWQ 4bit量化，或用更小的LLM（7B→1.8B）配合高质量蒸馏数据；(3) **Projection层**：通常参数量很小，不需要单独优化；(4) **推理优化**：Flash Attention 2、KV Cache、视觉Token缓存（同一图片多轮对话只编码一次）、Speculative Decoding；(5) **系统层面**：视觉编码和LLM推理分离部署，视觉部分用GPU池化处理。

#### Q15: 对比SAM（Segment Anything）和传统分割方法，SAM的革命性在哪里？

**答**：SAM的革命性：(1) **提示式分割**：用点/框/文本提示分割任意物体，不限于预定义类别（开放词汇）；(2) **零样本泛化**：在SA-1B(11M图像，1B掩码)上训练后，无需微调即可分割从未见过的物体类别；(3) **交互式**：支持多轮提示逐步精细化分割结果。架构创新：Image Encoder（一次编码复用）+ Prompt Encoder（轻量）+ Mask Decoder（快速），一张图片编码后可以亚秒级响应多个提示查询。局限：(1) 不做语义分类（只分割不识别）；(2) 细小物体和复杂边界仍有改进空间；(3) 视频场景需要SAM 2来处理时序一致性。
